3.3.4 Vnořené podmínky a logické operátory v JavaScriptu

Co se naučíš?

V této kapitole navážeme na if / else if / else a posuneme rozhodování na vyšší úroveň. Ukážeme si vnořené podmínky (nested if) a hlavně logické operátory, díky kterým dokážeš kombinovat více podmínek najednou.

  • Vnořený if: if uvnitř if – některé podmínky má smysl testovat až „po cestě“.
  • AND (&&): obě podmínky musí být pravda.
  • OR (||): stačí, když je pravda alespoň jedna.
  • NOT (!): obrátí hodnotu booleanu.
  • Praktické použití: přístupová práva, licence, slevy, víkendy, úkoly, kino lístky.
Tip: Když podmínky začnou být dlouhé, rozděl je na menší logické kroky (proměnné, vnoření, nebo else-if).
1) Vnořené podmínky (nested if)

Vnořený if je situace, kdy dáš jeden if dovnitř druhého. Je to užitečné ve chvíli, kdy má smysl kontrolovat další podmínku až ve chvíli, kdy první podmínka platí.

Příklad: přihlášení + role admin
let isLoggedIn = true;
let isAdmin = false;

if (isLoggedIn) {
  console.log("Vítej na stránce!");

  if (isAdmin) {
    console.log("Přístup do admin panelu povolen.");
  } else {
    console.log("Přístup do uživatelského dashboardu povolen.");
  }

} else {
  console.log("Nejsi přihlášený/á. Přihlas se.");
}

Jak to funguje: nejdřív se rozhodne, jestli je uživatel přihlášen. Teprve když ano, řeší se role (admin / uživatel).

2) Logické operátory (&&, ||, !)

Logické operátory umožňují kombinovat více podmínek do jednoho rozhodnutí. Používají se hlavně v if podmínkách.

  • AND (&&) – vrátí true jen tehdy, když jsou obě strany pravda.
  • OR (||) – vrátí true, když je pravda alespoň jedna strana.
  • NOT (!) – obrátí boolean: !true je false, !false je true.
&& (AND)
let age = 20;
let hasDriverLicense = true;

if (age >= 18 && hasDriverLicense) {
  console.log("Můžeš řídit.");
} else {
  console.log("Řídit nemůžeš.");
}
|| (OR)
let isHoliday = false;
let isWeekend = true;

if (isHoliday || isWeekend) {
  console.log("Je čas relaxovat!");
} else {
  console.log("Musíme pracovat.");
}
! (NOT)
let taskCompleted = false;

if (!taskCompleted) {
  console.log("Úkol ještě není hotový.");
}
Pozor: Logické operátory můžeš kombinovat (např. price > 0 && !isTuesday && hasLoyaltyCard). Tím vznikají silné, ale delší podmínky – udržuj je čitelné.
🧩 Cvičení: Kalkulačka ceny lístku do kina

Napiš skript, který určí cenu filmového lístku podle věku a speciálních slev. Loguj finální cenu do konzole.

Zadání

Věkové ceny:

  • Méně než 3 roky → zdarma
  • 3 až 12 let → 5 $
  • 13 až 17 let → 7 $
  • 18 až 64 let → 10 $
  • 65 a více → 6 $

Speciální slevy:

  • Pokud je úterý → všechny lístky jsou za 5 $ (bez ohledu na věk)
  • Pokud má věrnostní kartu → sleva 2 $ z normální ceny
  • Slevy neplatí na lístky zdarma
Tip: Vytvoř boolean proměnné isTuesday a hasLoyaltyCard. Podmínku pro věrnostní kartu řeš až po určení základní ceny.
  1. Nastav proměnné: age, isTuesday, hasLoyaltyCard.
  2. Urči základní cenu do proměnné price pomocí if / else if / else.
  3. Úterý řeš dřív než věk (přebije věkové ceny).
  4. Věrnostní kartu aplikuj jen pokud price > 0 a není úterý: price > 0 && !isTuesday && hasLoyaltyCard.
  5. Ošetři, aby cena neklesla pod nulu.
  6. Na konci vypiš cenu do konzole.
let age = 30;
let isTuesday = false;
let hasLoyaltyCard = true;

let price;

// 1) Základní cena
if (age < 3) {
  price = 0;
} else if (isTuesday) {
  price = 5;
} else if (age <= 12) {
  price = 5;
} else if (age <= 17) {
  price = 7;
} else if (age <= 64) {
  price = 10;
} else {
  price = 6;
}

// 2) Sleva věrnostní kartou (neplatí v úterý ani pro zdarma)
if (price > 0 && !isTuesday && hasLoyaltyCard) {
  price -= 2;
}

// 3) Cena nesmí být záporná
if (price < 0) {
  price = 0;
}

console.log("Cena lístku je $" + price);

Co jsme se naučili

  • Vnořený if je if uvnitř if – používá se, když druhá podmínka dává smysl až po splnění první.
  • && (AND) vyžaduje pravdu na obou stranách.
  • || (OR) stačí pravda na jedné straně.
  • ! (NOT) obrací boolean.
  • Díky kombinaci podmínek umíme řešit reálné scénáře (role uživatele, licence, slevy, pravidla v praxi).
Tip: Když začne být podmínka dlouhá, rozděl ji na menší části do proměnných – bude čitelnější.
