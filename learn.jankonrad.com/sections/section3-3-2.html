3.3.2 Základy programování v JavaScriptu – proměnné, datové typy, typeof

V této kapitole se naučíš úplné základy „gramatiky“ JavaScriptu: proměnné (let, const), datové typy (string, number, boolean) a nástroj typeof pro kontrolu typu hodnoty. Vše budeme psát ve VS Code jako nový index.html a výsledek budeme kontrolovat v prohlížeči přes konzoli.

Zadání (workflow)

  1. Vytvoř novou složku projektu (např. 3-3-2).
  2. Vytvoř nový soubor index.html.
  3. Vlož JavaScript do stránky přes <script>...</script> (na konec souboru).
  4. Soubor otevři v prohlížeči a sleduj:
    • vyskakovací okna (alert),
    • konzoli (Inspect → Console) pro console.log a chyby.
Tip: JavaScript se čte shora dolů. Když něco „nefunguje“, první kontrola je Console (chyby + řádek).

Co je cílem této části

Než začneme stavět větší webové věci, potřebujeme ovládnout úplné základy jazyka – podobně jako se nejdřív učí slovíčka a gramatika. Budeme psát krátké úseky kódu, hned je spouštět v prohlížeči a kontrolovat výstupy v Console.

Jak si ověříš, že kód funguje

  • alert(...): rychlá vizuální kontrola (pop-up).
  • console.log(...): „výpis“ do konzole – nejčastější nástroj při učení i ladění.
  • Console pro chyby: když uděláš překlep nebo chybí znak, prohlížeč obvykle ukáže chybu a řádek.

Co je proměnná (jednoduše)

Proměnná je „pojmenované místo“, kam si program ukládá hodnotu (třeba jméno, věk, stav přihlášení). Představ si ji jako krabičku se štítkem – štítek je název proměnné, obsah je hodnota.

V praxi proměnné používáš pořád: hodnoty si ukládáš, později je čteš, a často je v průběhu programu měníš.

Deklarace a inicializace

  • Deklarace = vytvoříš proměnnou (rezervuješ místo v paměti) a dáš jí jméno.
  • Inicializace = dáš proměnné první hodnotu.
let userName;          // deklarace (zatím bez hodnoty)
userName = "Tom";      // inicializace (přiřazení hodnoty)

Přiřazení není porovnání

Znak = v JavaScriptu znamená přiřazení (dát hodnotu do proměnné), ne „rovná se“ ve smyslu matematiky. K porovnávání se používají jiné operátory (např. >, později ===).

Přepis hodnoty (reassign)

Hodnotu v proměnné vytvořené přes let můžeš změnit kdykoli. JavaScript čte kód shora dolů – takže když hodnotu přepíšeš a hned potom ji vypíšeš, uvidíš nový stav. 

let userName = "Tom";
console.log(userName);

userName = "John";
console.log(userName);

Konstanty: const

Někdy nechceš, aby se hodnota měnila (např. URL endpoint, pevná konstanta, nastavení). V tom případě použiješ const.

  • const musí být deklarované i inicializované na stejném řádku.
  • Pokud se ho pokusíš přepsat, dostaneš chybu.
const apiUrl = "https://example.com/api";
// apiUrl = "https://other.com"; // TypeError: Assignment to constant variable

Pravidla a zvyklosti pro názvy proměnných

  • Začíná písmenem, $ nebo _ (ne číslem).
  • Bez mezer (jeden „kus“ textu).
  • Nepoužívej rezervovaná slova (const, function, class…).
  • U více slov používej camelCase (např. userName, isLoggedIn).
  • Vol smysluplné názvy (krátké, ale výstižné).

String (text)

String je text – posloupnost znaků. Může obsahovat písmena, čísla, interpunkci i speciální znaky. Text v JavaScriptu zapisuješ do uvozovek (jednoduché, dvojité) nebo do backticků. 

let name = "abc123.?!"; // string s písmeny, čísly a symboly

Number (číslo)

Čísla zapisuješ bez uvozovek. JavaScript nerozlišuje zvlášť „int“ a „float“ typem – oboje je prostě number. 

let age = 31;   // celé číslo
let price = 5.5; // desetinné číslo (stále number)

Proč je rozdíl mezi string a number důležitý

Stejný operátor se může chovat jinak podle typu hodnot. Typický příklad je +:

  • number + number → matematika
  • string + string → spojování textu
  • string + number → často se převede na text a spojí
alert(7 + 4);       // 11
alert("7" + "4");   // "74"
alert("7" + 4);     // "74"

Boolean (true / false)

Boolean má jen dvě hodnoty: true nebo false. Používá se hlavně pro rozhodování (později v if a cyklech). 

let isLoggedIn = false; // typicky „výrokový“ název

Booleany často vznikají porovnáním

Porovnáním hodnot získáš boolean. Například: 

let age = 31;
console.log(age > 20); // true (porovnání → boolean)

K čemu je typeof

Ne vždy kontroluješ typ dat ty sám – často data přijdou „zvenku“ (API, databáze, formulář). typeof ti pomůže zjistit, jaký typ má aktuální hodnota. 

let name = "abc123.?!";
console.log(typeof name); // "string"

undefined = deklarováno, ale bez hodnoty

Když proměnnou vytvoříš, ale nic do ní nedáš, je její hodnota undefined. Je to jako krabička se štítkem, ale zatím prázdná.

null = záměrně „nic“

nulltypeof null vrací "object", což je historicky matoucí, ale null se běžně bere jako primitivní hodnota.)

Kroky

  1. Vytvoř index.html a vygeneruj šablonu (! + Enter).
  2. Nastav <title>3.3.2</title>.
  3. Na konec souboru vlož <script> a napiš kód přesně v tomto pořadí.
  4. Otevři soubor v prohlížeči:
    • vyskakovací okna se zobrazí postupně,
    • v Console uvidíš výpisy console.log.

✅ Finální výstup (soubor index.html)

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>3.3.2</title>
</head>
<body>

</body>
<script>
    let userName = "Tom";
    console.log(userName);
    userName = "John";
    console.log(userName);
    let name = "abc123.?!" // String with letters, digits, and special characters

    let age = 31; // Number representing age

    alert(7 + 4);
    alert("7" + "4");
    alert("7" + 4);

    let isLoggedIn = false; // Boolean value

    console.log(age > 20);

    console.log(typeof name); // Outputs: string
</script>
</html>

✅ Kontrolní checklist

  • Po otevření stránky se zobrazí tři alert okna (11, "74", "74).
  • V Console se vypíše Tom a potom John (po přepsání proměnné).
  • V Console se vypíše výsledek porovnání age > 20 (true).
  • V Console se vypíše typeof name jako "string".

Proč je Console základní nástroj vývojáře
  • Okamžitý feedback: vidíš výpisy z programu bez toho, aby ses „hrabal“ do stránky.
  • Chyby a řádky: prohlížeč často ukáže přesně soubor a řádek, kde problém vznikl.
  • Testování nápadů: můžeš rychle zkusit výraz nebo řádek kódu a teprve pak ho přepsat do souboru.
Přiřazení (=) vs porovnání

= znamená: „vezmi hodnotu vpravo a ulož ji do proměnné vlevo“. Neptej se tím, jestli jsou dvě věci stejné. 

let x = 10;     // přiřazení
console.log(x > 5); // porovnání (výsledek je boolean)
let vs const v praxi
  • const: výchozí volba – když nechceš hodnotu přepisovat (bezpečnější).
  • let: když víš, že se hodnota bude měnit (např. skóre, jméno uživatele, stav).
Pojmenování proměnných – rychlé příklady

Dobré: výstižné a čitelné

let userName = "Tom";
let age = 31;
let isLoggedIn = false;

Slabé: nic neříkají (po týdnu nevíš, co to je)

let a = 31;
let x = "Tom";
Proč "7" + 4 není 11

Operátor + má dva „režimy“: s čísly počítá, se stringy spojuje. Jakmile je na jedné straně string, JavaScript často převede druhou hodnotu na string a provede spojení. 

console.log(7 + 4);     // 11 (number)
console.log("7" + "4"); // "74" (string)
console.log("7" + 4);   // "74" (string + number → spojení)
undefined vs null
  • undefined: proměnná existuje, ale ještě nemá přiřazenou hodnotu.
  • null: hodnotu nastavíš záměrně na „nic“ (záměrná prázdnota).
let a;
console.log(a);            // undefined
let b = null;
console.log(b);            // null
console.log(typeof a);     // "undefined"
console.log(typeof b);     // "object" (historická zvláštnost)
Explain this further: let a středníky

V moderním JavaScriptu se nejčastěji používají let a const (od ES6). Středníky jsou technicky často volitelné, protože existuje automatické vkládání středníků, ale konzistentní používání středníků snižuje šanci na „divné“ chyby při spojování řádků. 

let name;
name = "Tom";
