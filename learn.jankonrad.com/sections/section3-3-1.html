3.3.1 Základy programování v JavaScriptu – úvod

V této kapitole si ujasníme, proč je JavaScript klíčový jazyk pro moderní web, kde všude se používá a jak s ním začneš prakticky během pár minut.

Cíl kapitoly

  • pochopit, co JavaScript dělá (a proč HTML + CSS nestačí)
  • vědět, kde se JavaScript používá (web, server, mobil, desktop)
  • rozlišit „běží v prohlížeči“ vs. „běží na serveru“
  • spustit první JavaScript a vytvořit první miniaturní interakci na stránce
  • nastavit si způsob práce: teorie → cvičení → řešení → krátký test

Proč se učit JavaScript

Pokud už umíš HTML a CSS, dokážeš postavit stránku, která dobře vypadá. Problém je, že je pořád „statická“: nic nepočítá, nereaguje, nezpracuje kliknutí, nezmění obsah, nenačte data, nevaliduje formulář. JavaScript je jazyk, který stránce přidá chování – dá jí „mozek“.

A co je důležité: JavaScript není jen „jazyk do prohlížeče“. Díky ekosystému okolo něj můžeš stejným jazykem pracovat na frontendu i backendu, stavět mobilní aplikace i desktopové programy. Proto je to velmi univerzální a praktická dovednost.

Kde všude se JavaScript používá

  • Web (frontend): interaktivní weby – kliknutí, animace, dynamický obsah, práce s DOM
  • Server (backend): JavaScript na serveru přes Node.js (API, databáze, autentizace…)
  • Mobilní aplikace: např. React Native (sdílená logika napříč platformami)
  • Desktop aplikace: např. Electron (aplikace pro Windows/macOS)

Pokud tě v budoucnu láká React, Next nebo TypeScript, JavaScript je jejich základ. Nejsnáze se pak učí, když máš pevné základy syntaxe a přemýšlení „jako programátor“.

Co JavaScript vlastně je: prohlížeč vs. server

JavaScript vznikl jako jazyk, který běží přímo v prohlížeči uživatele. To znamená: k reakci na kliknutí nemusíš vždy posílat požadavek na server a čekat na odpověď. Pro mnoho věcí může kód reagovat okamžitě – a to je základ moderního webu.

Na druhou stranu, řada jazyků (např. Python, PHP) se typicky spouští na serveru. U takových webů prohlížeč často „požádá“ server o data nebo stránku a čeká na odpověď. JavaScript je výjimečný v tom, že dnes pokrývá oba světy: prohlížeč i server.

Poznámka k učení: JavaScript umí být záludnější než HTML/CSS. Narazíš na chování, které je jiné než v jiných jazycích, a navíc se jazyk i standard (ECMAScript) průběžně vyvíjí. To je normální – důležité je stavět pevné základy a průběžně si věci ověřovat praxí.

Jak z kurzu (a z této kapitoly) vytěžit maximum

  1. Nejen koukat – psát kód. Otevři editor a u každého příkladu si ho přepiš.
  2. Po teorii hned cvičení. Bez praxe se znalost „neukotví“.
  3. Řešení až po pokusu. Nejdřív se s úkolem chvíli poper sám.
  4. Zastav se u otázek. Kdykoli si nejsi jistý, tipni odpověď a ověř v kódu.

První JavaScript za 5 minut (rychlý start)

Varianta A – Konzole prohlížeče (nejrychlejší):

  1. Otevři webovou stránku (klidně prázdnou).
  2. Stiskni F12 → záložka Console.
  3. Napiš:
console.log("Ahoj, JavaScript!");

Pokud se text vypíše, JavaScript ti funguje a právě jsi spustil první program.

Varianta B – Mini projekt v HTML souboru:

Vytvoř soubor index.html a vlož do něj tento základ:

<!doctype html>
<html lang="cs">
<head>
  <meta charset="utf-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1" />
  <title>Můj první JavaScript</title>
</head>
<body>
  <h1>JavaScript: první interakce</h1>

  <button id="btn">Klikni na mě</button>
  <p id="out">Zatím se nic nestalo.</p>

  <script>
    // 1) najdeme prvky na stránce
    const btn = document.querySelector("#btn");
    const out = document.querySelector("#out");

    // 2) zareagujeme na kliknutí
    btn.addEventListener("click", () => {
      out.textContent = "Kliknutí zachyceno! Stránka reaguje díky JS.";
    });
  </script>
</body>
</html>
Co se tu děje?
  • document.querySelector vybere prvek ze stránky.
  • addEventListener „poslouchá“ událost (klik).
  • Uvnitř funkce změníme text odstavce – stránka se dynamicky aktualizuje.

Cvičení (prakticky, krok za krokem)

Zadání:

  1. Otevři konzoli v prohlížeči.
  2. Vypiš do konzole text „Ahoj světe“ pomocí console.log.
  3. Zkus vypsat i jednoduchý výpočet (např. 12 + 30).
Řešení 
console.log("Ahoj světe");
console.log(12 + 30);

Zadání:

  1. Vytvoř index.html a vlož tlačítko a odstavec.
  2. Po kliknutí na tlačítko změň text v odstavci na „Hotovo!“
  3. Přidej počítadlo kliknutí: po každém kliknutí se má zobrazit „Kliknutí: 1, 2, 3…“
Řešení 
<button id="btn">Klikni</button>
<p id="out">Kliknutí: 0</p>

<script>
  const btn = document.querySelector("#btn");
  const out = document.querySelector("#out");
  let count = 0;

  btn.addEventListener("click", () => {
    count++;
    out.textContent = `Kliknutí: ${count}`;
  });
</script>

Zadání:

  1. Přidej na stránku blok (např. <div id="panel">) s nějakým textem.
  2. Tlačítkem přepínej, jestli je panel vidět nebo ne (show/hide).
  3. Bonus: ať se text tlačítka přepíná „Skrýt“ / „Zobrazit“.
Řešení 
<button id="toggle">Skrýt</button>
<div id="panel">Toto je panel, který umíme skrývat.</div>

<script>
  const toggle = document.querySelector("#toggle");
  const panel = document.querySelector("#panel");

  toggle.addEventListener("click", () => {
    const isHidden = panel.style.display === "none";
    panel.style.display = isHidden ? "block" : "none";
    toggle.textContent = isHidden ? "Skrýt" : "Zobrazit";
  });
</script>

Správné odpovědi (stručně)
  • HTML/CSS řeší strukturu a vzhled, ne chování.
  • V prohlížeči; reakce na události může být okamžitá.
  • addEventListener("click", ...)
  • První prvek odpovídající CSS selektoru (nebo null).

Mini tahák (z této kapitoly)

  • console.log(...) – výpis pro ladění a kontrolu hodnot
  • document.querySelector("...") – výběr prvku na stránce
  • element.addEventListener("click", fn) – reakce na událost
  • element.textContent = "..." – změna textu na stránce
