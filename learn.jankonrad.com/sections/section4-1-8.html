4.1.7a.1 Booleany & rozhodování – krok po kroku, opravdu pomalu

Cílem je umět číst a psát malé kousky logiky, které pak použijeme v To-Do a v Košíku. Vždy: mikro-příklad → co to dělá → mini-úkol → (volitelně) modál s vysvětlením. 

# boolean = typ pro logiku
pravda = True
nepravda = False
print(pravda)    # True
print(nepravda)  # False
  • Boolean má jen dvě hodnoty: True a False.
  • V podmínkách (if) rozhoduje, která větev se spustí.
Mini-úkol: Vytvoř sviti = False. Když bude True, znamená to „světlo je zapnuté“; když False, „zhasnuto“. Jen si to vypiš print(sviti).

hotovo = False

if hotovo:
    print("✅ Úkol splněn!")
print("Konec programu.")
  • Vytiskne se jen "Konec programu.", protože hotovo je False.
  • Kdyby bylo hotovo = True, vytiskne se i první řádek se „✅“.
Mini-úkol: Změň hotovo na True a spusť. Co se změnilo?

hotovo = False

if hotovo:
    print("✅ Úkol splněn!")
else:
    print("⌛ Ještě pracuji…")
  • Spustí se přesně jedna větev. Buď „splněno“, nebo „ještě pracuji“.
Mini-úkol: Nastav hotovo = True a ověř, že se vytiskne jen první větev.

sviti = False
sviti = not sviti   # přepínač: False -> True
print(sviti)        # True
  • not je „ne“ – neguje hodnotu (True ↔ False).
  • Často použijeme na přepnutí stavu („hotovo ↔ nehotovo“).
Mini-úkol: Vytvoř hotovo = False, pak dvakrát za sebou udělej hotovo = not hotovo. Jaká je výsledná hodnota?

prazdny = []   # prázdný seznam
if not prazdny:
    print("Seznam je prázdný.")  # vypíše se

neprazdny = [1]
if neprazdny:
    print("Seznam má prvky.")    # vypíše se
Pravidlo: prázdné struktury ([], (), "", {}, set()), 0, None jsou „False“. Neprázdné a nenulové jsou „True“.
Mini-úkol: Napiš funkci je_prazdny(xs), která vytiskne „prázdný“/„neprázdný“ podle obsahu.

hotovo = True
box = "x" if hotovo else " "
print(f"[{box}] Zalít kytku")
# je to totéž jako:
hotovo = True
if hotovo:
    box = "x"
else:
    box = " "
Mini-úkol: Nastav hotovo = False. Co se vytiskne?

# jeden úkol jako dvojice (hotovo, text)
u2 = (True, "Zalít kytku")

# rozbalování: první prvek do 'hotovo', druhý do 'txt'
hotovo, txt = u2
box = "x" if hotovo else " "
print(f"[{box}] {txt}")
  • Tuple = „balíček“ hodnot k sobě (jako řádek tabulky).
  • Počet proměnných vlevo musí odpovídat počtu prvků v n-tici.
Mini-úkol: Zkus u3 = (False, "Cvičit") a vytiskni řádek se správným boxem.

todo = []
todo.append((False, "Uklidit stůl"))
todo.append((True,  "Zalít kytku"))

for i, (done, text) in enumerate(todo, start=1):
    box = "x" if done else " "
    print(f"{i}. [{box}] {text}")
  • enumerate(xs, start=1) dává dvojice (poradi, prvek) – pro lidi číslujeme od 1.
Mini-úkol: Přidej třetí úkol (False, "Domácí úkol") a ověř číslování.

def tisk_todo(polozky):
    if not polozky:                 # prázdný list je False → not False = True
        print("Seznam je prázdný.")
        return                      # ukončíme funkci dřív (nemá smysl pokračovat)
    for i, (done, text) in enumerate(polozky, start=1):
        box = "x" if done else " "
        print(f"{i}. [{box}] {text}")
„Early return“ zjednoduší zbytek funkce – dál řešíme už jen neprázdný případ.
Mini-úkol: Zavolej tisk_todo([]) a tisk_todo([(True,"Test")]) a porovnej.

todo = [(False, "Uklidit stůl"), (True, "Zalít kytku")]

def pridej(todo, text):
    todo.append((False, text))   # nový úkol = zatím nehotový

def prepnout(todo, index_1based):
    i = index_1based - 1
    if 0 <= i < len(todo):
        done, txt = todo[i]
        todo[i] = (not done, txt)   # ← přepínač booleanu
    else:
        print("Neplatné číslo.")

tisk_todo(todo)
pridej(todo, "Cvičit")
prepnout(todo, 1)
tisk_todo(todo)
Hlídáme hranice indexu – ať nesáhneme mimo seznam.
Mini-úkol: Napiš odeber(todo, index_1based), které smaže úkol s kontrolou hranic.

„1.5“ označuje mezikrok: mezi „1) co je list“ a „2) tuple“ se vloží krátké procvičení (např. pop(), remove(), in…), aby žáci nebyli zahlceni a měli jistotu v předchozím konceptu. Můžeš ho přejmenovat na „2a“, pokud ti to líp sedí do osnov.

🧩 Slož to dohromady (od jedné hodnoty k řádku seznamu)
# 1) boolean -> znak do boxu
done = False
box = "x" if done else " "
print(f"[{box}] Uklidit stůl")

# 2) tuple (balíček) -> rozbal -> box
radek = (True, "Zalít kytku")
d, t = radek
box = "x" if d else " "
print(f"[{box}] {t}")

# 3) list n-tic -> vypiš očíslovaně
todo = [(False, "Uklidit stůl"), (True, "Zalít kytku")]
def tisk_todo(xs):
    if not xs:
        print("Seznam je prázdný.")
        return
    for i, (dd, tt) in enumerate(xs, start=1):
        box = "x" if dd else " "
        print(f"{i}. [{box}] {tt}")

tisk_todo(todo)

Reálné příklady booleanů
  • „Svítí světlo?“ → True/False
  • „Mám v košíku aspoň 1 položku?“ → True/False
  • „Je heslo správné?“ → True/False
Počítač potřebuje taková ano/ne rozhodnutí pro větvení programu.
Typ bool v Pythonu

True a False jsou hodnoty typu bool. Vznikají i jako výsledek porovnání, např. 5 > 3 je True.

Jak if rozhoduje krok za krokem
  1. Vyhodnotí výraz v závorce/za if → boolean.
  2. Když je True → spustí blok podmínky.
  3. Když False → blok přeskočí.
Negace not – obrazně

Představ si vypínač: zapnuto ↔ vypnuto. not překlápí stav na opačný.

Proč funguje if not seznam?

Protože prázdný seznam má „pravdivost“ False. Negace not False je True → větev se spustí. U neprázdného seznamu je to naopak.

Ternární vs. klasický if/else
  • Ternární použij, když přiřazuješ jednu hodnotu (krátké rozhodnutí).
  • If/else použij, když uvnitř děláš víc kroků (tisky, volání funkcí, další logika).
Rozbalování n-tic (unpacking)
# Musí sedět počet prvků:
(u_done, u_text) = (False, "Uklidit")
# (u_done, u_text, extra) = (False, "Uklidit")  # ValueError – moc proměnných

V cyklu funguje také: for done, text in seznam_n_tic: – každý řádek (n-tice) se rozbalí.

Co dělá enumerate uvnitř
seznam = ["A", "B", "C"]
for poradi, prvek in enumerate(seznam, start=1):
    print(poradi, prvek)
# Vracené dvojice jsou (1,"A"), (2,"B"), (3,"C")

Bez start=1 by číslování začínalo od nuly.

„Early return“ – proč to zlepšuje čitelnost

Vyřešíš „okrajový případ“ hned na začátku a zbytek funkce není zanořený v else. Vznikne kratší a čistší kód.
