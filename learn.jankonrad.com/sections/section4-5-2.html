4.5.2 GUI v Pythonu – první kroky k rezervačnímu systému

V minulých částech vznikl konzolový rezervační systém. Program už umí pracovat s daty, ale zatím se ovládá přes textové menu.

Teď přichází další krok: dát programu grafickou podobu. To znamená, že místo konzolových voleb začne mít vlastní okno, textové popisky, vstupní pole, tlačítka a tabulku.

V této části se ještě nebude řešit ukládání do databáze. Cílem je nejdřív opravdu pochopit, jak se GUI aplikace skládá, co znamenají zápisy jako import tkinter, from tkinter import ttk, root = tkinter.Tk(), root.mainloop() a jak se jednotlivé prvky do okna přidávají a testují.

Co se v této lekci naučíš

  • Jak navázat na už hotový projekt a začít jeho grafickou verzi.
  • Co je GUI a proč je vhodné pro rezervační systém.
  • Co je modul, import, alias a část modulu.
  • Co znamenají zápisy import tkinter a from tkinter import ttk.
  • Co je objekt okna root.
  • Co je metoda a co znamená tečka v zápisu root.title(...).
  • Proč je potřeba mainloop() a co dělá hlavní smyčka GUI aplikace.
  • Jak postupně testovat okno, text, tlačítko, formulář a tabulku.
  • Jak vytvořit první grafickou kostru budoucího rezervačního systému.
Hlavní cíl kapitoly:
Vytvořit první grafickou verzi rezervačního systému a přitom poctivě pochopit, co dělá každý důležitý krok.

Na co tato kapitola navazuje

  1. Existuje konzolový program pro rezervace.
  2. Existuje databázová logika a práce s daty.
  3. Dalším krokem je připravit grafické rozhraní.
  4. Nejdřív se ale nebude řešit ukládání, ale samotné GUI.
Důležité:
Tahle kapitola navazuje na předchozí práci, ale zatím používá jen grafické prvky. Logika rezervací se doplní až v další části.
Krok 1: co je GUI a proč na něj teď navázat

GUI je zkratka pro Graphical User Interface, tedy grafické uživatelské rozhraní.

Zatím program fungoval v konzoli. To znamená, že:

  • vypisoval menu textem,
  • čekal na číselnou volbu,
  • a data se zadávala přes input().

V GUI programu bude místo toho:

  • okno aplikace,
  • textové popisky,
  • vstupní pole,
  • tlačítka,
  • tabulka s daty.
Smysl tohoto kroku:
Nezačíná se nový projekt. Jen se ke stávající logice přidává nový způsob ovládání.
Krok 2: první importy – co je modul a co je ttk

Než vznikne první okno, musí se načíst potřebné moduly.

První dva řádky GUI programu 
import tkinter
from tkinter import ttk

Co znamená první řádek

import tkinter
  • import = načti modul do programu.
  • tkinter = modul pro tvorbu GUI v Pythonu.

Co znamená druhý řádek

from tkinter import ttk
  • from = z něčeho vezmi,
  • tkinter = z modulu tkinter,
  • import ttk = načti část modulu jménem ttk.

ttk obsahuje další widgety, například velmi užitečný prvek Treeview, který se hodí pro tabulku rezervací.

Pro tuto lekci:
tkinter dodá základní prvky okna a ttk se bude hodit hlavně pro tabulku.
Krok 3: první test – vytvoření samotného okna

Teď přijde první opravdu spustitelný krok. Cílem není udělat celou aplikaci, ale jen ověřit, že se umí vytvořit a zobrazit okno.

První test GUI okna 
import tkinter

root = tkinter.Tk()
root.title("Rezervační systém")
root.geometry("760x500")

root.mainloop()

Jak tenhle test číst

  • root = tkinter.Tk() = vytvoř hlavní okno aplikace.
  • root.title(...) = nastav titulek okna.
  • root.geometry(...) = nastav velikost okna.
  • root.mainloop() = spusť hlavní smyčku aplikace.

Co se má po spuštění stát

  • otevře se prázdné okno,
  • v horní liště bude napsáno „Rezervační systém“,
  • okno zůstane otevřené, dokud ho někdo nezavře.
Tohle je první test:
Nepřidává se nic dalšího, dokud není jisté, že funguje samotné okno.
Krok 4: proč je potřeba mainloop() a co je hlavní smyčka

To je jeden z nejdůležitějších principů celé GUI aplikace.

Když se vytvoří okno přes tkinter.Tk(), ještě to neznamená, že bude samo fungovat. Program totiž běží odshora dolů. Kdyby došel na konec souboru, okamžitě by skončil.

A právě proto je potřeba: 

root.mainloop()

Tato metoda spustí hlavní smyčku GUI aplikace.

Smyčka znamená, že program pořád dokola kontroluje:

  • jestli někdo klikl myší,
  • jestli někdo píše do pole,
  • jestli se má okno překreslit,
  • jestli bylo okno zavřeno.
Jednoduše:
mainloop() drží GUI aplikaci naživu a umožňuje jí reagovat na události.
Krok 5: druhý test – přidání textového popisku

Jakmile funguje samotné okno, je čas přidat první widget. Nejjednodušší je textový popisek Label.

Druhý test – Label 
import tkinter

root = tkinter.Tk()
root.title("Rezervační systém")
root.geometry("760x500")

label = tkinter.Label(root, text="Rezervační systém")
label.pack()

root.mainloop()

Co je zde nové

  • tkinter.Label(...) = vytvoření textového widgetu.
  • root = rodič, tedy okno, do kterého Label patří.
  • text="Rezervační systém" = text, který se zobrazí.
  • label.pack() = vložení widgetu do okna.

Co se má po spuštění stát

  • otevře se okno,
  • uvnitř bude vidět text „Rezervační systém“.
Krok 6: třetí test – přidání tlačítka

Dalším logickým krokem je tlačítko. Zatím ještě nebude mít žádnou funkci, ale ukáže se, jak se do okna přidává další widget.

Třetí test – Button 
import tkinter

root = tkinter.Tk()
root.title("Rezervační systém")
root.geometry("760x500")

label = tkinter.Label(root, text="Rezervační systém")
label.pack()

button = tkinter.Button(root, text="Přidat rezervaci")
button.pack()

root.mainloop()

Co se má po spuštění stát

  • otevře se okno,
  • zobrazí se text,
  • pod ním bude tlačítko.

Tlačítko zatím nic nedělá. A to je správně. V této chvíli se testuje jen jeho vytvoření a zobrazení.

Krok 7: čtvrtý test – vstupní pole

Rezervační systém nebude mít jen text a tlačítka. Bude potřebovat i místa, kam se budou zapisovat údaje. K tomu slouží Entry.

Čtvrtý test – Entry 
import tkinter

root = tkinter.Tk()
root.title("Rezervační systém")
root.geometry("760x500")

label = tkinter.Label(root, text="Jméno:")
label.pack()

entry = tkinter.Entry(root)
entry.pack()

button = tkinter.Button(root, text="Přidat rezervaci")
button.pack()

root.mainloop()

Tady už se začíná objevovat to, co bude později skutečný formulář.

Co se testuje

  • zobrazení textového popisku,
  • zobrazení vstupního pole,
  • zobrazení tlačítka pod nimi.
Krok 8: pátý test – seskupení prvků do LabelFrame

Když je widgetů víc, je dobré je seskupit do části s vlastním názvem. K tomu slouží LabelFrame.

Pátý test – LabelFrame 
import tkinter

root = tkinter.Tk()
root.title("Rezervační systém")
root.geometry("760x500")

frame = tkinter.LabelFrame(root, text="Nová rezervace", padx=10, pady=10)
frame.pack(padx=15, pady=15, fill="x")

label = tkinter.Label(frame, text="Jméno:")
label.pack()

entry = tkinter.Entry(frame)
entry.pack()

button = tkinter.Button(frame, text="Přidat rezervaci")
button.pack(pady=10)

root.mainloop()

Co je důležité

  • frame je nový rodič pro vnitřní prvky.
  • Label, Entry i Button už nepatří přímo do root, ale do frame.
  • Díky tomu vzniká přehlednější část formuláře.
Krok 9: šestý test – první tabulka přes ttk.Treeview

Druhá velká část budoucí aplikace bude seznam rezervací. Pro něj se hodí ttk.Treeview.

Šestý test – Treeview 
import tkinter
from tkinter import ttk

root = tkinter.Tk()
root.title("Rezervační systém")
root.geometry("760x500")

tree = ttk.Treeview(root, columns=("id", "name"), show="headings")
tree.heading("id", text="ID")
tree.heading("name", text="Jméno")
tree.pack(fill="both", expand=True)

root.mainloop()

Co se testuje

  • že se zobrazí tabulka,
  • že má záhlaví sloupců,
  • že se rozšiřuje do prostoru okna.
Krok 10: spojení všech otestovaných částí do jedné kostry

Teď už jsou otestované základní stavební kameny:

  • hlavní okno,
  • Label,
  • Entry,
  • Button,
  • LabelFrame,
  • Treeview.

Až teď dává smysl spojit je do jedné větší ukázky.

První grafická kostra rezervačního systému 
import tkinter
from tkinter import ttk

root = tkinter.Tk()
root.title("Rezervační systém")
root.geometry("760x500")

title_label = tkinter.Label(root, text="Rezervační systém", font=("Arial", 20, "bold"))
title_label.pack(pady=10)

subtitle_label = tkinter.Label(root, text="Grafická příprava aplikace")
subtitle_label.pack(pady=5)

form_frame = tkinter.LabelFrame(root, text="Nová rezervace", padx=10, pady=10)
form_frame.pack(fill="x", padx=15, pady=10)

label_name = tkinter.Label(form_frame, text="Jméno:")
label_name.pack()

entry_name = tkinter.Entry(form_frame, width=40)
entry_name.pack(pady=5)

label_date = tkinter.Label(form_frame, text="Datum (YYYY-MM-DD):")
label_date.pack()

entry_date = tkinter.Entry(form_frame, width=40)
entry_date.pack(pady=5)

label_arrival = tkinter.Label(form_frame, text="Příchod (HH:MM):")
label_arrival.pack()

entry_arrival = tkinter.Entry(form_frame, width=40)
entry_arrival.pack(pady=5)

label_departure = tkinter.Label(form_frame, text="Odchod (HH:MM):")
label_departure.pack()

entry_departure = tkinter.Entry(form_frame, width=40)
entry_departure.pack(pady=5)

button_add = tkinter.Button(form_frame, text="Přidat rezervaci")
button_add.pack(pady=10)

table_frame = tkinter.LabelFrame(root, text="Seznam rezervací", padx=10, pady=10)
table_frame.pack(fill="both", expand=True, padx=15, pady=10)

tree = ttk.Treeview(
    table_frame,
    columns=("id", "name", "date", "arrival", "departure"),
    show="headings"
)

tree.heading("id", text="ID")
tree.heading("name", text="Jméno")
tree.heading("date", text="Datum")
tree.heading("arrival", text="Příchod")
tree.heading("departure", text="Odchod")

tree.pack(fill="both", expand=True)

root.mainloop()
Krok 11: co tahle verze umí a co ještě ne

Co už umí

  • otevřít hlavní okno,
  • zobrazit nadpis aplikace,
  • zobrazit formulář pro novou rezervaci,
  • zobrazit tlačítko,
  • zobrazit tabulku pro seznam rezervací.

Co ještě neumí

  • ukládat rezervaci do databáze,
  • načítat skutečné řádky,
  • reagovat na kliknutí tlačítka,
  • vyčistit pole po odeslání.
A to je správně.
Tahle kapitola má vytvořit a otestovat vzhled. Chování se doplní až v navazující části.
Krok 12: proč je důležité testovat po malých krocích

Kdyby se všechno napsalo najednou do jednoho velkého souboru, bylo by těžší poznat, kde vznikla chyba.

Postupné testování má několik výhod:

  • okamžitě je vidět, co už funguje,
  • je jasné, ve kterém kroku se případně objevil problém,
  • každý nový widget se pochopí samostatně,
  • větší finální kód pak nepůsobí jako jedna nepřehledná zeď textu.
Dobrá praxe:
nejdřív malé funkční testy, potom větší celek.

GUI × konzolový program

Konzolový program pracuje hlavně s textem. Program vypíše nabídku a čeká, co se napíše do terminálu.

GUI program se ovládá přes viditelné prvky v okně: tlačítka, pole, seznamy a tabulky.

Prakticky:
logika programu může zůstat podobná, ale způsob ovládání je jiný.

Co je modul a co dělá import

Modul je kus hotového Python kódu, který si program může načíst a používat.

Příkaz import znamená: načti modul do programu. 

import tkinter

Tím se zpřístupní vše potřebné pro tvorbu GUI přes modul tkinter.

Co je ttk

ttk je část modulu tkinter, která obsahuje některé další widgety.

V této kapitole je důležitý hlavně kvůli prvku Treeview, který se hodí jako tabulka pro výpis rezervací. 

from tkinter import ttk

Tečka a metoda

Zápis jako root.title(...) nebo root.geometry(...) používá tečku.

Tečka znamená: vezmi něco, co patří objektu root.

  • root = objekt hlavního okna,
  • title = metoda tohoto objektu,
  • geometry = další metoda tohoto objektu.

Metoda je akce, kterou objekt umí provést.

Proč je potřeba mainloop()

mainloop() spustí hlavní smyčku GUI aplikace.

Díky ní program stále běží a kontroluje, jestli někdo klikl, něco napsal nebo zavřel okno. 

root.mainloop()
Bez této smyčky by se program vytvořil a hned skončil.

Co je widget

Widget je grafický prvek uvnitř okna.

Například:

  • Label = text,
  • Entry = vstupní pole,
  • Button = tlačítko,
  • Treeview = tabulka.

Co dělá pack()

Samotné vytvoření widgetu ještě neznamená, že se hned ukáže v okně.

Metoda pack() widget do okna umístí a tím ho zobrazí. 

label.pack()
Zjednodušeně:
vytvořit widget nestačí, je potřeba ho i umístit.

Co je Treeview

Treeview je widget z části ttk.

Umí zobrazit data po řádcích a sloupcích, takže se hodí jako tabulka.

V rezervačním systému bude později sloužit pro výpis rezervací.
