6.1.4 Backend – první odpověď serveru

V minulé kapitole jsme si připravili backend tak, aby uměl načíst data z databáze. Teď uděláme další krok: backend už nebude jen pracovat s daty, ale začne skutečně fungovat jako server, který přijímá požadavky a posílá odpovědi.

Co se naučíš?

  • Třídu RequestHandler – pochopíš, proč server potřebuje speciální třídu pro obsluhu požadavků.
  • Metodu do_GET(self) – zjistíš, co se stane, když prohlížeč pošle na server běžný GET požadavek.
  • První textovou odpověď – uvidíš, jak server vrátí jednoduchý text místo HTML nebo JSON.
  • Funkci run_server() – pochopíš, jak se server skutečně spustí a proč potom „čeká“ na požadavky.
Tip: Ve finálním souboru je RequestHandler už rozšířený o JSON, HTML soubory a 404 odpovědi. V této lekci si ale nejdřív oddělíme úplný základ: server přijme požadavek a pošle jednoduchou textovou odpověď. :contentReference[oaicite:1]{index=1}

1) Proč už nestačí jen funkce jako get_movies()

Funkce get_movies() umí načíst data z databáze a vrátit je jako výsledek. To je důležité. Jenže prohlížeč nevolá běžné Python funkce přímo. Prohlížeč posílá na server HTTP požadavky. Proto teď potřebujeme novou vrstvu: objekt, který umí takový požadavek přijmout, rozpoznat ho a odpovědět na něj.

  1. Prohlížeč pošle požadavek na server.
  2. Server tento požadavek zachytí.
  3. Nějaký kus kódu rozhodne, co má vrátit.
  4. Server odešle odpověď zpět prohlížeči.
  • Funkce jako get_movies() řeší data.
  • RequestHandler bude řešit komunikaci přes web.
  • Teprve spojením těchto dvou vrstev vznikne skutečný backend pro stránku.
2) První blok kódu – třída RequestHandler

Ve finálním backendu je server postavený na této třídě: :contentReference[oaicite:2]{index=2} 

class RequestHandler(BaseHTTPRequestHandler):
    pass

Tohle je zatím jen úplný základ. Ještě nic neumí, ale vytváříme si vlastní třídu, která bude dědit chování z BaseHTTPRequestHandler.

  1. class znamená, že definujeme třídu.
  2. RequestHandler je název naší vlastní třídy.
  3. (BaseHTTPRequestHandler) znamená, že z této základní třídy dědíme.
  4. : ukončuje hlavičku třídy a otevírá její tělo.
  5. pass znamená „zatím tu nic nedělej“.
  • Třída je něco jako plán nebo šablona objektu.
  • Základní handler už umí obecné HTTP chování.
  • My do něj budeme postupně dopisovat naši konkrétní logiku.
3) Druhý blok kódu – pomocná metoda _send_text()

Ve finálním backendu už existuje metoda _send_text(), která umí poslat textovou HTTP odpověď. :contentReference[oaicite:3]{index=3} Pro tuto lekci si ji vysvětlíme jako první stavební blok odpovědi serveru: 

def _send_text(self, text, status=200, content_type="text/plain; charset=utf-8"):
    body = text.encode("utf-8")
    self.send_response(status)
    self.send_header("Content-Type", content_type)
    self.send_header("Content-Length", str(len(body)))
    self.end_headers()
    self.wfile.write(body)

Na první pohled je to delší blok. Ale ve skutečnosti jen připraví text, nastaví základní informace o odpovědi a potom text odešle klientovi.

  1. def _send_text(...) definuje metodu uvnitř třídy.
  2. self odkazuje na konkrétní handler, který právě obsluhuje požadavek.
  3. text je text, který chceme vrátit.
  4. status=200
  5. status=200 je výchozí HTTP stavový kód „OK“.
  6. body = text.encode("utf-8") převede text na bajty, které lze skutečně poslat po síti.
  7. send_response(...) nastaví stav odpovědi.
  8. send_header(...) přidá hlavičky odpovědi.
  9. end_headers() ukončí část s hlavičkami.
  10. wfile.write(body) zapíše samotné tělo odpovědi.
  • Tohle je první reálná metoda, která skutečně něco pošle klientovi.
  • Text zatím není HTML stránka ani JSON.
  • Je to jen jednoduchá, ale plnohodnotná webová odpověď.
4) Třetí blok kódu – metoda do_GET(self)

Teď přidáme metodu, která se spustí tehdy, když server dostane GET požadavek. Ve finálním backendu je do_GET() už delší a rozpoznává více cest, :contentReference[oaicite:4]{index=4} minimální verzí. fileciteturn3file0 

def do_GET(self):
    self._send_text("Server běží.")

Tato metoda říká velmi jednoduchou věc: „Když přijde GET požadavek, pošli zpět text Server běží.“

  1. def do_GET(self): je speciální metoda handleru pro GET požadavky.
  2. Když uživatel otevře adresu v prohlížeči, velmi často jde právě o GET.
  3. self._send_text(...) zavolá naši pomocnou metodu.
  4. Server tedy nevrací databázi ani HTML, ale jen krátký text.
  • Toto je ideální první test serveru.
  • Neřešíme ještě cesty jako /api/movies.
  • Jen si ověřujeme, že server umí odpovědět.
5) Jak má třída vypadat dohromady

Po spojení předchozích bloků bude mít naše třída tuto podobu: 

class RequestHandler(BaseHTTPRequestHandler):

    def _send_text(self, text, status=200, content_type="text/plain; charset=utf-8"):
        body = text.encode("utf-8")
        self.send_response(status)
        self.send_header("Content-Type", content_type)
        self.send_header("Content-Length", str(len(body)))
        self.end_headers()
        self.wfile.write(body)

    def do_GET(self):
        self._send_text("Server běží.")
  1. Nejprve definujeme třídu.
  2. Do ní vložíme pomocnou metodu _send_text().
  3. Pod ni vložíme do_GET().
  4. Obě metody musí být správně odsazené uvnitř třídy.
  • Odsazení je v Pythonu zásadní.
  • Když se metoda neodsadí správně, Python ji nebude chápat jako součást třídy.
  • Třída teď už opravdu umí odpovědět na jednoduchý GET požadavek.
6) Čtvrtý blok kódu – spuštění serveru přes run_server()

Třída sama o sobě nestačí. Musíme ještě skutečně vytvořit server a spustit ho. Ve finálním backendu to :contentReference[oaicite:5]{index=5}_server(). fileciteturn3file0 

def run_server(host="127.0.0.1", port=8000):
    server = HTTPServer((host, port), RequestHandler)
    print(f"Server běží na http://{host}:{port}")
    print("Zastavíš ho: CTRL+C")
    server.serve_forever()
  1. def run_server(...) definuje funkci pro spuštění serveru.
  2. host="127.0.0.1" znamená, že server poběží lokálně na tomto počítači.
  3. port=8000 je číslo portu, na kterém bude server naslouchat.
  4. HTTPServer((host, port), RequestHandler) vytvoří server a propojí ho s naší třídou handleru.
  5. serve_forever() spustí nekonečné čekání na požadavky.
  • Server po spuštění „visí“ a čeká.
  • To není chyba, to je správné chování serveru.
  • Zastavuje se ručně, například přes CTRL+C.
7) Jak to poprvé spustit a otestovat

Abychom server skutečně spustili, potřebujeme dole v souboru zavolat: 

if __name__ == "__main__":
    run_server()
  1. Soubor spusť v Pythonu.
  2. V konzoli uvidíš adresu serveru, například http://127.0.0.1:8000.
  3. Tuto adresu otevři v prohlížeči.
  4. Místo stránky zatím uvidíš text Server běží.
  • To je v této chvíli správně.
  • Zatím jsme neřešili HTML ani API.
  • Jen jsme ověřili, že server funguje a umí odpovědět.
Pamatuj: první úspěch serveru není hezká stránka. První úspěch serveru je už to, že přijme požadavek a pošle smysluplnou odpověď.
8) Co máš mít po této kapitole hotové

Po této kapitole ještě neposíláme HTML soubor ani JSON z databáze. Ale už máme skutečný základ serveru: třídu handleru, první textovou odpověď a funkci pro spuštění serveru.

  1. Máš třídu RequestHandler.
  2. Uvnitř ní máš metodu _send_text().
  3. Uvnitř ní máš také do_GET().
  4. Máš funkci run_server().
  5. Server jde spustit a v prohlížeči vrátí textovou odpověď.
  • Teď už opravdu máme server.
  • V další lekci ho naučíme rozlišovat cesty.
  • Až potom přidáme HTML stránku, API a 404 odpověď.

Shrnutí

  • Žák chápe, že RequestHandler obsluhuje HTTP požadavky.
  • Žák ví, že metoda do_GET() se spustí při GET požadavku.
  • Žák rozumí tomu, že _send_text() pošle jednoduchou textovou odpověď klientovi.
  • Žák chápe, že run_server() vytvoří a spustí server, který potom čeká na požadavky.

Co je request a response

Web funguje jako rozhovor mezi klientem a serverem:

  • request = požadavek od klienta na server,
  • response = odpověď serveru klientovi.

Když do prohlížeče zadáš adresu, prohlížeč pošle request. Server ho přijme a vrátí response.

Co znamená třída a dědičnost zde konkrétně

Třída RequestHandler je naše vlastní specializace základního HTTP handleru.

  • BaseHTTPRequestHandler už zná obecná pravidla HTTP.
  • My do něj dopisujeme naši konkrétní logiku.
  • Dědičnost tedy znamená: převezmi základ a rozšiř ho o vlastní chování.

Rozbor _send_text() po krocích

Tato metoda dělá tři hlavní věci:

  • připraví tělo odpovědi,
  • nastaví HTTP stav a hlavičky,
  • pošle data klientovi.

Díky tomu nemusíme stejnou logiku psát znovu a znovu v každé části serveru.

Co přesně znamená do_GET(self)

do_GET() je speciální metoda, kterou základní HTTP handler očekává.

  • Když přijde GET požadavek, zavolá se tato metoda.
  • Uvnitř ní rozhodujeme, co server vrátí.
  • Později v ní budeme rozlišovat i různé cesty, například / a /api/movies.

Proč server po spuštění jen čeká

Server není program, který něco spočítá a hned skončí. Jeho úkolem je běžet a čekat, až ho klient osloví.

  • serve_forever() znamená „obsluhuj požadavky pořád dokola“.
  • Proto server po spuštění nevypíše „Hotovo“ a nekončí.
  • Naopak tím dává najevo, že je připravený.
