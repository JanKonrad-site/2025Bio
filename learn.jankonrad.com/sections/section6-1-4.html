6.1.2 GUI v Pythonu – jak z konzolového programu udělat okno s tlačítky

V minulých částech vznikl funkční Python program, který pracuje s databází reservations.db. Umí vytvořit tabulku, přidávat rezervace a vypsat je do konzole.

Teď přijde další krok: místo textového menu v konzoli se stejná logika začne ovládat přes GUI, tedy grafické uživatelské rozhraní.

To znamená: místo input() budou textová pole, místo číselné volby budou tlačítka, a místo textového výpisu v konzoli se data zobrazí v okně aplikace.

Co se v této lekci naučíš

  • Co znamená zkratka GUI.
  • Jaký je rozdíl mezi konzolovým programem a grafickou aplikací.
  • Proč je pro první GUI v Pythonu vhodný modul tkinter.
  • Jak vytvořit první okno aplikace.
  • Jak do okna vložit text, vstupní pole a tlačítko.
  • Jak reagovat na kliknutí tlačítka.
  • Jak napojit GUI na existující databázové funkce.
  • Jak převést konzolový rezervační systém na jednoduchou desktopovou aplikaci.
Hlavní myšlenka této kapitoly:
Logika programu zůstává stejná, mění se způsob ovládání.

Na co tato kapitola navazuje

  1. Existuje databázový soubor reservations.db.

  5. Existuje funkce add_reservation(), která zatím používá input().
  6. V další fázi se konzolové ovládání nahradí grafickým oknem.

Krok 1: co je GUI

GUI je zkratka pro Graphical User Interface, tedy grafické uživatelské rozhraní.

Je to způsob ovládání programu pomocí vizuálních prvků, například:

  • oken,
  • tlačítek,
  • textových polí,
  • seznamů,
  • nabídek.

V konzolovém programu se hodnoty zadávají textově do terminálu. V GUI aplikaci se stejné hodnoty zapisují do polí v okně.

Rozdíl jednou větou:
Konzole = program se ovládá textem.
GUI = program se ovládá grafickými prvky.
Krok 2: proč začít právě s tkinter

Pro první GUI v Pythonu je velmi vhodný modul tkinter.

  • Je jednoduchý na začátek.
  • Bývá součástí běžné instalace Pythonu.
  • Umožní rychle vytvořit první okno, tlačítko i vstupní pole.
  • Je ideální pro pochopení základních principů GUI.

Na začátku je výhodné zůstat u co nejjednoduššího řešení. Cílem není vytvořit „krásnou moderní aplikaci“, ale pochopit, jak se GUI skládá.

Ověření, že tkinter funguje 
py -m tkinter

Když se po spuštění otevře malé okno Tcl/Tk, je všechno připravené a není potřeba nic doinstalovávat.

Tohle je nejdůležitější metodický bod celé kapitoly.

Při přechodu z konzole na GUI se nemění všechno. Ve skutečnosti se mění hlavně ovládání programu.

Co zůstane stejné

  • funkce connect(),
  • funkce init_db(),
  • funkce fetch_reservations(),
  • princip ukládání rezervace do tabulky reservations. :contentReference[oaicite:2]{index=2}

Co se změní

  • místo menu v konzoli vznikne okno,

  • místo zadání čísla volby budou tlačítka,

Jedna velmi přesná věta:
GUI není nový program od nuly. GUI je nový způsob, jak ovládat stejnou logiku.
Krok 4: proč současná funkce add_reservation() nestačí pro GUI

Ve stávajícím skriptu funkce add_reservation() čte data přes input(). :contentReference[oaicite:3]{index=3}

Současná konzolová verze 
def add_reservation():
    name = input("Jméno: ").strip()
    date = input("Datum (YYYY-MM-DD): ").strip()
    arrival = input("Příchod (HH:MM): ").strip()
    departure = input("Odchod (HH:MM): ").strip()

    conn = connect()
    conn.execute(
        "INSERT INTO reservations (name, date, arrival, departure) VALUES (?, ?, ?, ?);",
        (name, date, arrival, departure)
    )
    conn.commit()
    conn.close()
    print("Uloženo.")

Tato verze funguje dobře v konzoli, ale v grafické aplikaci se nehodí, protože GUI nebude číst data z terminálu.

V GUI je potřeba mít funkci, která dostane hodnoty jako parametry, například:

Verze vhodná pro GUI 
def add_reservation_record(name, date, arrival, departure):
    conn = connect()
    conn.execute(
        "INSERT INTO reservations (name, date, arrival, departure) VALUES (?, ?, ?, ?);",
        (name, date, arrival, departure)
    )
    conn.commit()
    conn.close()
Důvod této změny:
GUI přečte text z polí v okně a potom zavolá funkci s konkrétními hodnotami.
Krok 5: první okno aplikace

Úplně první GUI aplikace může být velmi malá. Stačí vytvořit okno a nechat ho otevřené.

Úplně první okno 
import tkinter as tk

root = tk.Tk()
root.title("Moje první aplikace")
root.mainloop()

Co znamená každý řádek

  • import tkinter as tk = načte modul tkinter a zkrátí jeho název na tk.
  • root = tk.Tk() = vytvoří hlavní okno aplikace.
  • root.title(...) = nastaví titulek okna.
  • root.mainloop() = spustí hlavní smyčku aplikace a udržuje okno otevřené.
Bez mainloop() by okno hned zmizelo.
Krok 6: přidání textu, vstupních polí a tlačítka

Jakmile existuje okno, mohou se do něj vložit grafické prvky. V tkinteru se jim často říká widgety.

Nejdůležitější widgety pro tuto aplikaci

  • Label = textový popisek,
  • Entry = vstupní pole pro text,
  • Button = tlačítko,
  • Treeview = tabulka nebo seznam dat.

Jednoduchý příklad s jedním polem a jedním tlačítkem může vypadat takto:

Základní vstupní prvek s tlačítkem 
import tkinter as tk

root = tk.Tk()
root.title("Zkouška GUI")

label = tk.Label(root, text="Jméno:")
label.pack()

entry = tk.Entry(root)
entry.pack()

button = tk.Button(root, text="Uložit")
button.pack()

root.mainloop()

Tohle zatím nic neukládá do databáze, ale ukazuje základní stavební kameny GUI.

Krok 7: co znamená kliknutí na tlačítko

Tlačítko samo o sobě nic nedělá. Musí mít přiřazenou funkci, která se spustí po kliknutí.

Tlačítko s reakcí na kliknutí 
def pozdrav():
    print("Tlačítko bylo stisknuto.")

button = tk.Button(root, text="Klikni", command=pozdrav)

Co je tady důležité

  • command=pozdrav znamená: po kliknutí spusť funkci pozdrav.
  • Nezapisuje se pozdrav(), ale jen pozdrav.
  • Tím se nepouští funkce hned při vytvoření tlačítka, ale až po kliknutí.
Klíčový princip GUI:
Uživatel klikne → spustí se funkce.
Krok 8: jak přečíst text z pole

V GUI se data nezadávají přes input(), ale přes vstupní pole. Hodnota pole se přečte metodou .get().

Čtení textu z Entry 
name = entry_name.get().strip()

To znamená:

  • entry_name = konkrétní vstupní pole,
  • .get() = vezmi text, který je v něm napsaný,
  • .strip() = odstraň mezery na začátku a na konci.

Přesně takto se pak v GUI načtou jméno, datum, příchod i odchod.

Krok 9: celá první GUI verze rezervační aplikace

Teď už se všechny části spojí dohromady. Vznikne samostatný soubor, například app_gui.py.

app_gui.py – první GUI verze nad existující databází 
import tkinter as tk
from tkinter import ttk, messagebox
import sqlite3
from pathlib import Path

DB_FILE = Path(__file__).resolve().with_name("reservations.db")


def connect():
    return sqlite3.connect(str(DB_FILE))


def init_db():
    conn = connect()
    conn.execute("""
    CREATE TABLE IF NOT EXISTS reservations (
        id        INTEGER PRIMARY KEY AUTOINCREMENT,
        name      TEXT    NOT NULL,
        date      TEXT    NOT NULL,
        arrival   TEXT    NOT NULL,
        departure TEXT    NOT NULL
    );
    """)
    conn.commit()
    conn.close()


def add_reservation_record(name, date, arrival, departure):
    conn = connect()
    conn.execute(
        "INSERT INTO reservations (name, date, arrival, departure) VALUES (?, ?, ?, ?);",
        (name, date, arrival, departure)
    )
    conn.commit()
    conn.close()


def fetch_reservations():
    conn = connect()
    rows = conn.execute("""
        SELECT id, name, date, arrival, departure
        FROM reservations
        ORDER BY date ASC, arrival ASC;
    """).fetchall()
    conn.close()
    return rows


def refresh_table():
    for item in tree.get_children():
        tree.delete(item)

    rows = fetch_reservations()
    for row in rows:
        tree.insert("", tk.END, values=row)


def save_reservation():
    name = entry_name.get().strip()
    date = entry_date.get().strip()
    arrival = entry_arrival.get().strip()
    departure = entry_departure.get().strip()

    if not name or not date or not arrival or not departure:
        messagebox.showwarning("Chyba", "Vyplň všechna pole.")
        return

    add_reservation_record(name, date, arrival, departure)
    messagebox.showinfo("Hotovo", "Rezervace byla uložena.")

    entry_name.delete(0, tk.END)
    entry_date.delete(0, tk.END)
    entry_arrival.delete(0, tk.END)
    entry_departure.delete(0, tk.END)

    refresh_table()

Krok 10: jak je aplikace složená

Tahle aplikace se skládá ze tří hlavních částí:

  1. Databázová logika – připojení, vytvoření tabulky, vložení záznamu, načtení dat.
  2. GUI prvky – okno, texty, pole, tlačítko, tabulka.
  3. Propojovací funkce – například save_reservation(), která vezme data z GUI a uloží je do databáze.
Tohle je nejdůležitější vztah:
GUI sbírá vstup od uživatele, funkce ho zpracuje, databáze ho uloží, a tabulka v okně se znovu načte.
Krok 11: co dělá save_reservation()

Funkce save_reservation() je hlavní spojka mezi GUI a databází.

  1. Přečte text ze čtyř vstupních polí.
  2. Zkontroluje, že žádné pole není prázdné.
  3. Zavolá add_reservation_record(...).
  4. Zobrazí potvrzovací okno.
  5. Vymaže vstupní pole.
  6. Obnoví tabulku rezervací v okně.

To je přesně to, co dříve v konzolové verzi dělalo menu a funkce add_reservation(), jen teď už ne přes textový vstup, ale přes okno aplikace. :contentReference[oaicite:4]{index=4}

Krok 12: jak aplikaci spustit

GUI verze bude uložená v novém souboru, například app_gui.py.

Spuštění GUI aplikace 
py app_gui.py

Po spuštění se otevře okno rezervačního systému.

  • Do polí se zapíše jméno, datum, příchod a odchod.
  • Po stisknutí tlačítka se rezervace uloží.
  • Ve spodní části okna se zobrazí seznam uložených rezervací.
