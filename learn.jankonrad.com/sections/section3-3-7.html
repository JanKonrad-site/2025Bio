3.3.7 Funkce – základní nástroj pro čistý a opakovatelný kód

Co se naučíš?

Funkce jsou „balíčky instrukcí“, které si můžeš pojmenovat a spouštět kdykoliv znovu. Díky tomu nebudeš kopírovat stejný kód pořád dokola a tvoje programy budou přehlednější.

  • Co je funkce: proč ji používáme a jak ti šetří práci.
  • Deklarace vs volání: proč se po vytvoření funkce nic nevypíše.
  • Scope: kde proměnné „žijí“ (global / function / block).
  • Parametry a argumenty: jak udělat funkci „flexibilní“.
  • Funkce + smyčka: praktický nástroj printRange(start, end).
Tip: Když se ti kód opakuje 2× a víc, je to kandidát na funkci.

1) Co je funkce (a proč ji vůbec chceme)

JavaScript má spoustu hotových funkcí – třeba alert(), console.log(), Math.random() nebo různé funkce na práci s textem a datem. Ale hlavní síla je v tom, že si umíš napsat vlastní.

Funkce = znovupoužitelný „recept“

Funkce je pojmenovaný blok kódu, který můžeš spustit kdykoliv. Místo kopírování kódu ho napíšeš jednou a pak ho jen „zavoláš“.

Příklad z praxe: „skok ve hře“ = pohyb nahoru + animace + zvuk. Nechceš to psát pokaždé znovu.
Kdy použít funkci
  • Když se něco opakuje.
  • Když chceš kód zpřehlednit (pojmenovat kus logiky).
  • Když chceš snadno změnit chování na jednom místě.
  • Když chceš udělat „nástroj“, který funguje s různými vstupy.
2) Vytvoř si první funkci (deklarace vs volání)

Funkci vytvoříš (deklaruješ) klíčovým slovem function, jménem a tělem v { }. Důležitý detail: deklarace funkci nespustí. Spustí ji až volání. 

function sayHello() {
  console.log("Hello World!");
}

// Zatím se nic nestane (funkce jen existuje)
sayHello(); // teď se spustí a vypíše text
Zapamatuj si:
Deklarace = „vytvoř funkci“ (ulož instrukce).
Volání = „spusť funkci“ (vykonej instrukce).
Mini úkol: zavolej funkci 4×

Zavolej sayHello() čtyřikrát a sleduj konzoli.

sayHello();
sayHello();
sayHello();
sayHello();
3) Scope (kde proměnné „žijí“ a kdo je může používat)

Scope říká, kde můžeš proměnnou použít. Jinak řečeno: kde je proměnná „viditelná“ a kde už ne.

Přehled scope (rychlá tabulka)
Typ scope Kde vzniká Kde je dostupná Poznámka
Global scope Mimo funkce / mimo blok „Skoro všude“ (i uvnitř funkcí) Pozor na kolize názvů a „nechtěné přepisování“.
Function scope Uvnitř funkce Jen uvnitř té funkce Bezpečné: zvenku se to nedá omylem přepsat.
Block scope Uvnitř bloku { } (if/for/while) Jen uvnitř toho bloku Platí pro let a const. var blok nerespektuje.

Global → funkce vidí globální proměnnou:

let greeting = "Ahoj!"; // globální proměnná

function showGreeting() {
  console.log(greeting); // funguje, protože funkce vidí globál
}

showGreeting();

Funkce → zvenku neuvidíš proměnnou uvnitř funkce:

function testScope() {
  let localValue = 123; // lokální (function scope)
  console.log(localValue); // OK
}

testScope();
console.log(localValue); // ❌ ReferenceError (neexistuje mimo funkci)
Proč je to dobré? Lokální proměnné chráníš před tím, aby je někdo omylem měnil z jiné části programu.
Block scope: proč je let bezpečnější než var (klikni) 
if (true) {
  let x = "jen uvnitř bloku";
}
// console.log(x); // ❌ nejde

if (true) {
  var y = "var blok nerespektuje";
}
console.log(y); // ✅ jde (a to je často zdroj bugů)
4) Parametry a argumenty (udělej funkci flexibilní)

Zatím funkce dělala pořád to samé. Parametry umožní poslat do funkce „vstup“.

Důležité rozlišení
  • Parametr = „proměnná“ v definici funkce (uvnitř závorek při deklaraci).
  • Argument = konkrétní hodnota, kterou pošleš při volání funkce.

Ukázka: pozdrav různých lidí

function sayHello(name) { // name = parametr
  console.log("Ahoj " + name);
}

sayHello("Tom");    // "Tom" = argument
sayHello("Alice");  // "Alice" = argument

Více parametrů: sčítání dvou čísel

function addNumbers(num1, num2) {
  console.log(num1 + num2);
}

addNumbers(5, 3); // 8
Pořadí argumentů je důležité: první argument jde do prvního parametru, druhý do druhého, atd.
Mini cvičení: další matematické funkce

Vytvoř podobné funkce pro: odčítání, násobení a dělení. Funkce ať vezmou dvě čísla a vypíšou výsledek.

Řešení (zobraz až po pokusu) 
function subtractNumbers(num1, num2) {
  console.log(num1 - num2);
}

function multiplyNumbers(num1, num2) {
  console.log(num1 * num2);
}

function divideNumbers(num1, num2) {
  console.log(num1 / num2);
}

subtractNumbers(10, 4); // 6
multiplyNumbers(6, 7);  // 42
divideNumbers(20, 5);   // 4
5) Cvičení: Custom Range Printer (funkce + smyčka)

Napiš funkci printRange(start, end), která vypíše do konzole všechna čísla od start až po end (včetně).

Příklad: printRange(3, 7) → vypíše: 3 4 5 6 7
Řešení (zobraz až po pokusu) 
function printRange(start, end) {
  for (let i = start; i <= end; i++) {
    console.log(i);
  }
}

printRange(3, 7);
printRange(10, 15);
Co je na tom důležité:
Funkce je „stroj“: jednou ji napíšeš a pak ji používáš s různými čísly. Smyčka uvnitř funkce bere start/end jako „živá“ data (ne pevně napsaná).

Tahák: Funkce – rychlé shrnutí

  • Deklarace funkce:
    function name() { ... }
  • Volání funkce:
    name();
  • Parametr vs argument:
    Parametr je v definici (function greet(name)), argument při volání (greet("Tom")).
  • Scope:
    Globální proměnné vidíš skoro všude. Lokální (uvnitř funkce) vidíš jen uvnitř funkce. let/const respektují bloky, var ne.
  • Funkce + smyčka = nástroj:
    printRange(start, end) je „stroj“ na výpis rozsahu čísel.
Pravidlo čistého kódu: Pokud to pojmenuješ dobře (např. printRange), kód se čte sám.
Scope detailně: proč je to důležité

Scope je „hranice viditelnosti“ proměnné. Pomáhá ti:

  • Neudělat nechtěný bug přepsáním hodnoty z jiné části programu.
  • Vyhnout se konfliktům názvů (stejný název v jiné funkci nevadí).
  • Udržet program přehledný (proměnné existují jen tam, kde dávají smysl).
Častý problém s var
for (var i = 0; i < 3; i++) {}
console.log(i); // ✅ 3 (var „proteče“ ven)

for (let j = 0; j < 3; j++) {}
// console.log(j); // ❌ j není definované (správně)

Proto dnes skoro vždy používáme let/const.

Nápověda: matematické funkce
  • Zkopíruj addNumbers a změň jen název + operátor.
  • Odčítání: -, násobení: *, dělení: /.
  • Parametry mohou zůstat stejné (num1, num2) – jsou lokální.
Nápověda: printRange(start, end)
  1. Definuj funkci se dvěma parametry: start a end.
  2. Uvnitř udělej for smyčku od start do end včetně.
  3. Vypisuj i přes console.log(i).
function printRange(start, end) {
  // for smyčka sem
}
Detail: proč funkce šetří práci

Když později změníš, co má „pozdrav“ dělat, upravíš to jen v jednom místě (v těle funkce) a všechny výskyty se automaticky chovají podle nové verze. 

function greet() {
  console.log("Ahoj!");
}

// 20× v kódu jen voláš greet();
// a když chceš změnu, upravíš jen greet()
Detail: správný tvar deklarace

Function declaration vypadá takto:

function name(parameters) {
  // body
}

Brzy si ukážeme i další způsoby (function expression, arrow functions), ale začínáme nejčistší klasikou.

Detail: volání funkce

Závorky () jsou signál: „teď to spusť“. Bez závorek máš jen odkaz na funkci jako na hodnotu. 

function test() { console.log("OK"); }

test;   // jen odkaz
test(); // spustí se
Detail: function scope

Lokální proměnné existují jen v „prostoru“ té funkce. Říká se tomu execution context. 

function a() {
  let x = 1;
  console.log(x);
}

function b() {
  let x = 2; // stejné jméno, žádný konflikt
  console.log(x);
}

a(); // 1
b(); // 2
Detail: parametr vs argument

Jedna věta na zapamatování:

Parametr je „krabička“ v definici funkce, argument je „obsah“, který do krabičky vložíš při volání. 
function greet(name) {   // parametr: name
  console.log("Ahoj " + name);
}

greet("Tom"); // argument: "Tom"
