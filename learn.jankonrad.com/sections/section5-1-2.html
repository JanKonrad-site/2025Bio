Speciální proměnné v Pythonu: __name__ a __file__

Některé proměnné nevytváří programátor, ale nastavuje je Python automaticky. Poznáš je často podle dvou podtržítek na začátku i na konci názvu (např. __name__).

__name__ – „pod jakým jménem Python tento soubor právě používá“

__name__ ukládá jméno modulu. Záleží, jestli soubor spouštíš nebo importuješ.

  • Když soubor spustíš přímo: __name__ == "__main__"
  • Když soubor importuješ: __name__ == "nazev_souboru" (bez .py)

Mini příklad:

print("__name__ =", __name__)
Proč je to užitečné: Díky tomu můžeš mít v souboru funkce (knihovnu) a zároveň „spouštěcí část“, která se spustí jen při přímém spuštění.
__file__ – „kde na disku leží tento soubor“

__file__ ukládá cestu k aktuálnímu .py souboru. Hodí se, když chceš ukládat soubory (např. databázi) do stejné složky jako skript. 

print("__file__ =", __file__)
Proč se hodí u databází: SQLite databáze je soubor. Když znáš cestu ke skriptu, umíš přesně určit, kde má ležet DB soubor.

3.2.2 SQLite v Pythonu – první databáze: otevřít, zapsat, přečíst (stavíme z bloků)

Budeme stavět jednoduchou miniaplikaci „Moje filmy“ po malých blocích. V každém kroku zkopíruješ kód, spustíš ho a ověříš výstup. Každý blok má jasný cíl: co přesně se má stát a jak to poznáš.

Workflow v IDE (jak to budeme dělat prakticky)

  1. Máš jednu složku projektu (např. Databaze/).
  2. Pro každý krok vytvoříš nový soubor: movies_step1.py, movies_step2.py, …
  3. Databáze je soubor movies.db ve stejné složce jako aktuální .py soubor.
  4. Po spuštění vždy ověříš: výpis v konzoli + existenci souboru ve složce (F5).
Výstup celé kapitoly: Umíš databázi založit (soubor), vytvořit tabulku, vložit film a vypsat filmy.
Krok 1: databáze = soubor (cesta, resolve, with_name, exists)
movies_step1.py 
import sqlite3
from pathlib import Path

DB_FILE = Path(__file__).resolve().with_name("movies.db")

print("DB path:", DB_FILE)
print("DB exists:", DB_FILE.exists())

input("ENTER...")
Vysvětlení kódu „slovo od slova“
  • import sqlite3 = načti modul, který umí pracovat se SQLite databází (zatím ho jen připravíme).
  • from pathlib import Path = z modulu pathlib si vezmeme třídu Path pro práci s cestami.
  • DB_FILE = ... = vytvoříme proměnnou, do které uložíme cestu k databázi.
  • Path(__file__) = „udělej objekt cesty“ z cesty k aktuálnímu souboru (to nastavuje Python).
  • .resolve() = převede cestu na „vyřešenou“ (typicky absolutní). Je to Python metoda (ne SQL).
  • .with_name("movies.db") = vezmi stejnou složku, ale změň název souboru na movies.db.
  • print("DB path:", DB_FILE) = vypiš, kam databáze přesně míří.
  • DB_FILE.exists() = zeptá se systému, jestli soubor existuje (Python metoda, ne SQLite).
  • input("ENTER...") = pauza, aby se okno nezavřelo hned.
Spusť a ověř
  1. Spusť: python movies_step1.py
  2. Uvidíš cestu k DB souboru.
  3. Pokud DB ještě není, uvidíš DB exists: False.
Krok 2: otevřít DB (connect) a zavřít (close) – vznik souboru
movies_step2.py 
import sqlite3
from pathlib import Path

DB_FILE = Path(__file__).resolve().with_name("movies.db")

print("DB path:", DB_FILE)
print("DB exists BEFORE:", DB_FILE.exists())

conn = sqlite3.connect(str(DB_FILE))
print("Spojení otevřeno.")

conn.close()
print("Spojení zavřeno.")

print("DB exists AFTER:", DB_FILE.exists())
input("ENTER...")
Vysvětlení kódu „slovo od slova“
  • conn = proměnná, do které uložíme „spojení“ na databázi.
  • sqlite3.connect(...) = otevře databázi. Pokud soubor neexistuje, SQLite ho obvykle vytvoří.
  • str(DB_FILE) = převede objekt Path na obyčejný text (cestu). Děláme to pro jistotu konzistence.
  • conn.close() = zavře spojení. Je to důležité, aby se spojení „nezapomnělo“ otevřené.
  • DB exists AFTER = kontrola: soubor by teď měl existovat.
Spusť a ověř
  1. Spusť: python movies_step2.py
  2. Ve složce by měl být nový soubor movies.db (obnov F5).
  3. Očekávání: DB exists AFTER: True
Krok 3: první SQL – vytvoření tabulky (CREATE TABLE) + commit
movies_step3.py 
import sqlite3
from pathlib import Path

DB_FILE = Path(__file__).resolve().with_name("movies.db")

conn = sqlite3.connect(str(DB_FILE))

conn.execute("""
CREATE TABLE IF NOT EXISTS movies (
    id    INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT    NOT NULL,
    year  INTEGER
);
""")

conn.commit()
conn.close()

print("Hotovo: tabulka movies je připravená.")
input("ENTER...")
Vysvětlení „slovo od slova“ (Python část)
  • conn = sqlite3.connect(...) = otevřeme spojení.
  • conn.execute(""" ... """) = pošleme databázi SQL příkaz jako text.
  • conn.commit() = potvrdíme změny (CREATE TABLE mění databázi).
  • conn.close() = zavřeme spojení.
Vysvětlení „slovo od slova“ (SQL část uvnitř trojitých uvozovek)
  • CREATE TABLE = vytvoř tabulku (strukturu).
  • IF NOT EXISTS = pokud už existuje, nic nedělej (nevyhodí to chybu při opakovaném spuštění).
  • movies = název tabulky.
  • ( ... ) = v závorkách jsou definice sloupců (jméno, typ, pravidla).
  • id = jméno sloupce.
  • INTEGER = typ (celé číslo).
  • PRIMARY KEY = unikátní identifikátor řádku.
  • AUTOINCREMENT = SQLite si pamatuje poslední ID a zvyšuje ho.
  • title TEXT NOT NULL = textový sloupec, který nesmí být prázdný (NULL).
  • year INTEGER = rok jako celé číslo (může chybět → NULL).
Krok 4: ověření tabulek (sqlite_master) + proč se objeví sqlite_sequence
movies_step4.py 
import sqlite3
from pathlib import Path

DB_FILE = Path(__file__).resolve().with_name("movies.db")

conn = sqlite3.connect(str(DB_FILE))

rows = conn.execute("SELECT name FROM sqlite_master WHERE type='table';").fetchall()

conn.close()

print("Tabulky v databázi:", rows)
input("ENTER...")
Vysvětlení „slovo od slova“
  • sqlite_master = systémová tabulka SQLite, kde jsou informace o tabulkách a dalších objektech.
  • SELECT name = „vyber sloupec name“ (název).
  • FROM sqlite_master = „z tabulky sqlite_master“.
  • WHERE type='table' = „jen ty řádky, kde je typ = tabulka“.
  • fetchall() = vrať všechny výsledné řádky jako seznam.
  • Výsledek vypadá jako [('movies',), ('sqlite_sequence',)], protože každý řádek je tuple.
Krok 5: vložení filmu (INSERT) + proč parametry (?, ?)
movies_step5.py 
import sqlite3
from pathlib import Path

DB_FILE = Path(__file__).resolve().with_name("movies.db")

title = input("Název filmu: ").strip()
year_text = input("Rok (ENTER = nevyplnit): ").strip()
year = int(year_text) if year_text else None

conn = sqlite3.connect(str(DB_FILE))

conn.execute(
    "INSERT INTO movies (title, year) VALUES (?, ?);",
    (title, year)
)

conn.commit()
conn.close()

print("Uloženo.")
input("ENTER...")
Vysvětlení „slovo od slova“ (hlavní řádky)
  • title = input(...) = načtu název filmu od uživatele.
  • .strip() = odstraní mezery na začátku a na konci.
  • year_text = načtu rok jako text (protože input vždy vrací text).
  • year = int(year_text) = převedu text na číslo (jen když něco bylo zadáno).
  • else None = pokud rok nebyl zadán, uložíme „None“ (v DB se to uloží jako NULL).
  • INSERT INTO movies = vkládáme do tabulky movies.
  • (title, year) = sloupce, které vyplňujeme.
  • VALUES (?, ?) = dvě hodnoty dodáme bezpečně jako parametry.
  • (title, year) (v Pythonu) = tuple s hodnotami, které doplní otazníky.
  • commit() = bez něj se změna nemusí uložit do souboru.
Krok 6: čtení filmů (SELECT) + for cyklus (včetně rozbalení tuple)
movies_step6.py 
import sqlite3
from pathlib import Path

DB_FILE = Path(__file__).resolve().with_name("movies.db")

conn = sqlite3.connect(str(DB_FILE))

rows = conn.execute("SELECT id, title, year FROM movies ORDER BY id DESC;").fetchall()

conn.close()

print("Obsah databáze:")

for (movie_id, title, year) in rows:
    yr = year if year is not None else "?"
    print(f"#{movie_id} | {title} ({yr})")

input("ENTER...")
Vysvětlení SELECT „slovo od slova“
  • SELECT id, title, year = vyber tři sloupce.
  • FROM movies = z tabulky movies.
  • ORDER BY id DESC = seřaď podle id sestupně (nejnovější nahoře).
  • fetchall() = vezmi všechny řádky výsledku do seznamu.
  • rows = seznam, kde každý prvek je tuple, např. (1, "Matrix", 1999).
Vysvětlení for cyklu „slovo od slova“
  • for = „pro každou položku“.
  • (movie_id, title, year) = tři proměnné, do kterých Python „rozbalí“ tuple.
  • in rows = „v seznamu rows“.
  • : = začátek bloku cyklu.
  • odsazení = řádky pod tím patří do cyklu a opakují se pro každý řádek z DB.
  • yr = year if ... else ... = pokud je rok None, vypíšeme otazník (jen pro hezký výpis).
  • print(f"...") = f-string: vloží proměnné do textu.
Proč rozbalujeme tuple: místo row[0], row[1], row[2] máme rovnou pojmenované proměnné.

Mini kontrola (co musí fungovat)

  • Po kroku 2 existuje soubor movies.db.
  • Po kroku 3 existuje tabulka movies.
  • Po kroku 5 vložíš film a nepadne to chybou.
  • Po kroku 6 se film vypíše (to je „podívám se do databáze“).

resolve / with_name / exists (Python, ne SQL)

Tohle jsou metody třídy Path z Pythonu (pathlib), ne funkce SQLite.

  • resolve() – dá „vyřešenou“ cestu (typicky absolutní), aby bylo jasné, kam cesta míří.
  • with_name("movies.db") – ponechá složku, změní jen název souboru.
  • exists() – vrátí True/False, jestli soubor na disku existuje.

Proč str(DB_FILE)?

DB_FILE je objekt typu Path. sqlite3.connect() potřebuje cestu k souboru.

  • V mnoha instalacích jde předat i Path přímo.
  • str(DB_FILE) je nejjistější: převede cestu na obyčejný text, který funguje všude stejně.
  • Když by někde nastal problém s typem, str(...) ho odstraní.

Proč je movies.db v editoru „šifrovaný“?

.db je binární formát, ne textový soubor.

  • Editor textu ho otevře jako text → proto vypadá jako „nesmysly“.
  • Některé kousky textu (např. názvy) se mohou náhodně objevit čitelně, protože uvnitř DB jsou uložené jako text.
  • „Podívat se do DB“ se dělá buď přes SELECT (Pythonem), nebo přes DB nástroj (např. DB Browser for SQLite).

Proč [('movies',), ('sqlite_sequence',)]?

fetchall() vrací seznam řádků. Každý řádek je tuple.

  • Dotaz vrací 1 sloupec (name) → každý řádek je tuple s 1 prvkem: ('movies',)
  • Ta čárka uvnitř ('movies',) říká: „tohle je tuple“.

Když chceš čistý seznam jmen:

names = [name for (name,) in rows]

Co je sqlite_sequence?

sqlite_sequence je interní tabulka SQLite, která souvisí s AUTOINCREMENT.

  • Když máš sloupec INTEGER PRIMARY KEY AUTOINCREMENT, SQLite si ukládá poslední použité číslo.
  • Díky tomu může jistě pokračovat v číslování při dalších INSERTech.
  • Často se objeví až po prvním vložení (když má co evidovat).

SQL slovníček: SELECT / FROM / WHERE / ORDER BY

  • SELECT = vyber sloupce (co chci vidět)
  • FROM = z jaké tabulky
  • WHERE = podmínka (které řádky chci)
  • ORDER BY = řazení výsledků
  • DESC = sestupně (od největšího)

Příklad: SELECT title FROM movies WHERE year = 1999 ORDER BY id DESC;

Co dělá commit()?

commit() potvrdí změny, které mění databázi (např. CREATE, INSERT, UPDATE, DELETE).

  • Bez commit se změny mohou po zavření spojení ztratit.
  • Proto po INSERT i CREATE TABLE voláme commit.

CREATE TABLE – typy a omezení

  • INTEGER = celé číslo
  • TEXT = text
  • PRIMARY KEY = unikátní identifikátor
  • AUTOINCREMENT = automatické navyšování ID
  • NOT NULL = hodnota nesmí být prázdná (NULL)

INSERT – parametry (?, ?)

VALUES (?, ?) = hodnoty se doplní bezpečně přes parametry (ne skládáním textu).

Parametry posíláme jako tuple: (title, year).

for cyklus a rozbalení tuple

rows je seznam řádků. Každý řádek je tuple, např. (1, "Matrix", 1999).

for (movie_id, title, year) in rows:
    print(movie_id, title, year)

Rozbalení znamená: první prvek tuple jde do movie_id, druhý do title, třetí do year.

Ternární výraz (zkrácený if/else)

yr = year if year is not None else "?"

Je to zkráceně:

if year is not None:
    yr = year
else:
    yr = "?"
