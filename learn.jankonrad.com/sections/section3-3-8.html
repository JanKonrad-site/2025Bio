3.3.8 Funkce v JavaScriptu – return, hoisting a callback funkce

V této kapitole se naučíš, jak funkce „vracejí“ hodnoty, proč lze některé funkce volat dřív než jsou napsané, a jak fungují callbacky (funkce předávané jako argument).

Co se naučíš

Funkce nejsou jen „výpis do konzole“. Často potřebujeme, aby funkce něco spočítala a výsledek nám vrátila, abychom s ním mohli dál pracovat. Zároveň je dobré rozumět tomu, jak JavaScript „čte“ kód a proč se někdy funkce chovají jinak než proměnné.

  • Return: jak funkce vrací hodnotu a proč to mění způsob použití funkce.
  • Return ukončuje funkci: kód za return se už nespustí.
  • Hoisting: proč lze volat funkci dřív, než je zapsaná v souboru.
  • Callback: funkce jako argument (např. „co se má stát po dokončení“).
  • Praktické cvičení: login simulace s kontrolou chyb a callbackem.
Tip: Pracuj ve VS Code vždy jako nový mini-projekt: index.html + app.js a konzoli sleduj v DevTools (Inspect → Console).

Teorie

Doteď jsme často používali funkce tak, že uvnitř udělaly console.log(). To je užitečné pro kontrolu, ale má to omezení: výsledek „jen vypíšeš“ a dál s ním nic neuděláš.

Když chceme výsledek použít později, funkce ho musí vrátit. K tomu slouží return. Hodnota se vrátí zpět na místo, kde byla funkce zavolána.

Varianta A: jen vypisuju (výsledek „mizí“)

function multiply(a, b) {
  console.log(a * b);
}

multiply(5, 3);

Tohle je v pohodě, když jen něco ukazuješ. Ale nemáš hodnotu uloženou pro další práci.

Varianta B: vracím hodnotu (výsledek můžu uložit)

function multiply(a, b) {
  let result = a * b;
  return result;
}

let result = multiply(5, 3);
console.log(result);

return pošle hodnotu ven z funkce. Tady ji ukládáme do proměnné a pak vypíšeme.

Return se dá použít i „přímo“

Když funkce vrací hodnotu, můžeš ji použít třeba rovnou uvnitř console.log: 

console.log(multiply(5, 3));

Return ukončuje funkci

Jakmile se provede return, funkce končí. Kód pod return se už nespustí. 

function demo() {
  return "Hotovo";
  console.log("Toto se nikdy nespustí");
}

Mini cvičení: funkce isEven

Vytvoř funkci isEven(number), která vrátí true pro sudé číslo a false pro liché. Otestuj ji alespoň na 20 a 7 tak, že výsledky vypíšeš do konzole.

Start (index.html + app.js)

Vytvoř index.html a připoj app.js: 

<!-- index.html -->
<!DOCTYPE html>
<html lang="cs">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>3.3.8</title>
</head>
<body></body>
<script src="app.js"></script>
</html>

Úkol

  1. Napiš funkci isEven(number).
  2. Uvnitř použij modulo: number % 2.
  3. Funkce má vracet boolean (true/false), ne text.
  4. Otestuj: console.log(isEven(20)) a console.log(isEven(7)).
Řešení (referenční) 
function isEven(number) {
  return number % 2 === 0;
}

console.log(isEven(20)); // true
console.log(isEven(7));  // false

Teorie

JavaScript má chování zvané hoisting. Zjednodušeně: některé deklarace si JavaScript „připraví dopředu“ ještě před spuštěním kódu v daném scope. Díky tomu lze některé funkce volat i před tím, než jsou v souboru napsané.

Funkce (function declaration) jde volat i „nahoře“

doSomething();

function doSomething() {
  console.log("Funguje to.");
}

Let/const se před deklarací použít nesmí

console.log(a); // ReferenceError
let a = 10;

Let/const nejsou dostupné, dokud se nedeklarují.

Poznámka k var (starší styl)

U var se může stát, že se před deklarací nevypíše chyba, ale undefined. Je to tím, že se „zvedne“ deklarace, ale ne přiřazení hodnoty. 

console.log(b); // undefined
var b = 10;
Doporučení: Používej let a const. Deklarace (hlavně globální) je dobré mít přehledně nahoře v souboru.

Teorie

Callback je funkce, kterou předáš jako argument do jiné funkce. Ta druhá funkce rozhodne, kdy ji zavolá. Prakticky tím říkáš: „Až dokončíš tuhle část, udělej ještě toto.“

Příklad: odpočítávání a start

function startRace() {
  console.log("Závod začíná!");
}

function countdown(callback) {
  for (let i = 3; i >= 1; i--) {
    console.log(i);
  }

  // až odpočítání skončí, zavolej callback
  callback();
}

countdown(startRace);

Důležité: předáváš jméno funkce (startRace), ne volání (startRace()).

Zadání

Představ si, že vytváříš mini login pro administrátorskou stránku. Správné přihlašovací údaje jsou:

  • username: admin
  • password: 1234

Pokud jsou údaje správně, zavolej callback funkci (např. ta vypíše do konzole zprávu o přihlášení). Pokud je chyba, vypiš do konzole jasnou informaci, co je špatně.

Požadavky

  1. Napiš funkci login(username, password, callback).
  2. Ošetři chybové stavy:
    • chybí username
    • chybí password
    • špatné username
    • špatné password
  3. Při chybě vždy napiš do konzole jednu jasnou hlášku a ukonči funkci (pomocí return).
  4. Při úspěchu zavolej callback().

Testy (musí fungovat)

login("admin", "1234", loginAdmin);         // správně
login("", "1234", loginAdmin);              // chybí username
login("admin", "", loginAdmin);             // chybí password
login("user", "1234", loginAdmin);          // špatné username
login("admin", "password123", loginAdmin);  // špatné password

Řešení (referenční)

Řešení ukazuje styl „early return“: jakmile najdeš problém, vypíšeš chybu a funkci ukončíš.

Zobrazit řešení 
function login(username, password, callback) {
  // 1) chybí vstupy
  if (!username || !password) {
    console.log("Chyba: Uživatelské jméno a heslo nesmí být prázdné.");
    return;
  }

  // 2) kontrola username
  if (username !== "admin") {
    console.log("Chyba: Neplatné uživatelské jméno.");
    return;
  }

  // 3) kontrola password
  if (password !== "1234") {
    console.log("Chyba: Nesprávné heslo.");
    return;
  }

  // 4) úspěch -> callback
  callback();
}

function loginAdmin() {
  console.log("Přihlašuji do administrátorského účtu.");
}

// Testy
login("admin", "1234", loginAdmin);
login("", "1234", loginAdmin);
login("admin", "", loginAdmin);
login("user", "1234", loginAdmin);
login("admin", "password123", loginAdmin);

Proč je to napsané takhle?

  • Return jako „stop“: při chybě funkci ukončíš a dál se už nic neděje.
  • Čitelnost: místo hlubokého zanořování do if/else máš jasné kontroly v řadě.
  • Callback: samotný login řeší jen ověření, a „co dělat po úspěchu“ řeší callback.

Co jsme se naučili

  • Return: funkce může vracet hodnotu a ta se dá uložit do proměnné nebo použít přímo ve výrazu.
  • Return ukončuje funkci: kód za return se nespustí, což se hodí pro „early return“.
  • Hoisting: běžné deklarace funkcí lze volat i před tím, než jsou napsané v souboru.
  • Callback: funkce může být argument – druhá funkce ji zavolá ve správný okamžik.
  • Praktický pattern: kontrola chyb → return → úspěch → callback().
Tip: Pokud ti kód začíná „utíkat do hloubky“ (moc vnořených if/else), zvaž early return.
Return: co může vracet a proč se hodí „early return“

Co může return vracet?

  • Číslo: return 42;
  • Text: return "Ahoj";
  • Boolean: return true;
  • null / undefined
  • Komplexní typy (později): objekty, pole, funkce

Early return znamená: jakmile zjistíš problém, ukončíš funkci hned, místo abys dál zanořoval další if/else.

function getDiscount(totalPrice) {
  if (totalPrice <= 0) return 0;   // rychlé ukončení při chybě
  if (totalPrice < 100) return 0;  // nesplňuje podmínku
  return totalPrice * 0.3;         // sleva
}
Shrnutí

Return je způsob, jak dostat hodnotu ven z funkce a současně je to „stopka“ pro běh funkce.

Nápověda: isEven(number)
  • Sudé číslo je „dělitelné 2 beze zbytku“.
  • number % 2 vrátí zbytek po dělení dvěma.
  • Když je zbytek 0, číslo je sudé.
// výraz number % 2 === 0 už je sám o sobě boolean
Callback: proč nepíšeme startRace() a kde se používá

Když napíšeš startRace(), tak funkci okamžitě zavoláš a do argumentu se už předává její výsledek (většinou undefined). Callback ale potřebuješ předat jako funkci, aby ji druhá funkce mohla zavolat později. 

// správně: předávám funkci
countdown(startRace);

// špatně: volám ji hned teď
countdown(startRace());
Kde se callbacky používají v praxi
  • Události v DOM: tlačítko klik → zavolá se callback
  • Asynchronní akce: „až se něco dokončí, zavolej funkci“
  • Práce s kolekcemi: některé metody polí používají callback (později)
Co je admin?

Admin (administrátor) je uživatel s vyššími právy než běžný návštěvník. Typicky může:

  • spravovat obsah (např. přidávat/mazat články)
  • měnit nastavení webu
  • spravovat uživatele

V cvičení je „admin“ jen zjednodušená role pro trénink podmínek a callbacku.

Nápovědy: login(username, password, callback)
1) Nejprve zkontroluj prázdné vstupy
  • Můžeš použít if (!username) nebo if (username === "").
  • Stejně pro password.
  • Po vypsání chyby dej return; (ukončíš funkci).
2) Pak kontroluj údaje postupně
  • Když username !== "admin" → vypiš chybu a return.
  • Když password !== "1234" → vypiš chybu a return.
3) Při úspěchu zavolej callback
  • Když je vše OK, zavolej callback();.
  • Callback je funkce, kterou předáváš jako třetí argument při volání loginu.
