3.3.8 Funkce v JavaScriptu – return, hoisting a callback funkce

V této kapitole si pomalu a na jednoduchých příkladech vysvětlíme, co přesně dělá return, jak funguje hoisting (proč někdy lze funkci volat „dřív“), a proč se callbacky používají v praxi.

Co se naučíš

Funkce si můžeš představit jako „stroj“: dáš mu vstup, uvnitř něco zpracuje a může ti dát výstup. Return je právě ten výstup (návratová hodnota). Důležité je pochopit, že: volání funkce je výraz a jeho výsledkem je návratová hodnota funkce.

  • Return: návratová hodnota a proč ji bez console.log neuvidíš.
  • console.log vs return: výpis pro člověka vs výstup pro program.
  • Return ukončuje funkci: kód pod return se už nespustí.
  • Hoisting: kdy lze volat funkci „nahoře“ a kdy to naopak spadne.
  • Callback: funkce předaná jako argument – spustí se až uvnitř jiné funkce.
Připomínka operátorů:

Klíčová myšlenka (jednou větou)

Return nastaví návratovou hodnotu funkce. Výsledek volání funkce f() je právě to, co funkce vrátí přes return.

Příklad A: funkce vrací pevnou hodnotu

Pokud funkce vrátí 5, pak výraz giveMeFive() má hodnotu 5. 

function giveMeFive() {
  return 5;
}

let x = giveMeFive();
console.log(x); // 5

Příklad B: return nic „nevypisuje“

Tohle je častý zmatek: return vrací, ale neukazuje. Aby byl výstup vidět, musíš ho vypsat ty. 

function giveMeFive() {
  return 5;
}

giveMeFive();              // nic neuvidíš
console.log(giveMeFive()); // 5

console.log vs return (proč to není totéž)

console.log je jen „okno pro programátora“ – vypíše hodnotu, ale funkci tím nedáš návratovou hodnotu. Pokud funkce nemá return, její návratová hodnota je undefined.

Špatně pro další práci: jen log

function multiplyLog(a, b) {
  console.log(a * b);
}

let r = multiplyLog(5, 3);
console.log("r =", r); // r = undefined

Vidíš 15 (výpis), ale do r se uloží návratová hodnota funkce: undefined.

Správně: return

function multiply(a, b) {
  return a * b;
}

let r = multiply(5, 3);
console.log("r =", r); // r = 15

Tady už má program výsledek „v ruce“ a může s ním pracovat dál.

Return ukončuje funkci

function demo() {
  console.log("A");
  return "Hotovo";
  console.log("B"); // nikdy se nespustí
}

Jakmile se vykoná return, funkce okamžitě skončí. To se hodí pro tzv. „early return“ (např. při chybě ukončíš funkci hned).

Mini cvičení: isEven(number)

Napiš funkci isEven(number), která vrátí true pro sudé číslo a false pro liché. Použij % a ===.

Úkol

  1. Napiš function isEven(number) { ... }
  2. Vrať výsledek výrazu number % 2 === 0
  3. Otestuj: console.log(isEven(20)) a console.log(isEven(7))
Řešení (referenční) 
function isEven(number) {
  return number % 2 === 0;
}

console.log(isEven(20)); // true
console.log(isEven(7));  // false

Hoisting si představ jako dvě fáze

JavaScript při spuštění scope udělá zjednodušeně dvě věci:

  1. Příprava: „zaregistruje“ deklarace (např. function declaration).
  2. Vykonávání: potom teprve provádí řádek po řádku.

Příklad 1: function declaration (funguje i před zápisem)

main(); // funguje

function main() {
  console.log("Hlavní logika");
  helper();
}

function helper() {
  console.log("Pomocná funkce");
}

Proč to jde: main i helper jsou function declaration, takže jsou dostupné už při vykonávání prvního řádku.

Příklad 2: funkce uložená v const (tady to spadne)

sayHi(); // ReferenceError

const sayHi = function () {
  console.log("Ahoj");
};

Tady se neřeší „hoisting funkce“, ale „kdy existuje proměnná sayHi“. U const nesmíš proměnnou použít před deklarací.

Praktická rada: Hoisting je dobré znát, ale pro čitelnost je fajn psát kód tak, aby šel číst shora dolů. Když dáš pomocné funkce dolů, tak je dej jako function helper(){...} (declaration), ne jako const helper = function(){...}.

Co je callback (nejjednodušší definice)

Callback je funkce, kterou předáš jiné funkci jako argument. Vnější funkce pak rozhodne, kdy callback zavolá.

Příklad 1: nejjednodušší callback

function doWork(callback) {
  alert("1) Dělám práci...");
  callback(); // 2) až teď spustím další krok
}

function nextStep() {
  alert("2) Další krok.");
}

doWork(nextStep);

callback je parametr. Když voláš doWork(nextStep), parametr callback uvnitř dostane funkci nextStep.

Nejčastější chyba: závorky

Rozdíl:

  • doWork(nextStep) = předáváš funkci
  • doWork(nextStep()) = spouštíš funkci hned a předáváš její návratovou hodnotu

Kdy callback dává smysl (rozhodování)

Callback je nejvíc užitečný, když nechceš natvrdo psát „co přesně se stane“, ale jen rozhodnout „který scénář nastal“. Konkrétní akce necháš na tom, kdo funkci volá.

Příklad 2: dvě možnosti (povolit/zakázat)

function checkAge(age, onAllowed, onDenied) {
  if (age >= 18) {
    onAllowed();
  } else {
    onDenied();
  }
}

function allowed() { alert("Povoleno"); }
function denied()  { alert("Zakázáno"); }

checkAge(20, allowed, denied);
checkAge(15, allowed, denied);

Zadání

Zatím nemáme uživatelské inputy, takže si stav nasimulujeme pomocí proměnných. Napíšeš funkci, která na základě dvou booleanů spustí správnou akci.

Požadavky

Napiš funkci: 

function accessControl(isLoggedIn, isAdmin, onAdmin, onUser, onDenied) {
  // ...
}
  • Pokud isLoggedIn je false → zavolej onDenied().
  • Pokud je přihlášený a isAdmin je true → zavolej onAdmin().
  • Pokud je přihlášený a není admin → zavolej onUser().

Testy (zkopíruj a spusť)

function adminPanel()    { alert("Admin panel"); }
function userDashboard() { alert("Uživatelský dashboard"); }
function denied()        { alert("Nejsi přihlášen"); }

accessControl(false, false, adminPanel, userDashboard, denied);
accessControl(true,  false, adminPanel, userDashboard, denied);
accessControl(true,  true,  adminPanel, userDashboard, denied);
Řešení (referenční) 
function accessControl(isLoggedIn, isAdmin, onAdmin, onUser, onDenied) {
  if (!isLoggedIn) {
    onDenied();
    return;
  }

  if (isAdmin) {
    onAdmin();
  } else {
    onUser();
  }
}
Smysl cvičení: Tvoje funkce jen rozhoduje (logika). Konkrétní akce (alerty) dodá volající přes callbacky.

Co jsme se naučili

  • Return: návratová hodnota je výsledkem volání funkce f().
  • console.log: výpis pro člověka (ne návratová hodnota).
  • Hoisting: function f(){} lze volat i před definicí; funkce v const ne.
  • Callback: funkce jako argument – voláš ji až uvnitř jiné funkce.
  • ===: porovnává hodnotu i typ; == převádí typy; = přiřazuje.
Tip: Když se ztrácíš, vrať se k jedné větě: „Volání funkce je výraz a jeho hodnota je to, co funkce vrátí.“
Připomenutí: = vs == vs ===
Operátor Co dělá Příklad Výsledek
= Přiřazení hodnoty do proměnné let x = 5; x je 5
== Porovnání s převodem typů (může překvapit) 5 == "5" true
=== Striktní porovnání (hodnota i typ) 5 === "5" false
Doporučení: V moderním JS používej téměř vždy === a !==. U == vznikají překvapení kvůli automatickým převodům.
Return krok za krokem (co se děje při volání funkce)

Uvažuj tento řádek:

let x = giveMeFive();
  1. JS začne vyhodnocovat pravou stranu: giveMeFive().
  2. „Vstoupí“ do funkce a spustí její tělo.
  3. Narazí na return 5 → funkce okamžitě končí a vrací 5.
  4. Volání giveMeFive() se vyhodnotí na hodnotu 5.
  5. Teprve pak se dokončí přiřazení: x = 5.
Přesná formulace: Return vrací návratovou hodnotu funkce. Tato hodnota je výsledkem výrazu „volání funkce“.
Nejčastější zmatky u return
1) Funkce nemá return → výsledkem je undefined
function f() { console.log("A"); }
console.log(f()); // undefined
2) Return je jen v jedné větvi
function g(x) {
  if (x > 0) return "ok";
  // tady return chybí
}
console.log(g(-1)); // undefined
3) „Nic se nevypsalo“

Return nic nevypisuje. Chceš-li návratovou hodnotu vidět, musíš ji vypsat ty (např. console.log).

Nápověda: isEven(number)
  • % vrací zbytek po dělení.
  • Sudé číslo má zbytek po dělení 2 rovný 0.
  • === používáme, protože chceme přesně porovnat čísla bez převodů typů.
return number % 2 === 0;
Hoisting krok za krokem (na function declaration)

Uvažuj kód: 

main();

function main() {
  helper();
}

function helper() {
  console.log("Pomocná funkce");
}
Co se děje při startu:
  1. Příprava: JS si „zaregistruje“, že existují funkce main a helper.
  2. Vykonávání: začne běžet řádek 1 → main() už existuje, takže volání funguje.
  3. Uvnitř main se volá helper() → také existuje, takže se zavolá.
Výsledek: U function declaration jsou funkce dostupné už od začátku vykonávání daného scope.
Hoisting – tabulka: co funguje předem a co ne
Typ Ukázka Před deklarací Co se stane
Function declaration function f(){} funguje funkce je dostupná
let / const let x = 1; nefunguje ReferenceError
const + function expression const f = function(){}; nefunguje ReferenceError
var var x = 1; „projde“ undefined (deklarace nahoře, přiřazení až později)
Praxe: function declaration se hodí pro „pomocné funkce dole“. U let/const a function expression to takhle nefunguje.
Callback krok za krokem (co je v parametru callback)

Uvažuj kód:

function doWork(callback) {
  alert("Práce...");
  callback();
}

function nextStep() {
  alert("Další krok");
}

doWork(nextStep);
Co přesně se stane:
  1. Voláš doWork(nextStep).
  2. Parametr callback uvnitř doWork dostane hodnotu „funkce nextStep“.
  3. Uvnitř doWork se provede alert("Práce...").
  4. Potom callback() → tím se ve skutečnosti zavolá nextStep().
Klíč: callback není nic magického. Je to jen proměnná (parametr), ve které je uložená funkce.
Proč při předání callbacku nepíšu závorky?

nextStep (bez závorek) znamená „funkce jako hodnota“.
nextStep() (se závorkami) znamená „spusť funkci teď hned“. 

doWork(nextStep);   // předávám funkci
doWork(nextStep()); // spouštím ji hned a předávám výsledek

Callback chceme předat, aby ho vnější funkce mohla spustit až později (ve správný okamžik).

Kde se callbacky používají v praxi
1) Události v prohlížeči (kliknutí)
button.addEventListener("click", function () {
  console.log("Klik!");
});

Callback se zavolá až když uživatel klikne. (To se bude detailně probírat v DOM kapitole.)

2) „Až se něco dokončí“

Např. kontrola, validace, výpočet, který chceš rozdělit na kroky: udělej práci a pak zavolej další krok callbackem.

3) Práce s poli (později)

Metody jako forEach, map, filter berou callback, který se spustí pro každý prvek.

Nápověda: accessControl

Doporučený postup:

  1. Nejdřív řeš nepřihlášeného uživatele: pokud !isLoggedIn, zavolej onDenied() a return.
  2. Pak už řešíš jen admin vs user.
if (!isLoggedIn) {
  onDenied();
  return;
}
