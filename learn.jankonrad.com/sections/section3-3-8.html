3.3.8 Funkce v JavaScriptu – return, hoisting a callback funkce

V této kapitole se naučíš, jak funkce vrací hodnoty (return), proč lze některé funkce volat dřív než jsou napsané (hoisting), a jak fungují callbacky (funkce předávané jako argument).

Co se naučíš

Funkci si představ jako „stroj“: dostane vstup (argumenty), uvnitř něco udělá a může vrátit výstup. Dokud funkce jen vypisuje do konzole, výsledek „vidíš“, ale program ho neumí dál používat. return je způsob, jak výsledek opravdu předat zpět do zbytku kódu.

  • return: jak se vrácená hodnota objeví přesně v místě volání funkce.
  • console.log vs return: výpis pro tebe vs výstup pro program.
  • return ukončuje funkci: kód pod return se nevykoná.
  • hoisting: proč jde některé funkce volat „nahoře“.
  • callback: funkce jako argument (co se má stát po dokončení).
Tip: Kdykoli narazíš na =, ==, ===, otevři si připomínku:

Nejjednodušší představa

Funkce je jako výrobní linka. Když ji zavoláš, „na chvíli se přepneš“ dovnitř, linka něco udělá a na konci ti může podat hotový výrobek. Ten výrobek je return.

  • Argumenty = suroviny (co do funkce posíláš)
  • Tělo funkce = práce linky (výpočty, podmínky, smyčky)
  • return hodnota = hotový výrobek (výsledek, se kterým jde dál pracovat)

Tahák: 3 věty o return

  • return vrátí hodnotu na místo volání
  • po return funkce končí
  • bez return funkce vrací undefined

Příklad 0: funkce, která vrací pevnou hodnotu

Tohle je nejjednodušší „důkaz“, že volání funkce se chová jako hodnota: 

function giveMeFive() {
  return 5;
}

let x = giveMeFive();   // sem se uloží 5
console.log(x);         // 5

Důležitá myšlenka: výraz giveMeFive() se po provedení funkce chová, jako kdyby tam bylo prostě 5.

Příklad 1: funkce vrací výsledek výpočtu

function addOne(n) {
  return n + 1;
}

console.log(addOne(10)); // 11

Tady je vidět, že return není „výpis“. Je to výstup funkce. Hodnota se vrátí a teprve ty se rozhodneš, co s ní uděláš (vypsat, uložit, použít v podmínce…).

Příklad 2: proč console.log nestačí

Podívej, co udělá funkce, která jen vypisuje: 

function multiplyLog(a, b) {
  console.log(a * b);
}

let r = multiplyLog(5, 3); // co je v r?
console.log(r);

Výsledek:

  • na konzoli uvidíš 15 (protože uvnitř byl log)
  • pak uvidíš undefined (protože funkce nic nevrací)

Příklad 3: stejná věc správně s return

function multiply(a, b) {
  return a * b;
}

let r = multiply(5, 3); // r je 15
console.log(r);

Return ukončuje funkci

function demo() {
  console.log("A");
  return "Hotovo";
  console.log("B"); // nikdy se nespustí
}

Jakmile se vykoná return, funkce skončí. To je důležité i v podmínkách: když se return uvnitř if nespustí (podmínka je false), funkce pokračuje dál.

Více returnů: funkce vrací různé hodnoty podle situace

function isAdult(age) {
  if (age >= 18) {
    return true;
  } else {
    return false;
  }
}

Poznámka: výraz age >= 18 už sám o sobě vrací boolean, takže jde napsat i krátce: return age >= 18;

Mini cvičení: isEven(number)

Vytvoř funkci isEven(number), která vrátí true pro sudé číslo a false pro liché.

Start (index.html + app.js)

<!-- index.html -->
<!DOCTYPE html>
<html lang="cs">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>3.3.8</title>
</head>
<body></body>
<script src="app.js"></script>
</html>

Úkol

  1. Napiš funkci isEven(number).
  2. Uvnitř použij modulo: number % 2.
  3. Otestuj: console.log(isEven(20)) a console.log(isEven(7)).
Řešení (referenční) 
function isEven(number) {
  return number % 2 === 0;
}

console.log(isEven(20)); // true
console.log(isEven(7));  // false

Teorie (jednoduše)

Hoisting si představ jako „předběžné přečtení“. JavaScript si při startu daného scope všimne některých deklarací dřív, než začne vykonávat kód řádek po řádku.

Deklarace funkce (funguje i před zápisem)

doSomething();

function doSomething() {
  console.log("Funguje to.");
}

Let/const (nepoužitelné před deklarací)

console.log(a); // ReferenceError
let a = 10;

Poznámka k var

U var se může objevit undefined (deklarace je „nahoře“, přiřazení až později). 

console.log(b); // undefined
var b = 10;

Teorie (přirovnání)

Callback si představ jako „instrukci na papírku“. Jedna funkce udělá práci a na konci se podívá na papírek: „Co mám udělat potom?“ Ten papírek je callback.

Důležité pravidlo: když callback předáváš, nevoláš ho. Předáváš jen název funkce.

Nejjednodušší callback

function afterWork(callback) {
  console.log("Hotovo.");
  callback();
}

function sayBye() {
  console.log("Konec.");
}

afterWork(sayBye);

Zadání

Napiš funkci login(username, password, callback). Správný login je "admin" + "1234". Při úspěchu zavolej callback. Při chybě vypiš jasnou hlášku a ukonči funkci pomocí return.

Testy

login("admin", "1234", loginAdmin);         // správně
login("", "1234", loginAdmin);              // chybí username
login("admin", "", loginAdmin);             // chybí password
login("user", "1234", loginAdmin);          // špatné username
login("admin", "password123", loginAdmin);  // špatné password

Řešení (referenční)

Zobrazit řešení 
function login(username, password, callback) {
  if (!username) {
    console.log("Chyba: chybí uživatelské jméno.");
    return;
  }

  if (!password) {
    console.log("Chyba: chybí heslo.");
    return;
  }

  if (username !== "admin") {
    console.log("Chyba: neplatné uživatelské jméno.");
    return;
  }

  if (password !== "1234") {
    console.log("Chyba: nesprávné heslo.");
    return;
  }

  callback();
}

function loginAdmin() {
  console.log("Přihlašuji do administrátorského účtu.");
}

Co jsme se naučili

  • return vrací hodnotu do místa, kde byla funkce zavolána (volání funguje jako „hodnota“).
  • console.log je jen výpis pro člověka, ne výstup pro další výpočty.
  • po return funkce končí, kód pod ním se neprovede.
  • === je bezpečné porovnání (hodnota i typ), == dělá automatické převody.
  • callback je funkce předaná jako argument — voláš ji až uvnitř jiné funkce.
Tip: Pokud si nejsi jistý, co funkce vrací, dočasně si přidej log uvnitř funkce i log venku.
Připomenutí: = vs == vs === (a proč se v lekci používá ===)
Operátor Co znamená Příklad Výsledek
= Přiřazení hodnoty do proměnné let x = 5; x je 5
== Porovnání s převodem typů (může být zrádné) 5 == "5" true
=== Striktní porovnání (hodnota i typ) 5 === "5" false
Doporučení: V moderním JavaScriptu používej téměř vždy === a !==. Vyhneš se „automatickým převodům“ (coercion), které u == umí dělat překvapení.
Typické překvapení u ==
false == 0        // true
"" == 0           // true
null == undefined // true
Return krok za krokem: co se děje při volání funkce

Uvažuj tento řádek:

let r = addOne(10);
Krok Co JavaScript udělá Výsledek
1 Začne vyhodnocovat pravou stranu: addOne(10) přepne se do funkce
2 Nastaví parametr n = 10 uvnitř funkce je n připravené
3 Provede výpočet n + 1 vznikne 11
4 Narazí na return 11 a ukončí funkci funkce končí
5 Vrácená hodnota se „objeví“ v místě volání jako kdyby bylo let r = 11;
Mentální trik: Volání funkce si představ jako „díru“, do které se po doběhnutí doplní vrácená hodnota.
Nejčastější zmatky u return
1) „Nic se mi nevypsalo, tak to nefunguje“

Funkce může vracet hodnotu a nic nevypisovat. Když ji chceš vidět, musíš ji vypsat ty: 

function give() { return 5; }
give();              // nic neuvidíš
console.log(give()); // teď uvidíš 5
2) „Proč mám undefined?“

Nejčastěji proto, že funkce nemá return. Nebo return běží jen v jedné větvi a v jiné ne: 

function f(x) {
  if (x > 0) return "ok";
  // tady return chybí → vrátí undefined
}
console.log(f(-1)); // undefined
3) „Proč se mi část kódu nespustí?“

Protože return ukončí funkci: 

function g() {
  return 1;
  console.log("tohle nikdy");
}
Hoisting detailně: deklarace vs výraz (function declaration vs expression)
1) Function declaration (typicky funguje volání „před“)
hello();

function hello() {
  console.log("Ahoj");
}
2) Function expression (závisí na proměnné)

Tady se nejdřív musí vytvořit proměnná, teprve pak v ní je funkce:

sayHi(); // ReferenceError

const sayHi = function () {
  console.log("Ahoj");
};
A proč var občas „nepadá“ hned?
sayHi(); // TypeError: sayHi is not a function

var sayHi = function () {
  console.log("Ahoj");
};
Nápověda: isEven(number)
  • Modulo % vrací zbytek po dělení.
  • Sudé číslo: zbytek po dělení 2 je 0.
  • number % 2 === 0 už je boolean (true/false).
return number % 2 === 0;
Callback detailně: proč se nepíše sayBye()

Rozdíl je mezi „předat funkci“ a „spustit funkci“: 

// Předávám funkci jako hodnotu (správně)
afterWork(sayBye);

// Spouštím funkci hned (to nechci)
afterWork(sayBye());

Když napíšeš sayBye(), tak ji spustíš okamžitě a do afterWork se předá její výsledek (většinou undefined). Callback potřebuješ předat bez závorek.

Co je admin?

Admin (administrátor) je uživatel s vyššími právy. V reálných systémech může upravovat obsah, nastavení, spravovat uživatele apod. Tady je to jen zjednodušená role pro trénink podmínek a callbacku.

Nápovědy: login(username, password, callback)
Praktický postup (po malých krocích)
  1. Nejprve ošetři prázdný username (if (!username)) a vrať se (return).
  2. Pak prázdný password.
  3. Pak špatné username.
  4. Pak špatné password.
  5. Když všechno sedí, zavolej callback().
Proč return? Jakmile je chyba, nemá smysl pokračovat dál. Return to utně hned a kód je čitelnější.
