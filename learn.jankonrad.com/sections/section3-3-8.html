3.3.8 Funkce v JavaScriptu – return, hoisting a callback funkce

V této kapitole si detailně vysvětlíme, jak funkce vrací hodnoty (return), proč lze některé funkce volat dřív než jsou v souboru napsané (hoisting), a jak fungují callbacky (funkce předávané jako argument).

Co se naučíš

Hodně začátečníků používá funkce hlavně na console.log(). To je dobré pro ladění, ale není to to hlavní. Funkce typicky něco spočítá / rozhodne / připraví a výsledek předá dál — právě k tomu slouží return. Když pochopíš rozdíl mezi „vypsat“ a „vrátit“, začne ti dávat smysl, proč jsou funkce tak zásadní.

  • Return: rozdíl mezi výpisem a vrácením hodnoty, použití výsledku dál v programu.
  • Tok programu: return ukončuje funkci (kód za return se neprovede).
  • Více returnů: jak funkce může vracet různé hodnoty podle podmínky.
  • Hoisting: jak JS „připravuje“ funkce před spuštěním a co to znamená v praxi.
  • Callback: funkce jako argument — „co se má udělat po dokončení“.
Tip: Pracuj ve VS Code vždy jako nový mini-projekt: index.html + app.js, otevři stránku a sleduj konzoli (Inspect → Console).

Teorie – proč nestačí console.log

console.log() je „výstupní okno“ pro programátora. Je super pro kontrolu, co se děje, ale nepředává hodnotu dál. Když uvnitř funkce něco jen vypíšeš, program si tu hodnotu sám od sebe nikam neuloží.

Return funguje opačně: místo „ukázání“ výsledku ho pošle zpět na místo, kde byla funkce zavolána. Díky tomu můžeš výsledek:

  • uložit do proměnné a použít později
  • poslat jako argument do jiné funkce
  • použít přímo ve výrazu (např. v podmínce)

Varianta A: jen vypisuju

Funkce něco udělá a vypíše. Výsledek ale „nevrátí“, takže s ním dál nepracuješ. 

function multiply(a, b) {
  console.log(a * b);
}

multiply(5, 3);

Varianta B: vracím hodnotu

Funkce spočítá hodnotu a vrátí ji volajícímu místu. Teprve tam se rozhoduje, co s ní udělat. 

function multiply(a, b) {
  let result = a * b;
  return result;
}

let result = multiply(5, 3);
console.log(result);

Return se dá použít i „přímo“

Pokud funkce vrací hodnotu, můžeš ji rovnou použít třeba v console.log bez proměnné: 

console.log(multiply(5, 3));

Co když funkce nic nevrací?

Pokud funkce nemá return, vrací se automaticky undefined. 

function doNothing() {
  // nic nevrací
}

console.log(doNothing()); // undefined

Return ukončuje funkci

Jakmile se provede return, funkce končí. Jakýkoli kód pod return se ignoruje. 

function demo() {
  return "Hotovo";
  console.log("Toto se nikdy nespustí");
}

Mini cvičení: isEven(number)

Napiš funkci, která vrací boolean. Nejde o to „vypsat správně/špatně“ – cílem je vrátit true nebo false a pak teprve rozhodnout, co s tím výsledkem uděláš.

Start (index.html + app.js)

<!-- index.html -->
<!DOCTYPE html>
<html lang="cs">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>3.3.8</title>
</head>
<body></body>
<script src="app.js"></script>
</html>

Úkol

  1. Napiš funkci isEven(number).
  2. Využij: „sudé číslo“ znamená zbytek po dělení 2 je 0 ⇒ number % 2 === 0.
  3. Otestuj: console.log(isEven(20)) a console.log(isEven(7)).
Řešení (referenční) 
function isEven(number) {
  return number % 2 === 0;
}

console.log(isEven(20)); // true
console.log(isEven(7));  // false

Teorie – „předskenování“ kódu

Hoisting je zjednodušený popis toho, že JavaScript si před spuštěním kódu v daném scope udělá „přípravu“. U deklarací funkcí (klasické function name() {}) si funkce „zaregistruje“ a díky tomu ji jde často volat i dříve, než se v souboru objeví.

Deklarace funkce (hoisted)

doSomething();

function doSomething() {
  console.log("Funguje to.");
}

Proměnné let/const (nejsou použitelné před deklarací)

console.log(a); // ReferenceError
let a = 10;

Důvod: proměnná existuje až ve chvíli, kdy ji deklaruješ (prakticky: je „zamčená“ před použitím).

Poznámka k var

var se chová odlišně: „zvedne“ se deklarace, ale přiřazení hodnoty ne. Proto se objeví undefined. 

console.log(b); // undefined
var b = 10;
Doporučení: V moderním JS používej let / const. Je to predikovatelnější.

Teorie – funkce je hodnota (stejně jako číslo nebo text)

V JavaScriptu můžeš funkci uložit do proměnné, poslat ji jako argument nebo ji vrátit z jiné funkce. Callback je konkrétní situace: funkci předáš jiné funkci a ta ji zavolá ve správný moment.

Příklad: odpočítávání a start

function startRace() {
  console.log("Závod začíná!");
}

function countdown(callback) {
  for (let i = 3; i >= 1; i--) {
    console.log(i);
  }

  callback(); // až je hotovo, zavoláme callback
}

countdown(startRace);

Zásadní detail: countdown(startRace) předává funkci jako hodnotu. Nevolá ji.

Callbacky uvidíš často u událostí (kliknutí) a u operací, které skončí „později“ (např. načítání dat).

Zadání

Naprogramuj funkci login(username, password, callback). Funkce simuluje přihlášení. Cílem není bezpečnost, ale trénink: podmínky, return a callback.

Pravidla

  • Správný username je "admin".
  • Správné password je "1234".
  • Při úspěchu se zavolá callback().
  • Při chybě vypiš jasnou hlášku a funkci ukonči return.

Co přesně má funkce kontrolovat

  1. Chybí username → vypiš „Chyba: chybí uživatelské jméno.“ a ukonči funkci.
  2. Chybí password → vypiš „Chyba: chybí heslo.“ a ukonči funkci.
  3. Špatné username → vypiš „Chyba: neplatné uživatelské jméno.“ a ukonči funkci.
  4. Špatné password → vypiš „Chyba: nesprávné heslo.“ a ukonči funkci.
  5. Vše správně → zavolej callback.

Testy, které musíš spustit

login("admin", "1234", loginAdmin);
login("", "1234", loginAdmin);
login("admin", "", loginAdmin);
login("user", "1234", loginAdmin);
login("admin", "password123", loginAdmin);

Řešení (referenční)

Všimni si vzoru: kontrola → výpis chyby → return. Díky tomu se funkce „nezanořuje“ do hlubokých if/else.

Zobrazit řešení 
function login(username, password, callback) {
  if (!username) {
    console.log("Chyba: chybí uživatelské jméno.");
    return;
  }

  if (!password) {
    console.log("Chyba: chybí heslo.");
    return;
  }

  if (username !== "admin") {
    console.log("Chyba: neplatné uživatelské jméno.");
    return;
  }

  if (password !== "1234") {
    console.log("Chyba: nesprávné heslo.");
    return;
  }

  callback();
}

function loginAdmin() {
  console.log("Přihlašuji do administrátorského účtu.");
}

login("admin", "1234", loginAdmin);
login("", "1234", loginAdmin);
login("admin", "", loginAdmin);
login("user", "1234", loginAdmin);
login("admin", "password123", loginAdmin);

Co jsme se naučili

  • console.log je pro kontrolu programátora, ale hodnotu „nepředává dál“.
  • return vrací hodnotu na místo volání funkce (lze uložit do proměnné nebo použít ve výrazu).
  • return ukončuje funkci – kód pod return se neprovede.
  • Hoisting vysvětluje, proč lze některé funkce volat dřív než jsou zapsané.
  • Callback je funkce předaná jako argument – volající funkce rozhodne, kdy ji spustí.
Tip: Pokud řešíš chyby a validace, často je nejčitelnější „kontrola → return“ (early return).
Co se děje při volání funkce a returnu (krok za krokem)

Představ si volání: 

let r = multiply(5, 3);
Jak to probíhá:
  1. Program narazí na volání multiply(5, 3).
  2. Vytvoří se „vnitřní prostor“ funkce (lokální scope) a parametry dostanou hodnoty: a = 5, b = 3.
  3. Uvnitř funkce se spočítá a * b.
  4. Na return se funkce okamžitě ukončí a vrácená hodnota se pošle zpět na místo volání.
  5. To místo volání pokračuje dál — jako kdyby tam místo volání byla už přímo ta hodnota.

Mentální trik: Představ si, že JS přepíše volání funkcí na jejich vrácenou hodnotu.

let r = multiply(5, 3);

// „jako kdyby“ po doběhnutí funkce bylo:
let r = 15;

Proto jde vrácenou hodnotu uložit do proměnné, použít v podmínce, poslat dál do další funkce atd.

Return: co může vracet + proč je early return čitelnější
Co může return vracet
  • Číslo: return 42;
  • Text: return "Ahoj";
  • Boolean: return true;
  • null / undefined
  • Později: pole, objekty, funkce…
Return ukončuje funkci
function f() {
  console.log("A");
  return "X";
  console.log("B"); // nikdy
}
Early return (rychlé ukončení)

Když máš více chybových stavů, je často čitelnější ukončovat funkci hned při chybě, než budovat hluboké vnoření. 

function validate(age) {
  if (age === undefined) return false;
  if (age < 0) return false;
  if (age >= 18) return true;
  return false;
}
Hoisting detailně: declaration vs expression

Důležité je rozlišit dva typy „vytváření funkcí“:

1) Function declaration (typicky hoisted)
hello();

function hello() {
  console.log("Ahoj");
}
2) Function expression (závisí na proměnné)

Tady už nejde o deklaraci funkce „jako takovou“ nahoře, ale o proměnnou, do které ukládáš funkci. A proměnné s let/const nejdou použít před deklarací: 

sayHi(); // ReferenceError

const sayHi = function () {
  console.log("Ahoj");
};
A co var?

U var může být proměnná dostupná jako undefined (deklarace je „zvednutá“), ale funkce tam ještě není přiřazená: 

sayHi(); // TypeError: sayHi is not a function

var sayHi = function () {
  console.log("Ahoj");
};
Nápověda: isEven(number)
  • Modulo % vrací zbytek po dělení.
  • Sudé číslo: number % 2 je 0.
  • Výraz number % 2 === 0 už sám o sobě vrací boolean.
// nejjednodušší forma:
return number % 2 === 0;
Callback: rychlé připomenutí

Callback je funkce předaná jako argument. Nevoláš ji hned, jen ji předáš. 

// předávám funkci (nevolám ji)
countdown(startRace);

// volám funkci hned (to nechci)
countdown(startRace());

V praxi se callbacky používají u událostí (klik) a u operací, které se dokončí později.

Callback detailně: co je „předání funkce“

Klíčová myšlenka: v JS je funkce hodnota. Stejně jako číslo můžeš poslat do funkce i jinou funkci.

Mapování parametr → argument
function countdown(callback) {
  // callback je parametr
  callback();
}

function startRace() {
  console.log("Start");
}

// startRace je argument
countdown(startRace);

V momentě, kdy zavoláš countdown(startRace), se do parametru callback uloží odkaz na funkci startRace. Když pak uvnitř uděláš callback(), ve skutečnosti tím spouštíš startRace().

Proč to dává smysl
  • Funkce countdown neví dopředu, co má spustit po dokončení.
  • Volající rozhodne, co se má stát: jednou start závodu, podruhé třeba start animace.
  • Tohle je základ „znovupoužitelného“ kódu.
Nápovědy: login(username, password, callback)
Jak si to rozdělit na malé kroky
  1. Začni prázdnou funkcí login a ověř, že jde volat.
  2. Přidej kontrolu prázdného username a hned return.
  3. Přidej kontrolu prázdného password a hned return.
  4. Přidej kontrolu username !== "admin" a return.
  5. Přidej kontrolu password !== "1234" a return.
  6. Na konci zavolej callback().
Proč vracet „brzy“

Jakmile najdeš chybu, nemá smysl pokračovat dál. Return ti ušetří složité zanořování if/else.

Early return vs zanořené ify

Zanořené ify (často hůř čitelné):

function login(username, password) {
  if (username) {
    if (password) {
      if (username === "admin") {
        if (password === "1234") {
          console.log("OK");
        } else {
          console.log("Špatné heslo");
        }
      } else {
        console.log("Špatné jméno");
      }
    } else {
      console.log("Chybí heslo");
    }
  } else {
    console.log("Chybí jméno");
  }
}

Early return (často přímější):

function login(username, password) {
  if (!username) return console.log("Chybí jméno");
  if (!password) return console.log("Chybí heslo");
  if (username !== "admin") return console.log("Špatné jméno");
  if (password !== "1234") return console.log("Špatné heslo");
  console.log("OK");
}
