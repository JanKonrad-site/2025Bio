X.X.X Rychlé opakování: HTML, JavaScript a stránka pro výpis filmů

Než se pustíme do backendu a serveru, krátce si zopakujeme, co dělá webová stránka na straně uživatele. Nepůjde o celý kurz HTML a JavaScriptu — jen o to, co opravdu potřebujeme k pochopení našeho jednoduchého informačního systému.

Co se naučíš?

  • HTML kostra stránky – pochopíš, co na stránce znamenají tagy jako table, thead, tbody, tr a td.
  • JavaScript na stránce – zjistíš, jak skript najde prvky v HTML a jak s nimi pracuje.
  • Načítání dat ze serveru – uvidíš, proč stránka sama o sobě nezná data a musí si o ně říct backendu.
  • Praktická orientace – připravíš si půdu pro další kapitoly, kde budeme po malých blocích stavět backend v Pythonu.
Tip: Cílem této lekce není umět HTML a JavaScript dokonale. Cílem je nebát se souboru index.html a rozumět tomu, co se v něm děje.

1) Proč nezačínáme rovnou backendem

Backend a server budou pro žáky nová témata. Jenže náš výsledný projekt nebude tvořit jen Pythonový server. Součástí systému je i webová stránka, která data zobrazí uživateli. Kdybychom přeskočili HTML a JavaScript úplně, žáci by sice možná dokázali opsat hotový kód, ale nerozuměli by tomu, proč se po načtení stránky objeví tabulka, odkud se berou data a proč se bez spuštěného serveru nic nenačte.

  1. Na stránce je připravené místo pro data – například tabulka filmů.
  2. JavaScript po otevření stránky pošle požadavek na server.
  3. Server sáhne do databáze a vrátí data.
  4. JavaScript přijatá data vloží do tabulky.
  • Frontend = to, co vidí uživatel v prohlížeči.
  • Backend = to, co běží na serveru a zpracovává data.
  • Databáze = místo, kde jsou data uložená.
2) Rychlá mapa našeho systému

Náš jednoduchý informační systém má tři části. Je dobré si je hned na začátku pojmenovat, protože se k nim budeme vracet v každé další kapitole.

2.1 Frontend – stránka v prohlížeči

Frontend je soubor index.html. Obsahuje tabulku a JavaScript. Tabulka je místo, kam se budou vypisovat filmy. JavaScript se postará o to, aby se data načetla ze serveru a zobrazila.

  • Uživatel otevře stránku.
  • Stránka se vykreslí v prohlížeči.
  • Skript začne pracovat až po načtení stránky.
2.2 Backend – Pythonový server

Backend je soubor movies_server.py. Ten spustí server, obslouží požadavek na cestě /api/movies a vrátí seznam filmů ve formátu JSON.

  • Server čeká na požadavky.
  • Když přijde dotaz, rozhodne se podle adresy, co vrátí.
  • Jednou může vrátit HTML stránku, jindy data z databáze.
2.3 Databáze – uložená data

Databáze movies.db obsahuje tabulku filmů. Backend se na ni umí připojit, přečíst z ní data a poslat je dál do stránky.

  • Databáze si pamatuje data i po vypnutí programu.
  • Frontend do databáze přímo nevidí.
  • Mezi stránkou a databází stojí server.
3) HTML – co už na stránce máme připravené

HTML je kostra stránky. Určuje, co na stránce bude: nadpisy, odstavce, tabulky, tlačítka nebo obrázky. V našem případě je nejdůležitější tabulka, do které budeme vypisovat filmy, a krátká textová hláška o stavu načítání.

Pro naši ukázku je dobré mít v HTML tuto jednoduchou část: 

<p id="status">Načítám data...</p>

<table id="moviesTable">
  <thead>
    <tr>
      <th>ID</th>
      <th>Název</th>
      <th>Rok</th>
    </tr>
  </thead>
  <tbody></tbody>
</table>
  1. <p id="status"> je odstavec, do kterého budeme zapisovat zprávy jako „Načítám data…“ nebo „Načteno filmů: 5“.
  2. <table> vytváří tabulku.
  3. <thead> je hlavička tabulky – tedy názvy sloupců.
  4. <tr> znamená jeden řádek tabulky.
  5. <th> je buňka hlavičky tabulky.
  6. <tbody> je tělo tabulky – sem budeme JavaScriptem doplňovat skutečná data.
  • HTML ještě samo nic nenačítá. Jen připraví místo, kam data později přijdou.
  • Prázdný <tbody> je v pořádku – při načtení stránky tam zatím žádná data nejsou.
  • Prvek status je důležitý, protože ho později najdeme pomocí JavaScriptu.
4) JavaScript – jak stránka najde tabulku a načte data

JavaScript je jazyk, který stránku „oživí“. Umí najít prvky v HTML, vytvořit nové řádky tabulky, poslat dotaz na server a výsledek zobrazit.

Na úplný začátek nám stačí pochopit tento malý blok: 

const status = document.getElementById("status");
const tbody = document.querySelector("#moviesTable tbody");

const res = await fetch("/api/movies");
const movies = await res.json();
  1. const vytváří proměnnou, jejíž odkaz nechceme později měnit.
  2. document znamená celou právě otevřenou webovou stránku.
  3. getElementById("status") najde prvek, který má v HTML atribut id="status".
  4. querySelector("#moviesTable tbody") najde v tabulce její část tbody.
  5. fetch("/api/movies") pošle požadavek na server.
  6. await znamená „počkej, než odpověď opravdu dorazí“.
  7. res.json() převede odpověď serveru z JSONu na data, se kterými už JavaScript umí pracovat.
  • Stránka si data nevymýšlí sama — musí si o ně požádat backend.
  • Adresa /api/movies je cesta na serveru, která vrací seznam filmů.
  • Bez spuštěného serveru nebude mít fetch() odkud data získat.
5) Co je JSON a proč se nám hodí

Když server pošle data do stránky, potřebuje je zapsat v nějakém formátu, kterému rozumí obě strany. Pro tento účel se velmi často používá JSON. 

[
  { "id": 1, "title": "Titanic", "year": 1997 },
  { "id": 2, "title": "Avatar", "year": 2009 }
]
  1. Hranaté závorky [ ] znamenají seznam více položek.
  2. Složené závorky { } znamenají jeden objekt – tedy jednu položku s vlastnostmi.
  3. "id", "title" a "year" jsou názvy vlastností.
  4. Za dvojtečkou je jejich hodnota.
  • Každý film je v JSONu jeden objekt.
  • Celý seznam filmů je pole objektů.
  • JavaScript si taková data umí snadno převzít a projít je jedno po druhém.
6) Co si z této lekce odnést do dalších kapitol

Tato kapitola je hlavně orientační. Nemá z vás udělat frontend vývojáře. Má vám dát jistotu, že když v dalších lekcích uvidíte soubor index.html, tabulku, JavaScript a volání serveru, nebudete mít pocit, že se díváte na něco úplně cizího.

  1. HTML připraví kostru stránky.
  2. JavaScript si najde prvky v HTML.
  3. JavaScript požádá server o data.
  4. Server vrátí data z databáze.
  5. JavaScript je vloží do tabulky.
Pamatuj: frontend zobrazuje, backend obsluhuje a databáze ukládá.

Rychlé zopakování HTML tagů

HTML používá značky, kterým říkáme tagy. Ty určují, co se na stránce zobrazí a jaký to má význam.

  • <p> – odstavec textu.
  • <table> – tabulka.
  • <thead> – hlavička tabulky.
  • <tbody> – tělo tabulky, kam vkládáme data.
  • <tr> – jeden řádek tabulky.
  • <th> – hlavičková buňka.
  • <td> – obyčejná datová buňka.

Důležité je, že HTML ještě samo nic nepočítá a nic nestahuje. Jen připraví kostru stránky.

Rychlé zopakování pojmů z JavaScriptu

V této lekci stačí rozumět několika pojmům:

  • document – právě otevřená webová stránka.
  • getElementById() – najde prvek podle jeho id.
  • querySelector() – najde první prvek, který odpovídá zadanému selektoru.
  • fetch() – pošle požadavek na server.
  • await – počká na dokončení asynchronní operace.
  • json() – převede JSON odpověď na data pro JavaScript.

Nemusíš si vše pamatovat nazpaměť. Důležité je vědět, co ten příkaz přibližně dělá a proč tam je.
