6.1.2 Backend, server a API – proč je potřebujeme v našem rezervačním systému

V minulé části jsme si ukázali, že Python umí zapisovat rezervace do SQLite databáze. Teď si vysvětlíme, co je backend, co znamená server, co je API a proč bez backendu webová stránka neumí sama pracovat s databází. V další lekci už na to navážeme prakticky a skutečně server postavíme.

Co se v této lekci naučíš?

  • Co je frontend, backend, server a databáze.
  • Co je API a proč ho v našem projektu potřebujeme.
  • Proč je pro nás FastAPI dobrá volba na první backend.
  • Jak vypadá nejmenší možný server ve FastAPI.
  • Co znamená zápis @app.get("/").
  • Jak bude propojená konzole, databáze a webová stránka v našem rezervačním systému.
Cíl celé kapitoly: Nechceme jen opisovat kód. Chceme přesně pochopit, proč backend vzniká, kde v projektu sedí a jakou práci dělá mezi webem a databází.

Jak to navazuje na náš projekt

  1. Máme Python skript, který v konzoli ukládá rezervace do reservations.db.
  2. Chceme, aby stejná data šla zobrazit také na webové stránce.
  3. Webová stránka ale neumí sama otevřít SQLite databázi na disku.
  4. Proto mezi stránku a databázi vložíme backend.
  5. Backend poběží jako server a stránce bude data poskytovat přes API.
Jedna věta, kterou si zapamatuj:
Databáze data ukládá, backend s nimi pracuje a webová stránka je zobrazuje.
Krok 1: kde v projektu sedí frontend, backend, server a databáze

Nejdřív si musíme ujasnit, že webová aplikace se obvykle neskládá jen z jedné věci. Má několik částí a každá dělá něco jiného.

  • Frontend = to, co vidí uživatel v prohlížeči (HTML, CSS, JavaScript).
  • Backend = Python program, který běží na serveru a řeší logiku aplikace.
  • Databáze = místo, kde se ukládají data.
  • Server = běžící program, který čeká na požadavky a vrací odpovědi.
V našem projektu:
Frontend = stránka s výpisem rezervací
Backend = Python + FastAPI
Databáze = reservations.db
Vysvětlení úplně lidsky
  • Frontend je „obličej“ aplikace – to, na co klikáš a co vidíš.
  • Backend je „mozek“ aplikace – rozhoduje, co se má stát.
  • Databáze je „paměť“ aplikace – uchovává informace.
  • Server je způsob, jak backend běží a komunikuje s okolím.
Krok 2: co je API a proč ho vůbec potřebujeme

API je způsob, jak spolu komunikují dva programy. V našem případě to znamená, že webová stránka pošle požadavek backendu a backend vrátí odpověď.

  • Stránka si řekne: „Pošli mi seznam rezervací.“
  • Backend otevře databázi a data načte.
  • Backend vrátí odpověď zpět stránce.
  • Stránka data zobrazí uživateli.
Důležité: Browser obvykle nepracuje přímo se SQLite souborem na disku. Právě proto potřebujeme backend, který bude mezi stránkou a databází.
Proč nepřipojíme stránku rovnou na databázi?
  • Databáze je soubor na disku serveru nebo tvého počítače.
  • Browser neumí bezpečně a přímo pracovat s Python modulem sqlite3.
  • Kdyby to šlo přímo, bylo by to nebezpečné a špatně řiditelné.
  • Proto databázi obsluhuje backend a browser komunikuje s backendem přes API.
Krok 3: proč budeme používat zrovna FastAPI

Na backend v Pythonu existuje více frameworků. My zvolíme FastAPI, protože je přehledné, krátké a na výuku backendu velmi vhodné.

  • Kód je krátký a dobře čitelný.
  • Rychle vytvoříš první funkční server.
  • Snadno se v něm vysvětlují cesty jako / nebo /reservations.
  • Dobře se propojuje s Pythonem a naší SQLite databází.
  • Hodí se pro malé i větší projekty.
Pro naši výuku je nejdůležitější:
Ve FastAPI velmi rychle uvidíš vztah:
URL adresa → Python funkce → odpověď serveru
Krok 4: nejmenší možný backend ve FastAPI
main.py 
from fastapi import FastAPI

app = FastAPI()

@app.get("/")
def read_root():
    return {"message": "Ahoj ze serveru"}
Vysvětlení kódu „slovo od slova“
  • from fastapi import FastAPI = importujeme třídu FastAPI.
  • app = FastAPI() = vytvoříme objekt aplikace, tedy náš backend.
  • @app.get("/") = říkáme: když přijde požadavek na adresu /, spusť funkci pod tímto řádkem.
  • def read_root(): = obyčejná Python funkce, kterou FastAPI použije pro obsluhu této adresy.
  • return {"message": "Ahoj ze serveru"} = server vrátí odpověď ve formě dat.
Co si z toho odnést
  • Každá adresa na serveru může mít vlastní Python funkci.
  • Backend tedy není nic jiného než Python kód, který reaguje na URL adresy.
  • Teď ještě nečteme databázi. Jen vracíme pevně napsanou odpověď.
Krok 5: jak server spustíme
terminál 
uvicorn main:app --reload
Vysvětlení příkazu „slovo od slova“
  • uvicorn = program, který umí spustit Python web server.
  • main:app = vezmi soubor main.py a v něm hledej proměnnou app.
  • --reload = když uložíš změny v souboru, server se během vývoje automaticky restartuje.
Po spuštění:
Server obvykle poběží na adrese http://127.0.0.1:8000
a tam uvidíš odpověď ze své FastAPI aplikace.
Krok 6: jak to celé zapadne do našeho rezervačního systému
struktura projektu (plán) 
rezervacni_system/
├── reservations_console.py
├── main.py
├── reservations.db
└── templates/
    └── index.html
Co bude dělat každý soubor
  • reservations_console.py = konzolový program, který zapisuje rezervace do databáze.
  • main.py = backend ve FastAPI, který databázi čte a vrací data do prohlížeče.
  • reservations.db = SQLite databáze, kde budou rezervace uložené.
  • templates/index.html = HTML šablona stránky, která zobrazí rezervace.
To nejdůležitější:
Konzolová aplikace i backend budou pracovat se stejnou databází. To znamená, že co zapíšeš v konzoli, backend potom může zobrazit na stránce.
Tok dat v našem projektu
  1. V konzoli zadáš rezervaci.
  2. Python ji uloží do reservations.db.
  3. FastAPI server otevře stejnou databázi.
  4. Načte rezervace a pošle je do stránky.
  5. Webová stránka je zobrazí uživateli.

Mini kontrola: co už teď musíš chápat

  • Backend je Python část aplikace, která běží na serveru.
  • API je způsob, jak si webová stránka říká backendu o data.
  • FastAPI použijeme proto, že se v něm backend dobře vysvětluje a rychle staví.
  • SQLite je souborová databáze, se kterou bude backend pracovat.
  • Webová stránka nebude sahat na databázi přímo – bude komunikovat s backendem.
  • V další lekci už vytvoříme skutečný server, který naše rezervace zobrazí na stránce.

Frontend × Backend × Databáze

  • Frontend = to, co vidí uživatel v prohlížeči.
  • Backend = program na serveru, který řeší logiku aplikace.
  • Databáze = místo, kde se ukládají data.

V našem projektu bude frontend zobrazovat rezervace, backend je načte a databáze je bude uchovávat.

Co je server?

Server je běžící program, který čeká na požadavky a vrací odpovědi.

  • Někdo otevře webovou adresu.
  • Server požadavek přijme.
  • Backend rozhodne, co vrátit.
  • Server pošle odpověď zpět.

V našem případě bude server spuštěný Python program s FastAPI.

Co je API?

API je domluvený způsob komunikace mezi programy.

  • Frontend pošle požadavek.
  • Backend ho zpracuje.
  • Backend vrátí odpověď.

API si můžeš představit jako „okénko“, přes které si jedna část systému říká druhé o data nebo akci.

Proč stránka neotevře SQLite databázi sama?

  • SQLite je databázový soubor na disku.
  • Browser neumí používat Python modul sqlite3.
  • Práce s databází musí být řízená backendem.
  • Backend pak data předá stránce přes API.
Zjednodušeně:
stránka si o data řekne, backend je vyzvedne z databáze, stránka je pak jen zobrazí.

GET a POST – dvě základní HTTP metody

  • GET = chci data přečíst / získat.
  • POST = chci na server něco poslat / vytvořit.

V našem projektu bude výpis rezervací typicky GET. Přidání nové rezervace přes web by později bylo typicky POST.

Proč budeme používat FastAPI?

  • Je přehledné a má krátký zápis.
  • Dobře se hodí pro výuku backendu.
  • Rychle v něm postavíš první funkční server.
  • Dobře se propojuje s Pythonem a databází.

Pro nás je nejdůležitější, že student jasně uvidí vztah:
adresa → Python funkce → odpověď

Co je route / endpoint?

Route nebo endpoint je konkrétní adresa na serveru, kterou backend obsluhuje.

  • / = domovská adresa
  • /reservations = třeba adresa pro výpis rezervací

Zápis @app.get("/") znamená: „Když někdo otevře adresu /, spusť tuto funkci.“

Co je JSON?

JSON je běžný textový formát pro předávání dat mezi frontendem a backendem. 

{
  "message": "Ahoj ze serveru"
}

Vypadá podobně jako Python slovník, ale používá se jako standardní datový formát na webu.

Co je Uvicorn?

Uvicorn je program, který umí spustit Python webovou aplikaci jako server.

  • Bez něj by soubor main.py nebyl webovým serverem.
  • Uvicorn spustí naši FastAPI aplikaci a otevře ji na webové adrese.

Co znamená 127.0.0.1 nebo localhost?

  • 127.0.0.1 = adresa tvého vlastního počítače.
  • localhost = lidsky čitelný název pro totéž.
  • Když spustíš server lokálně, běží zatím jen u tebe.

Proto adresa jako http://127.0.0.1:8000 znamená: „otevři webový server, který běží přímo na mém počítači.“

Proč oba programy vidí stejná data?

Protože oba programy pracují se stejným souborem reservations.db.

  • Konzolový program data do databáze zapisuje.
  • FastAPI backend stejnou databázi čte.
  • Co jeden program uloží, druhý program může zobrazit.
