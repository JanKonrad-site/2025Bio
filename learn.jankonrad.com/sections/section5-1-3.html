3.2.1 SQLite v Pythonu – otevření databáze a první data (krok za krokem)

Navazujeme na 3.2.0 (cesta k souboru databáze). Teď poprvé databázi v Pythonu otevřeme, vytvoříme tabulku, uložíme jeden záznam a průběžně ověříme, co se skutečně stalo (soubor, tabulka, počet záznamů).

Co se v této lekci naučíš?

  • Otevření databáze – co dělá sqlite3.connect() a co je conn.
  • Vytvoření tabulky – CREATE TABLE IF NOT EXISTS a jak ověřit, že tabulka existuje.
  • Uložení dat – INSERT s parametry ?.
  • Ověření výsledku – SELECT COUNT(*) a čtení výsledku přes fetchone().
  • commit() a close() – proč se změny potvrzují a proč spojení zavíráme.
Doporučení: Spouštěj skript z terminálu ve složce se souborem (kvůli ověřování souboru movies.db).
Primární akce (např. startovací soubor / zadání) Python sqlite3 (dokumentace)
0) Start: vytvoř soubor a připrav proměnnou DB_FILE

Vytvoř soubor db_step_open_create_insert.py. Na začátek vlož stejný základ jako v 3.2.0: importy a cestu k databázi. 

import sqlite3
from pathlib import Path

DB_FILE = Path(__file__).resolve().with_name("movies.db")

Tady se děje víc věcí najednou: Path(__file__) vytvoří objekt cesty, .resolve() zjistí skutečnou absolutní cestu k aktuálnímu souboru a .with_name("movies.db") vezme stejnou složku, ale změní název souboru na movies.db.

V této lekci budeme ověřovat, co se děje. Proto budeme občas vypisovat: DB_FILE a DB_FILE.exists().

1) Krok 1: otevření databáze (co dělá connect)

SQLite databáze je soubor. Připojení se otevře pomocí sqlite3.connect(...). Výsledkem je objekt conn (connection) – přes něj posíláme SQL příkazy. 

def open_and_close_db():
    print("DB path:", DB_FILE)
    print("DB exists before connect:", DB_FILE.exists())

    conn = sqlite3.connect(str(DB_FILE))
    conn.close()

    print("DB exists after close:", DB_FILE.exists())

Proč je tam str(DB_FILE)? Protože DB_FILE je objekt typu Path a zde ho převádíme na běžný textový řetězec.

Přidej spouštěcí blok, aby se funkce opravdu vykonala: 

if __name__ == "__main__":
    open_and_close_db()
    input("ENTER pro ukončení...")

Tento zápis říká: „Následující kód spusť jen tehdy, když tento soubor běží jako hlavní program.“

Spusť a ověř:

  • Před připojením bývá False.
  • Po zavření připojení se typicky objeví soubor a exists() bude True.
Co jsme ověřili? Umíme databázi „otevřít“ a víme, kam se ukládá soubor.
2) Krok 2: vytvoření tabulky (CREATE TABLE) a význam jednotlivých částí

Tabulka je struktura pro ukládání záznamů. V SQLite ji vytvoříme SQL příkazem CREATE TABLE. Tento příkaz posíláme přes conn.execute(...). 

def create_table_movies():
    conn = sqlite3.connect(str(DB_FILE))

    conn.execute("""
    CREATE TABLE IF NOT EXISTS movies (
        id    INTEGER PRIMARY KEY AUTOINCREMENT,
        title TEXT    NOT NULL,
        year  INTEGER
    );
    """)

    conn.commit()
    conn.close()

Co znamenají části SQL:

  • CREATE TABLE – vytvoří tabulku.
  • IF NOT EXISTS – druhé spuštění nespadne (tabulka už existuje).
  • movies – název tabulky.
  • id INTEGER PRIMARY KEY AUTOINCREMENT – automatické číselné ID (unikátní).
  • title TEXT NOT NULL – text, který nesmí být prázdný (NULL).
  • year INTEGER – rok (může zůstat nevyplněný = NULL).

Po změně struktury databáze voláme commit(), aby se změna opravdu zapsala do souboru. Potom dává smysl spojení zavřít přes close().

Spusť a ověř: uprav spouštěcí blok takto:

if __name__ == "__main__":
    create_table_movies()
    print("DB exists:", DB_FILE.exists())
    input("ENTER pro ukončení...")
Proč commit()? Potvrzuje změny (např. vytvoření tabulky) do databázového souboru.
3) Krok 3: ověření tabulky (čtení metadat SQLite)

Abychom neověřovali jen existenci souboru, zjistíme, jaké tabulky databáze skutečně obsahuje. SQLite to eviduje v systémové tabulce sqlite_master. 

def list_tables():
    conn = sqlite3.connect(str(DB_FILE))
    rows = conn.execute("SELECT name FROM sqlite_master WHERE type='table';").fetchall()
    conn.close()
    return [r[0] for r in rows]

Metoda fetchall() vrátí všechny nalezené řádky najednou. Každý řádek je zde n-tice, takže r[0] znamená „vezmi první hodnotu z každého řádku“.

Spouštěcí blok:

if __name__ == "__main__":
    create_table_movies()
    print("Tables:", list_tables())
    input("ENTER pro ukončení...")

Spusť a ověř: ve výpisu musí být movies.

4) Krok 4: uložíme film (INSERT) a ověříme to přes COUNT(*)

Teď uložíme jeden záznam. Pro ověření použijeme SELECT COUNT(*), což je nejjednodušší kontrola, že se počet řádků změnil. 

def insert_movie(title, year):
    conn = sqlite3.connect(str(DB_FILE))
    conn.execute(
        "INSERT INTO movies (title, year) VALUES (?, ?);",
        (title, year)
    )
    conn.commit()
    conn.close()

Znak ? je zástupné místo pro hodnotu. SQLite tam bezpečně dosadí data z n-tice (title, year). 

def count_movies():
    conn = sqlite3.connect(str(DB_FILE))
    n = conn.execute("SELECT COUNT(*) FROM movies;").fetchone()[0]
    conn.close()
    return n

Tady je dobré rozumět tomu, proč je na konci .fetchone()[0].

Spouštěcí blok (před/po):

if __name__ == "__main__":
    create_table_movies()

    before = count_movies()
    print("Count before:", before)

    title = input("Název filmu: ").strip()
    year_text = input("Rok (ENTER = nevyplnit): ").strip()
    year = int(year_text) if year_text else None

    insert_movie(title, year)

    after = count_movies()
    print("Count after:", after)

    input("ENTER pro ukončení...")

Výraz int(year_text) if year_text else None je zkrácený zápis podmínky: pokud uživatel něco zadal, převedeme to na číslo, jinak uložíme None.

Spusť a ověř: after musí být o 1 vyšší než before.

5) Proč máme if __name__ == "__main__": (a co přesně dělá)

Soubor může být použit dvěma způsoby: (1) spustím ho jako program, (2) naimportuji ho jako modul. Python podle toho nastaví proměnnou __name__.

  • Při přímém spuštění: __name__ == "__main__"
  • Při importu: __name__ je název souboru (modulu), např. "db_step_open_create_insert"

Blok if __name__ == "__main__": tedy říká: „spusť následující část jen tehdy, když tento soubor běží jako hlavní program“. Je to standardní způsob, jak mít v jednom souboru definice funkcí a zároveň testovací běh.

Volitelná kontrola:

print("__name__:", __name__)

Studentům se často plete rozdíl mezi = a ==. Tady je správně ==, protože porovnáváme hodnoty, nic nepřiřazujeme.

Praktický přínos: Později budeš chtít funkce importovat do jiné aplikace. Díky této podmínce se při importu nespustí testovací část.
6) Kompletní soubor po lekci 3.2.1 (pro kontrolu) 
import sqlite3
from pathlib import Path

DB_FILE = Path(__file__).resolve().with_name("movies.db")

def open_and_close_db():
    print("DB path:", DB_FILE)
    print("DB exists before connect:", DB_FILE.exists())
    conn = sqlite3.connect(str(DB_FILE))
    conn.close()
    print("DB exists after close:", DB_FILE.exists())

def create_table_movies():
    conn = sqlite3.connect(str(DB_FILE))
    conn.execute("""
    CREATE TABLE IF NOT EXISTS movies (
        id    INTEGER PRIMARY KEY AUTOINCREMENT,
        title TEXT    NOT NULL,
        year  INTEGER
    );
    """)
    conn.commit()
    conn.close()

def list_tables():
    conn = sqlite3.connect(str(DB_FILE))
    rows = conn.execute("SELECT name FROM sqlite_master WHERE type='table';").fetchall()
    conn.close()
    return [r[0] for r in rows]

def insert_movie(title, year):
    conn = sqlite3.connect(str(DB_FILE))
    conn.execute("INSERT INTO movies (title, year) VALUES (?, ?);", (title, year))
    conn.commit()
    conn.close()

def count_movies():
    conn = sqlite3.connect(str(DB_FILE))
    n = conn.execute("SELECT COUNT(*) FROM movies;").fetchone()[0]
    conn.close()
    return n

if __name__ == "__main__":
    # 1) Ověření cesty + existence
    open_and_close_db()

    # 2) Tabulka
    create_table_movies()
    print("Tables:", list_tables())

    # 3) Ověření dat
    before = count_movies()
    print("Count before:", before)

    title = input("Název filmu: ").strip()
    year_text = input("Rok (ENTER = nevyplnit): ").strip()
    year = int(year_text) if year_text else None

    insert_movie(title, year)

    after = count_movies()
    print("Count after:", after)

    input("ENTER pro ukončení...")

Cvičení (krátké)

  1. Spusť skript a vlož dva různé filmy. Po každém vložení si zapiš hodnoty Count before a Count after.
  2. Změň název tabulky z movies na games (v CREATE TABLE, INSERT i COUNT) a ověř, že vše funguje.
  3. Přidej do tabulky nový sloupec genre TEXT a zkontroluj, že tabulka se stále vytvoří (zatím ho nemusíš ukládat).

Shrnutí

  • Otevření databáze dělá sqlite3.connect() a vrací spojení conn.
  • Tabulku vytvoříš SQL příkazem CREATE TABLE přes conn.execute().
  • Změny se potvrdí pomocí commit() a spojení uzavřeš close().
  • Uložení záznamu ověříš dotazem SELECT COUNT(*).
  • if __name__ == "__main__": spouští testovací běh jen při přímém spuštění souboru.

Jak funguje Path(__file__).resolve().with_name("movies.db")?

__file__ je speciální proměnná s cestou k aktuálnímu Python souboru.

Path(__file__) z ní vytvoří objekt cesty.

.resolve() zjistí absolutní skutečnou cestu k souboru.

.with_name("movies.db") ponechá stejnou složku, ale změní název souboru na movies.db.

Prakticky: databáze bude uložena ve stejné složce jako tento Python soubor.

Co dělá DB_FILE.exists()?

Metoda exists() vrací True nebo False podle toho, zda soubor nebo složka opravdu existuje.

V našem případě ověřujeme, jestli už databázový soubor movies.db vznikl.

Proč používáme str(DB_FILE)?

DB_FILE je objekt typu Path, ne obyčejný text.

str(...) ho převede na řetězec, například:

WindowsPath('C:/projekt/movies.db')
# po převodu:
'C:/projekt/movies.db'

Tím pádem ho můžeme snadno předat funkci sqlite3.connect(...).

Co dělá sqlite3.connect() a co je conn?

sqlite3.connect(...) otevře připojení k databázi.

Vrací objekt spojení, který si často ukládáme do proměnné conn jako zkratku pro connection.

Přes conn pak provádíme SQL dotazy, potvrzujeme změny a zavíráme spojení.

conn = sqlite3.connect(str(DB_FILE))
conn.execute("SELECT 1;")
conn.close()

Co dělá conn.execute(...)?

Znamená to: „na tomto databázovém spojení proveď SQL příkaz“.

conn.execute("CREATE TABLE test (id INTEGER);")

Samotný SQL příkaz je textový řetězec. Metoda execute() ho pošle databázi ke zpracování.

Proč CREATE TABLE IF NOT EXISTS?

CREATE TABLE vytvoří novou tabulku.

IF NOT EXISTS přidává pojistku: když tabulka už existuje, skript nespadne chybou.

To je praktické při opakovaném spouštění stejného programu během vývoje.

Proč voláme commit() a close()?

commit() potvrdí změny v databázi. Bez něj se nová tabulka nebo vložený řádek nemusí uložit natrvalo.

close() ukončí spojení s databází. Je to dobrý zvyk – po práci spojení zavřít.

Jednoduché pravidlo:

  • měníš data nebo strukturu → obvykle commit()
  • končíš práci → close()

Co je sqlite_master?

Je to interní systémová tabulka SQLite, ve které databáze eviduje své objekty – například tabulky, indexy nebo pohledy.

Když se ptáme:

SELECT name FROM sqlite_master WHERE type='table';

získáváme seznam tabulek, které databáze skutečně obsahuje.

Proč používáme ? v INSERT?

Znak ? je zástupné místo pro hodnotu.

conn.execute(
    "INSERT INTO movies (title, year) VALUES (?, ?);",
    (title, year)
)

SQLite vezme hodnoty z n-tice (title, year) a bezpečně je dosadí na správná místa.

Je to lepší než ručně skládat SQL text, protože je to přehlednější a bezpečnější.

Jak funguje fetchone()[0]?

fetchone() vrátí jeden řádek výsledku dotazu.

U dotazu:

SELECT COUNT(*) FROM movies;

vrátí SQLite jeden řádek s jednou hodnotou, například:

(5,)

Proto píšeme [0] – vezmeme první hodnotu z n-tice. Výsledek pak bude prosté číslo 5.

Jak přesně funguje if __name__ == "__main__":?

__name__ je speciální proměnná, kterou Python nastavuje automaticky.

  • když soubor spustíš přímo → __name__ je "__main__"
  • když soubor jen importuješ → __name__ je název modulu

Podmínka tedy říká: spusť tento blok jen při přímém spuštění souboru.

if __name__ == "__main__":
    open_and_close_db()

Díky tomu můžeš mít v jednom souboru definice funkcí i testovací běh programu.
