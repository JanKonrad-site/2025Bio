3.6.0 Rezervační systém – konzole + SQLite (nejdřív jednoduše, potom jako aplikace)

Vytvoříš jednoduchý rezervační systém do konzole. Nejprve to uděláme bez funkcí (abychom se neztratili), a teprve potom to přepíšeme do „aplikace“ s menu. Data se ukládají do SQLite souboru reservations.db.

Co bude výstup na konci?

  • Databáze reservations.db existuje ve stejné složce jako skript.
  • Tabulka reservations se vytvoří (pokud ještě neexistuje).
  • Umíš vložit rezervaci: jméno, datum, příchod, odchod.
  • Umíš rezervace načíst a vypsat v konzoli.
  • (Bonus) Přepíšeš řešení do „aplikace“ s menu (funkce + while smyčka).
Formát vstupu: datum piš YYYY-MM-DD (např. 2026-04-10), čas piš HH:MM (např. 14:30). Zatím neřešíme validaci – nejdřív pochopíme tok programu.

0) Mini-hra: poskládej tok programu (a odhal výsledný kód)

Než začneš psát, zkus si „v hlavě“ poskládat správné pořadí kroků. Klikáním přidáváš bloky do „skládačky“. Když je pořadí správně, odhalí se celý kód.

Bloky

Tvoje skládání

    Skládej bloky. Až bude pořadí správně, objeví se kód.
    ✅ Odhalený kód (lineární verze)
    Smysl hry: Nejde o rychlost. Jde o to vědět, co musí přijít dřív (např. bez connect nejde execute).
    1) Nejjednodušší verze (bez funkcí): DB → tabulka → vlož → načti → vypiš

    Tohle je nejčitelnější varianta: jede se shora dolů. Neřešíme zatím menu ani funkce. Jen pochopíme tok: connect → execute → commit → SELECT → fetchall → for → print.

    reservations_linear.py 
    import sqlite3
from pathlib import Path

DB_FILE = Path(__file__).resolve().with_name("reservations.db")

if __name__ == "__main__":
    # 1) Kontrola: kam míří DB soubor
    print("DB path:", DB_FILE)
    print("DB exists BEFORE:", DB_FILE.exists())

    # 2) Připojení k DB (pokud soubor neexistuje, SQLite ho vytvoří)
    conn = sqlite3.connect(str(DB_FILE))

    # 3) Vytvoření tabulky (pokud ještě neexistuje)
    conn.execute("""
    CREATE TABLE IF NOT EXISTS reservations (
        id        INTEGER PRIMARY KEY AUTOINCREMENT,
        name      TEXT    NOT NULL,
        date      TEXT    NOT NULL,
        arrival   TEXT    NOT NULL,
        departure TEXT    NOT NULL
    );
    """)
    conn.commit()

    # 4) Vstup od uživatele
    name = input("Jméno: ").strip()
    date = input("Datum (YYYY-MM-DD): ").strip()
    arrival = input("Příchod (HH:MM): ").strip()
    departure = input("Odchod (HH:MM): ").strip()

    # 5) INSERT – uložení rezervace
    conn.execute(
        "INSERT INTO reservations (name, date, arrival, departure) VALUES (?, ?, ?, ?);",
        (name, date, arrival, departure)
    )
    conn.commit()

    # 6) SELECT – načtení všech rezervací
    rows = conn.execute("""
        SELECT id, name, date, arrival, departure
        FROM reservations
        ORDER BY date ASC, arrival ASC;
    """).fetchall()

    # 7) Zavřeme DB
    conn.close()

    # 8) Výpis výsledku
    print("\n--- Rezervace v databázi ---")
    for (rid, name, date, arrival, departure) in rows:
        print(f"#{rid} | {name} | {date} | {arrival}-{departure}")

    print("\nDB exists AFTER:", DB_FILE.exists())
    input("ENTER pro ukončení...")
    Ověření
    1. Spusť soubor.
    2. Vlož jednu rezervaci.
    3. Na konci se musí vypsat seznam rezervací (minimálně 1 řádek).
    2) Refaktor: stejná logika, ale jako „aplikace“ (funkce + menu)

    Teď už umíš tok programu. Teprve teď dává smysl rozdělit to do funkcí, aby se to dalo používat opakovaně. Princip se nemění – jen to uklidíme.

    Pravidlo: Funkce = balíček kroků. Uvnitř funkce pořád platí stejná sekvence: connect → execute → commit (když měním) → close.
