3.2.1 SQLite v Pythonu – první databáze „laboratorně“ (krok za krokem s ověřováním)

V této lekci budeme postupovat velmi pomalu. Vždy uděláš malou změnu v kódu, spustíš skript a ověříš výsledek. Cílem je, aby bylo jasné, co přesně dělá každý řádek: připojení, vytvoření tabulky, vložení dat a kontrolní dotazy.

Pravidla práce v této lekci

  • Po každém kroku skript spusť a zkontroluj výpis v konzoli.
  • Po každém kroku si ověř i soubor ve složce (Průzkumník → F5).
  • Vždy pracujeme s jedním souborem databáze: movies.db vedle skriptu.
Doporučení: Spouštěj z terminálu ve složce skriptu (powershell v adresním řádku složky).
Primární akce (např. zadání) Python sqlite3
Krok 0: Startovní soubor – jen cesta k databázi a kontrola existence

Vytvoř soubor lab_db.py a vlož pouze:

import sqlite3
from pathlib import Path

DB_FILE = Path(__file__).resolve().with_name("movies.db")

print("DB path:", DB_FILE)
print("DB exists:", DB_FILE.exists())

input("ENTER pro ukončení...")
  1. Spusť: python lab_db.py
  2. Ověř, že vidíš cestu k DB souboru.
  3. DB exists bude pravděpodobně False.
Co ověřujeme: kde přesně bude databáze ležet a zda soubor existuje.
Krok 1: Otevření a zavření databáze (connect/close) + ověření souboru

Teď přidáme otevření databáze pomocí sqlite3.connect(). SQLite při prvním připojení obvykle vytvoří databázový soubor, pokud ještě neexistuje. 

import sqlite3
from pathlib import Path

DB_FILE = Path(__file__).resolve().with_name("movies.db")

print("DB path:", DB_FILE)
print("DB exists BEFORE connect:", DB_FILE.exists())

conn = sqlite3.connect(str(DB_FILE))
print("Connection opened:", conn)

conn.close()
print("Connection closed.")

print("DB exists AFTER close:", DB_FILE.exists())
input("ENTER pro ukončení...")
  1. Spusť skript.
  2. V Průzkumníku obnov (F5) a ověř, že se objevil movies.db.
  3. V konzoli se DB exists AFTER close typicky změní na True.
Co ověřujeme: že connect skutečně pracuje se souborem databáze a že jej umíme vytvořit/otevřít.
Krok 2: Přesun do funkce – stejný test, ale přes open_and_close_db()

Stejný postup dáme do funkce. Důležité: funkce se nevykoná, dokud ji nezavoláš. 

import sqlite3
from pathlib import Path

DB_FILE = Path(__file__).resolve().with_name("movies.db")

def open_and_close_db():
    print("DB path:", DB_FILE)
    print("DB exists BEFORE connect:", DB_FILE.exists())

    conn = sqlite3.connect(str(DB_FILE))
    print("Connection opened:", conn)

    conn.close()
    print("Connection closed.")

    print("DB exists AFTER close:", DB_FILE.exists())

if __name__ == "__main__":
    open_and_close_db()
    input("ENTER pro ukončení...")
  1. Spusť skript.
  2. Výpis musí odpovídat tomu, co jsi viděl v kroku 1.
Co ověřujeme: že rozumíš rozdílu „definice funkce“ vs. „volání funkce“.
Krok 3: Vytvoření tabulky (CREATE TABLE) + ověření přes seznam tabulek

Vytvoříme tabulku movies. Nejdřív provedeme SQL příkaz a potom si ověříme, že tabulka opravdu existuje, pomocí dotazu do sqlite_master. 

def create_table_movies():
    conn = sqlite3.connect(str(DB_FILE))

    conn.execute("""
    CREATE TABLE IF NOT EXISTS movies (
        id    INTEGER PRIMARY KEY AUTOINCREMENT,
        title TEXT    NOT NULL,
        year  INTEGER
    );
    """)

    conn.commit()
    conn.close()

def list_tables():
    conn = sqlite3.connect(str(DB_FILE))
    rows = conn.execute("SELECT name FROM sqlite_master WHERE type='table';").fetchall()
    conn.close()
    return [r[0] for r in rows]

if __name__ == "__main__":
    open_and_close_db()

    print("--- creating table ---")
    create_table_movies()
    print("Tables:", list_tables())

    input("ENTER pro ukončení...")
  1. Spusť skript.
  2. Ve výpisu tabulek musí být movies.
Co ověřujeme: že SQL příkaz CREATE TABLE opravdu změnil strukturu databáze.
Krok 4: Ověření, že tabulka je prázdná (COUNT před vložením)

Před vložením dat si změříme počet záznamů. Tím dostaneš pevný výchozí bod pro ověření INSERTu. 

def count_movies():
    conn = sqlite3.connect(str(DB_FILE))
    n = conn.execute("SELECT COUNT(*) FROM movies;").fetchone()[0]
    conn.close()
    return n

if __name__ == "__main__":
    create_table_movies()

    before = count_movies()
    print("Count BEFORE insert:", before)

    input("ENTER pro ukončení...")

Očekávání: při první práci s tabulkou to bude obvykle 0.

Krok 5: INSERT (vložíme jeden film) + ověření přes COUNT

Teď přidáme funkci insert_movie() a ověříme, že se počet záznamů zvýšil přesně o 1. 

def insert_movie(title, year):
    conn = sqlite3.connect(str(DB_FILE))
    conn.execute("INSERT INTO movies (title, year) VALUES (?, ?);", (title, year))
    conn.commit()
    conn.close()

if __name__ == "__main__":
    create_table_movies()

    before = count_movies()
    print("Count BEFORE:", before)

    title = input("Název filmu: ").strip()
    year_text = input("Rok (ENTER = nevyplnit): ").strip()
    year = int(year_text) if year_text else None

    insert_movie(title, year)

    after = count_movies()
    print("Count AFTER:", after)

    print("Expected change:", after - before)
    input("ENTER pro ukončení...")
  1. Spusť skript.
  2. Zadej film.
  3. Expected change musí být 1.
Co ověřujeme: že INSERT opravdu uložil data a commit() je potvrdil.
Krok 6: Výpis obsahu tabulky (SELECT) – první čtení dat

Přidáme první čtení dat: vypíšeme uložené filmy. To je přímý důkaz, že data v tabulce opravdu jsou. 

def list_movies():
    conn = sqlite3.connect(str(DB_FILE))
    rows = conn.execute("SELECT id, title, year FROM movies ORDER BY id DESC;").fetchall()
    conn.close()
    return rows

if __name__ == "__main__":
    create_table_movies()

    rows = list_movies()
    print("Rows:", len(rows))
    for (movie_id, title, year) in rows:
        yr = year if year is not None else "?"
        print(f"#{movie_id} | {title} ({yr})")

    input("ENTER pro ukončení...")
Co ověřujeme: že umíme data nejen zapisovat, ale i číst (SELECT).
Poznámka: proč používáme if __name__ == "__main__": v těchto testech

Každý krok v této lekci má svůj „testovací běh“. Ten chceme spustit jen tehdy, když soubor spouštíme jako program. Pokud bychom jednou tyto funkce importovali do jiného souboru, nechceme, aby se testy spouštěly automaticky.

  • Přímé spuštění: python lab_db.py → __name__ je "__main__".
  • Import: import lab_db → __name__ je "lab_db".
Smysl v metodice: v každém kroku máme kontrolovaný „spouštěcí bod“ pro test.

Shrnutí

  • Umíš otevřít a zavřít databázi (connect/close) a ověřit existenci souboru.
  • Umíš vytvořit tabulku (CREATE TABLE) a ověřit ji přes sqlite_master.
  • Umíš vložit záznam (INSERT) a ověřit změnu přes COUNT(*).
  • Umíš číst data (SELECT) a vypsat obsah tabulky.
