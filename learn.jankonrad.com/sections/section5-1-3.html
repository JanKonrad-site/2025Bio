3.2.1 SQLite v Pythonu – první databáze úplně pomalu a polopatě

V této lekci nebudeme nic uspěchávat. Nepůjdeme stylem „zkopíruj hotový kód a doufej, že to funguje“. Naopak: po každém malém kroku skript spustíš, podíváš se do konzole a ověříš si, co se právě stalo. Cílem není jen „aby to jelo“, ale aby bylo jasné, co dělá connect(), co dělá execute(), proč existuje commit() a proč databázi na konci zavíráme pomocí close().

Co je vlastně databáze v této lekci?

Budeme používat SQLite. To je jednoduchá databáze, která nepotřebuje žádný server. Nepřipojuješ se nikam na internet. Databáze je v tomto případě normálně soubor, který bude uložený vedle tvého Python skriptu.

Ten soubor se bude jmenovat movies.db. Uvnitř toho souboru budou tabulky a v nich data.

Představa:
movies.db = krabice
tabulka movies = jeden šanon uvnitř krabice
jednotlivé řádky = jednotlivé kartičky / záznamy

Pravidla práce v této lekci

  • Po každém kroku skript spusť.
  • Po každém kroku zkontroluj výpis v konzoli.
  • Po každém kroku zkontroluj i soubor ve složce (Průzkumník → F5).
  • Nepřeskakuj kroky. Tady je důležitější pochopení než rychlost.
  • Celou dobu budeme pracovat s jedním databázovým souborem: movies.db.
Důležité: databáze se tady nechová jako proměnná v paměti. Je to soubor na disku. Takže některé změny opravdu uvidíš až po spuštění skriptu a po obnovení složky.
Python sqlite3
Krok 0: Zjisti, kde databáze bude, a ověř, jestli už existuje

Než databázi otevřeme, je dobré vědět dvě věci:

  1. Kde přesně bude databázový soubor uložený.
  2. Jestli ten soubor už existuje, nebo ještě ne.

Vytvoř soubor lab_db.py a vlož do něj toto:

import sqlite3
from pathlib import Path

DB_FILE = Path(__file__).resolve().with_name("movies.db")

print("DB path:", DB_FILE)
print("DB exists:", DB_FILE.exists())

input("ENTER pro ukončení...")

Co znamenají jednotlivé řádky?

  • import sqlite3 – načítáme knihovnu pro práci s databází SQLite.
  • from pathlib import Path – budeme pohodlně pracovat s cestou k souboru.
  • DB_FILE = ... – říkáme: databáze se bude jmenovat movies.db a bude ležet vedle tohoto skriptu.
  • DB_FILE.exists() – vrátí True nebo False podle toho, jestli soubor už existuje.
  1. Spusť: python lab_db.py
  2. Podívej se na vypsanou cestu.
  3. Podívej se na hodnotu DB exists.
Co teď čekáme:
Pokud databázi ještě nikdo nevytvořil, pravděpodobně uvidíš False.
To je v pořádku. Zatím jsme databázi ještě neotevřeli ani nevytvořili.
Krok 1: Otevření databáze pomocí connect() a zavření pomocí close()

Teď uděláme první skutečný kontakt s databází. Použijeme sqlite3.connect(...).

Tím Python řekne: „Otevři databázi v tomto souboru. Pokud soubor ještě neexistuje, vytvoř ho.“ 

import sqlite3
from pathlib import Path

DB_FILE = Path(__file__).resolve().with_name("movies.db")

print("DB path:", DB_FILE)
print("DB exists BEFORE connect:", DB_FILE.exists())

conn = sqlite3.connect(str(DB_FILE))
print("Connection opened:", conn)

conn.close()
print("Connection closed.")

print("DB exists AFTER close:", DB_FILE.exists())
input("ENTER pro ukončení...")

Co je tady nové?

  • conn = sqlite3.connect(...) – otevře spojení s databází.
  • conn je zkratka od connection, tedy spojení.
  • conn.close() – spojení korektně zavře.
Představa:
Databáze je jako sešit.
connect() = otevřít sešit
close() = zavřít sešit
  1. Spusť skript.
  2. Podívej se, jestli se ve složce objevil soubor movies.db.
  3. V konzoli si všimni rozdílu mezi BEFORE a AFTER.
Co ověřujeme:
Nepracujeme zatím s tabulkou ani s daty. Jen si dokazujeme, že Python umí databázový soubor otevřít a že SQLite ten soubor umí vytvořit.
Krok 2: Přesun do funkce – stejná práce, jen přehledněji

Teď uděláme úplně to samé, ale zabalíme to do funkce. Důvod je jednoduchý: nechceme mít všechno naházené jen tak pod sebe.

Funkce je pojmenovaný blok kódu. Sama od sebe se ale nespustí. Jen existuje. Spustí se až ve chvíli, kdy ji zavoláš. 

import sqlite3
from pathlib import Path

DB_FILE = Path(__file__).resolve().with_name("movies.db")

def open_and_close_db():
    print("DB path:", DB_FILE)
    print("DB exists BEFORE connect:", DB_FILE.exists())

    conn = sqlite3.connect(str(DB_FILE))
    print("Connection opened:", conn)

    conn.close()
    print("Connection closed.")

    print("DB exists AFTER close:", DB_FILE.exists())

if __name__ == "__main__":
    open_and_close_db()
    input("ENTER pro ukončení...")

Co je zde důležité pochopit?

  • def open_and_close_db(): – tady funkci jen vytváříš.
  • open_and_close_db() – tady ji opravdu spouštíš.
Typická chyba začátečníka:
Napíše funkci, ale zapomene ji zavolat. Program pak „nic nedělá“ a student neví proč.
  1. Spusť skript.
  2. Výpis by měl být skoro stejný jako v předchozím kroku.
Krok 3: Vytvoření tabulky – první skutečná struktura databáze

Databázový soubor už máme. Ale zatím je skoro prázdný. Teď v něm vytvoříme tabulku movies.

Tabulka je něco jako tabulka v Excelu: má sloupce a v nich budou uložená data. 

def create_table_movies():
    conn = sqlite3.connect(str(DB_FILE))

    conn.execute("""
    CREATE TABLE IF NOT EXISTS movies (
        id    INTEGER PRIMARY KEY AUTOINCREMENT,
        title TEXT    NOT NULL,
        year  INTEGER
    );
    """)

    conn.commit()
    conn.close()

def list_tables():
    conn = sqlite3.connect(str(DB_FILE))
    rows = conn.execute("SELECT name FROM sqlite_master WHERE type='table';").fetchall()
    conn.close()
    return [r[0] for r in rows]

if __name__ == "__main__":
    open_and_close_db()

    print("--- creating table ---")
    create_table_movies()
    print("Tables:", list_tables())

    input("ENTER pro ukončení...")

Co dělá conn.execute(...)?

To znamená: „Na tomto spojení s databází proveď SQL příkaz.“

V našem případě je SQL příkaz: 

CREATE TABLE IF NOT EXISTS movies (
    id    INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT    NOT NULL,
    year  INTEGER
);

Rozbor SQL úplně jednoduše:

  • CREATE TABLE – vytvoř tabulku
  • IF NOT EXISTS – ale jen pokud ještě neexistuje
  • movies – název tabulky
  • id INTEGER PRIMARY KEY AUTOINCREMENT – číselné ID, které se bude samo navyšovat
  • title TEXT NOT NULL – název filmu, text, nesmí být prázdný
  • year INTEGER – rok, celé číslo, může být i prázdný

Proč je tam conn.commit()?

commit() znamená: „Potvrď a ulož změny natrvalo.“

Bez commit() bys sice mohl příkaz provést, ale změna by nemusela být definitivně uložená.

Představa:
execute() = něco jsi napsal do sešitu
commit() = opravdu jsi to uložil
close() = zavřel jsi sešit

A co dělá list_tables()?

Tato funkce se podívá do interní tabulky SQLite jménem sqlite_master. To je takový „seznam objektů“ v databázi. Díky tomu si umíme ověřit, jestli tabulka movies opravdu vznikla.

  1. Spusť skript.
  2. Ve výpisu hledej, jestli mezi tabulkami vidíš movies.
Co teď víme:
Už nejen že existuje databázový soubor, ale uvnitř už existuje i konkrétní tabulka.
Krok 4: Ověření, že tabulka je zatím prázdná

Než něco vložíme, je dobré si změřit výchozí stav. Prostě zjistíme, kolik řádků je teď v tabulce. 

def count_movies():
    conn = sqlite3.connect(str(DB_FILE))
    n = conn.execute("SELECT COUNT(*) FROM movies;").fetchone()[0]
    conn.close()
    return n

if __name__ == "__main__":
    create_table_movies()

    before = count_movies()
    print("Count BEFORE insert:", before)

    input("ENTER pro ukončení...")

Co tady dělá SQL dotaz?

SELECT COUNT(*) FROM movies;
  • SELECT – vyber / přečti data
  • COUNT(*) – spočítej počet řádků
  • FROM movies – v tabulce movies

Co znamená fetchone()[0]?

Databáze vrátí výsledek jako řádek. U COUNT(*) dostaneme jeden řádek a v něm jedno číslo.

Takže:

  • fetchone() – vezmi jeden vrácený řádek
  • [0] – vezmi první hodnotu z toho řádku
Očekávání:
Pokud jsi ještě do tabulky nic nevložil, výsledek bude pravděpodobně 0.
Krok 5: INSERT – vložení jednoho filmu do tabulky

Teď uděláme to, co od databáze většina lidí čeká: uložíme do ní data. 

def insert_movie(title, year):
    conn = sqlite3.connect(str(DB_FILE))
    conn.execute("INSERT INTO movies (title, year) VALUES (?, ?);", (title, year))
    conn.commit()
    conn.close()

if __name__ == "__main__":
    create_table_movies()

    before = count_movies()
    print("Count BEFORE:", before)

    title = input("Název filmu: ").strip()
    year_text = input("Rok (ENTER = nevyplnit): ").strip()
    year = int(year_text) if year_text else None

    insert_movie(title, year)

    after = count_movies()
    print("Count AFTER:", after)

    print("Expected change:", after - before)
    input("ENTER pro ukončení...")

Rozbor INSERTu:

INSERT INTO movies (title, year) VALUES (?, ?);
  • INSERT INTO – vlož do tabulky
  • movies – název tabulky
  • (title, year) – do kterých sloupců zapisujeme
  • VALUES (?, ?) – sem se dosadí skutečné hodnoty

Proč tam nejsou hodnoty napsané natvrdo?

Protože používáme parametry: 

(title, year)

To je bezpečnější a správný způsob práce s databází.

Co dělá tato část?

year = int(year_text) if year_text else None

Přeloženo úplně lidsky:

  • Pokud uživatel něco napsal, převedeme to na číslo pomocí int().
  • Pokud nic nenapsal a jen zmáčkl Enter, uloží se None.

None se v SQLite uloží jako NULL, tedy „žádná hodnota“.

  1. Spusť skript.
  2. Zadej název filmu.
  3. Zadej rok nebo nech pole prázdné.
  4. Zkontroluj, že Expected change je 1.
Co tím dokazujeme:
Že jsme opravdu vložili jeden nový řádek do tabulky a že commit() změnu skutečně uložil.
Krok 6: SELECT – přečtení a výpis uložených filmů

Teď už nechceme jen věřit počítadlu řádků. Chceme vidět skutečný obsah tabulky. 

def list_movies():
    conn = sqlite3.connect(str(DB_FILE))
    rows = conn.execute("SELECT id, title, year FROM movies ORDER BY id DESC;").fetchall()
    conn.close()
    return rows

if __name__ == "__main__":
    create_table_movies()

    rows = list_movies()
    print("Rows:", len(rows))
    for (movie_id, title, year) in rows:
        yr = year if year is not None else "?"
        print(f"#{movie_id} | {title} ({yr})")

    input("ENTER pro ukončení...")

Rozbor SQL:

SELECT id, title, year FROM movies ORDER BY id DESC;
  • SELECT id, title, year – chceme tyto tři sloupce
  • FROM movies – z tabulky movies
  • ORDER BY id DESC – seřaď podle ID od nejvyššího k nejnižšímu

Co dělá fetchall()?

Vrátí všechny nalezené řádky najednou. Výsledek je seznam řádků.

Co dělá tento řádek?

yr = year if year is not None else "?"

Pokud film rok má, vypíše se rok. Pokud je rok v databázi prázdný, vypíše se místo něj otazník.

  1. Spusť skript.
  2. Podívej se, kolik řádků databáze vrací.
  3. Zkontroluj, že vidíš skutečně uložené filmy.
Teď už umíš obě hlavní věci:
data do databáze uložit i data z databáze přečíst.
Co přesně znamená conn.execute(), conn.commit() a conn.close()?

conn.execute(...)

Znamená: „Proveď SQL příkaz nad touto databází.“

conn.commit()

Znamená: „Potvrď a ulož změny natrvalo.“

conn.close()

Znamená: „Zavři spojení s databází, už s ní teď nebudu pracovat.“

Zapamatuj si to takto:
connect() = otevřít
execute() = něco udělat
commit() = uložit změnu
close() = zavřít
Proč používáme if __name__ == "__main__":?

Tento blok říká: „Následující kód spusť jen tehdy, když tento soubor spouštím přímo.“

To je praktické hlavně u testování. Funkce chceme mít definované pořád, ale nechceme, aby se testy spouštěly automaticky při importu do jiného souboru.

  • python lab_db.py → testovací část se spustí
  • import lab_db → testovací část se nespustí
Jednoduše:
Funkce = nástroje
if __name__ == "__main__": = místo, kde si ty nástroje zkoušíš

Co si máš z této lekce opravdu odnést

  • Databáze SQLite je v této lekci obyčejný soubor .db.
  • connect() otevře spojení s databází.
  • execute() provede SQL příkaz.
  • commit() uloží změny natrvalo.
  • close() zavře databázi.
  • CREATE TABLE vytvoří tabulku.
  • INSERT vloží data.
  • SELECT data čte.
