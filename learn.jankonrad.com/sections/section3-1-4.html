3.1.4 Odkazy (<a>) a obrázky (<img>): cesty, alt, portfolio

Co se naučíš?

  • Odkazy – jak funguje atribut href, kdy použít relativní/kořenově relativní cesty.
  • Obrázky – jak funguje atribut src a alt, práce s adresářem Assets/.
  • Struktura projektu – odkazy mezi lekcemi, vložení obrázku z podadresáře, základ portfolia.

Zápisky & procvičení seznamů s odkazy

Ukázka odkazů samostatně, v číslovaném i nečíslovaném seznamu, včetně záměrně špatného použití <ul> bez <li> (proč je to problém viz akordeon níže).

<!DOCTYPE html>
<html lang="en">
  <head>…</head>
  <body>
    <a href="">Odkaz1</a> … <a href="">Odkaz5</a>

    <ol>
      <li><a href="">Odkaz1</a></li>
      …
      <li><a href="">Odkaz5</a></li>
    </ol>

    <ul>
      <li><a href="">Odkaz1</a></li> … <li><a href="">Odkaz5</a></li>
    </ul>

    <!-- ŠPATNĚ: <ul> musí obsahovat <li>, ne přímo <a> -->
    <ul> <a href="">Odkaz1</a> </ul>
    … (opakovaně)
  </body>
</html>

Cílem je vytvořit testovací stránku s odkazy na různé podadresáře kurzu pomocí kořenově relativních cest (začínají /).

  1. Vytvoř složku (např. lekce-3-1-4) a v ní index.html. Vlož boilerplate: napiš ! → Tab/Enter.
  2. Do <body> vlož sadu odkazů:

VS Code tipy: Emmet pro odkazy: napiš a{1-Lekce}+a{2-Lekce}+… a kurzorem pak doplň href. Formátování: Win/Linux Shift+Alt+F, macOS ⌥⇧F.

Vytvoříš jednoduchou portfoliovou stránku s nadpisem, popisem, sadou odkazů a vloženým obrázkem z podadresáře Assets.

  1. Ve složce projektu vytvoř podadresář Assets a vlož do něj soubor obrázku (např. win.jpeg).
  2. Do <body> přidej hlavičku portfolia a odstavce s odkazy (můžeš recyklovat odkazy z Cvičení 1): 
    <div>
  <h1>Portfólio prací</h1>
  <p>Zde je můj seznam prací za první měsíc výuky v HTML.</p>
</div>

<div>
  <a href="/Kvinta - HTML & CSS/1_Lekce/index.html">1-Lekce</a>
  <a href="/Kvinta - HTML & CSS/2.Lekce/index.html">2-Lekce</a>
  <a href="/Kvinta - HTML & CSS/3.Lekce/index.html">3-Lekce</a>
  <a href="/Kvinta - HTML & CSS/4.Lekce/index.html">4-Lekce</a>
  <a href="/Kvinta - HTML & CSS/4.Lekce/1.Cviceni/index.html">4.1-Lekce</a>
  <a href="/Kvinta - HTML & CSS/4.Lekce/2.Cviceni/index.html">4.2-Lekce</a>
  <a href="/Kvinta - HTML & CSS/5.Lekce/index.html">5-Lekce</a>
  <a href="/Kvinta - HTML & CSS/5.Lekce/1.Cviceni/index.html">5.1-Lekce</a>
</div>
  3. Pod odkazy vlož obrázek z aktuální složky s relativní cestou ./Assets/win.jpeg a smysluplným alt: 
    <div>
  <img src="./Assets/win.jpeg" alt="Vítězný obrázek (ilustrace úspěchu)">
</div>

    Proč je alt důležitý? .

Kontrola cest: ./ = „odtud“, ../ = „o složku výš“. Pokud se obrázek nezobrazuje, zkontroluj, zda soubor opravdu leží v Assets/ vedle index.html (nebo uprav cestu).

Seznamy mají jasná pravidla: přímými potomky <ul>/<ol> jsou <li>. Špatně je psát <ul><a …>… bez <li>.

<!-- ŠPATNĚ -->
<ul>
  <a href="">Odkaz1</a>
</ul>

<!-- SPRÁVNĚ -->
<ul>
  <li><a href="">Odkaz1</a></li>
</ul>
Proč? Přístupnost, čtečky obrazovky, konzistence a CSS styly očekávají strukturu seznam → položky → obsah položky (např. odkaz).

Shrnutí

  • <a href="…"> odkazuje na dokument. Cesta může být relativní (./, ../) nebo kořenově relativní (/).
  • <img src="…" alt="…"> vloží obrázek; alt popíše obsah (přístupnost, SEO, fallback).
  • <ul>/<ol> → <li> → <a> je správná hierarchie pro seznam odkazů.
Jak funguje href u odkazu

<a href="…"> říká prohlížeči, kam má odkaz vést. Absolutní URL (např. https://…) míří na jiný web. Relativní cesta míří v rámci tvého projektu.

  • ./soubor.html – v aktuální složce.
  • ../soubor.html – o úroveň výš.
  • /cesta/od/korenoveho/umístění – kořenově relativní, závisí na tom, kde je „root“ serveru.

Chyby odkazů často způsobí překlepy ve jménech souborů/složek nebo špatně zvolený typ cesty.

Typy cest: relativní × kořenově relativní

Relativní (doporučeno v malých projektech): ./ a ../ – robustní při přenosu projektu. Kořenově relativní (/…) je pohodlná v rámci jednoho webu, ale mimo server s daným „rootem“ může selhat.

Příklad: stránka v /Kvinta - HTML & CSS/4.Lekce/ a obrázek v /Kvinta - HTML & CSS/4.Lekce/Assets/win.jpeg → použij ./Assets/win.jpeg.

Atribut alt u obrázků

alt popisuje, co je na obrázku – pomáhá čtečkám obrazovky, zobrazí se při chybě načtení a je užitečný pro SEO.

Příklad: <img src="./Assets/win.jpeg" alt="Vítězný pohár na stupních vítězů">

Nezadávej „ilustrační obrázek“ – napiš smysl (obsah/funkci) pro danou stránku.
