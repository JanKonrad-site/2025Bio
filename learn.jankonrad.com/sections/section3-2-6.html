3.2.6 Pokročilé CSS – atributové selektory, pseudo-třídy, transition a position

Co se naučíš?

  • Používat attribute selector pro výběr prvků podle atributu nebo jeho hodnoty.
  • Pracovat s důležitými pseudo-třídami jako :hover, :focus, :checked, :valid, :invalid, :first-child, :last-child a :nth-child().
  • Používat transition pro plynulé hover efekty a jemnější interakce.
  • Pochopit vlastnost position a rozdíl mezi relative, absolute, fixed a sticky.
  • Lépe chápat logiku CSS selektorů a vztahů mezi prvky.

1️⃣ Attribute selector – výběr podle atributu

Attribute selector slouží k tomu, aby sis v CSS vybral prvky podle jejich atributů. Můžeš například stylovat odkazy s konkrétním target, obrázky s určitým alt, ale velmi často se používá hlavně u formulářových prvků.

Typická situace: chceš jinak stylovat pole pro email, jinak heslo a jinak odesílací tlačítko. Všechny jsou to sice prvky input, ale mají jinou hodnotu atributu type.

Attribute selector – jak funguje

Základní zápis vypadá takto:

input[type="email"] {
  border: 2px solid steelblue;
}
  • input = vybíráme prvek input
  • [type="email"] = ale jen ten, který má atribut type nastavený na email

Tímto způsobem můžeš stylovat různé inputy odděleně, i když mají stejný tag.

Často se používá i zápis více selektorů najednou: 

input[type="email"],
input[type="password"] {
  width: 250px;
}
Proč mají inputy jinou skutečnou velikost?

I když zadáš stejnou šířku a výšku, inputy mohou na stránce působit různě velké. Důvodem je box model.

Ve výchozím stavu je box-sizing: content-box;, což znamená: šířka a výška se vztahují jen na obsah, ale padding a border se přičítají navíc. 

input {
  box-sizing: border-box;
}

Když použiješ border-box, padding i border se započítají dovnitř prvku, takže prvky drží přesnější velikost.

Proč použít min-height: 100vh; display: grid; place-items: center;

Tento zápis se často používá, když chceme vycentrovat obsah přesně doprostřed obrazovky:

body {
  min-height: 100vh;
  display: grid;
  place-items: center;
}
  • min-height: 100vh; znamená, že body bude mít minimálně výšku celé obrazovky.
  • display: grid; zapne gridový režim.
  • place-items: center; vycentruje obsah současně vodorovně i svisle.

Je to krátký a velmi přehledný způsob, jak umístit například formulář doprostřed stránky. 

<form class="login-form">
  <input type="email" placeholder="Zadej email">
  <input type="password" placeholder="Zadej heslo">
  <input type="submit" value="Přihlásit">
</form>

body {
  min-height: 100vh;
  margin: 0;
  display: grid;
  place-items: center;
  font-family: system-ui, sans-serif;
  background: #f8fafc;
}

.login-form {
  padding: 20px;
  background: white;
  border-radius: 14px;
  box-shadow: 0 10px 24px rgba(0,0,0,.08);
}

input {
  display: block;
  margin: 8px 0;
  box-sizing: border-box;
}

input[type="email"],
input[type="password"] {
  width: 260px;
  height: 42px;
  padding: 8px 12px;
  border: 1px solid #94a3b8;
  border-radius: 8px;
}

input[type="submit"] {
  width: 260px;
  height: 42px;
  border: none;
  border-radius: 8px;
  background-color: #2563eb;
  color: white;
  font-weight: bold;
  cursor: pointer;
}

2️⃣ Pseudo-třídy – stylování podle stavu prvku

Pseudo-třídy umožňují stylovat prvek podle jeho stavu. Například když na prvek najedeš myší, klikneš na něj, vložíš kurzor do inputu, zaškrtneš checkbox nebo chceš stylovat první či poslední dítě rodiče.

Pseudo-třídy – základní přehled
  • :hover – když uživatel najede myší na prvek
  • :visited – navštívený odkaz
  • :active – moment kliknutí na odkaz nebo tlačítko
  • :focus – když je kurzor uvnitř inputu
  • :valid – když je hodnota ve formuláři správná
  • :invalid – když je hodnota neplatná
  • :checked – zaškrtnutý checkbox nebo radio button
Doporučení: :hover používej hlavně u prvků, které jsou klikatelné nebo interaktivní. Uživatel pak lépe pozná, že s nimi může pracovat.
first-child, last-child a nth-child()

Tyto pseudo-třídy pracují se strukturou HTML:

li:first-child { ... }
li:last-child  { ... }
li:nth-child(5) { ... }
li:nth-child(even) { ... }
li:nth-child(odd) { ... }
li:nth-child(3n) { ... }
  • :first-child = první dítě rodiče
  • :last-child = poslední dítě rodiče
  • :nth-child(5) = páté dítě
  • :nth-child(even) = sudé prvky
  • :nth-child(odd) = liché prvky
  • :nth-child(3n) = každý třetí prvek

3️⃣ Transition – plynulé přechody mezi stavy

Vlastnost transition slouží k tomu, aby změna stylu nebyla skoková, ale plynulá. Nejčastěji se používá u hover efektů.

Důležité pravidlo: transition se většinou zapisuje do výchozího stavu prvku, ne do hoveru. Díky tomu je animace plynulá jak při najetí myší, tak při jejím odjetí.

Transition – základní princip

Základní zápis:

.box {
  transition: 1s;
}

.box:hover {
  border-radius: 30px;
  background-color: tomato;
}

Tento zápis znamená: změna mezi normálním stavem a hover stavem potrvá 1 sekundu.

Můžeš také určit konkrétní vlastnost: 

transition: border-radius 1s;
Transition timing function

Timing function určuje průběh rychlosti animace:

  • linear – stále stejná rychlost
  • ease – pomalý start, zrychlení, pomalý konec
  • ease-in – pomalý začátek
  • ease-out – pomalý konec
  • ease-in-out – pomalý začátek i konec
transition: 300ms ease-in-out;
V praxi často velmi dobře funguje krátká animace kolem 150ms až 300ms s funkcí ease-in-out. 

<div class="transition-box">Najetí myší</div>

.transition-box {
  width: 220px;
  height: 120px;
  background-color: steelblue;
  color: white;
  display: grid;
  place-items: center;
  border-radius: 0;
  transition: 300ms ease-in-out;
}

.transition-box:hover {
  background-color: tomato;
  border-radius: 24px;
  transform: scale(1.05);
}

4️⃣ Position – přesné umístění prvků

Vlastnost position určuje, jak bude prvek umístěn v layoutu. Když použiješ jinou hodnotu než static, můžeš navíc pracovat s vlastnostmi top, right, bottom, left a také z-index.

Přehled hodnot position
  • static – výchozí stav, bez speciálního chování
  • relative – posun relativně vůči původní poloze prvku
  • absolute – pozice vůči nejbližšímu předkovi, který má nastavenou position
  • fixed – prvek je přichycený k oknu prohlížeče
  • sticky – prvek se chová normálně, ale při scrollu se „přilepí“

U pozicovaných prvků se často používá i z-index, který určuje, která vrstva bude nad ostatními.

Proč se absolute často kombinuje s relative?

Prvek s position: absolute; se umisťuje vůči nejbližšímu rodiči, který má nastavenou position. Pokud takový rodič není, bere se jako reference celé body. 

.parent {
  position: relative;
}

.child {
  position: absolute;
  top: 0;
  right: 0;
}

To je velmi častý vzor například u ikonky v rohu boxu, tlačítka uvnitř inputu, badge prvku nebo překryvného štítku. 

<div class="box">Posunutý box</div>

.box {
  position: relative;
  top: 20px;
  left: 20px;
}

Prvek se posune vzhledem ke své původní poloze. 

<div class="popup">Jsem vycentrované okno</div>

.popup {
  position: fixed;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
}
Tento princip se často používá u modálních oken nebo vyskakovacích panelů. 

<nav class="navbar">Jsem sticky navigace</nav>

.navbar {
  position: sticky;
  top: 0;
  z-index: 10;
}

Navigace zůstane při scrollování „přilepená“ nahoře. 

<div class="search-box">
  <input type="text" placeholder="Hledat...">
  <button class="search-btn">🔍</button>
</div>

.search-box {
  position: relative;
  width: 300px;
}

.search-btn {
  position: absolute;
  top: 50%;
  right: 10px;
  transform: translateY(-50%);
}

<div class="overlay"></div>
<div class="modal-window">Obsah modálního okna</div>

.overlay {
  position: fixed;
  inset: 0;
  background: rgba(0,0,0,.5);
  z-index: 100;
}

.modal-window {
  position: fixed;
  z-index: 200;
}

Vyšší z-index znamená vyšší vrstvu nad ostatními prvky.

📘 Shrnutí kapitoly

  • Attribute selector vybírá prvky podle atributu a jeho hodnoty, například input[type="email"].
  • Pseudo-třídy stylují prvky podle stavu nebo pozice ve struktuře, např. :hover, :focus, :checked, :nth-child().
  • Transition dělá změny plynulé a zlepšuje uživatelský zážitek.
  • Position umožňuje přesněji umisťovat prvky a pracovat s vrstvami přes z-index.
  • V pokročilejším CSS často nejde jen o znalost vlastností, ale hlavně o pochopení logiky selektorů a vztahů mezi prvky.
