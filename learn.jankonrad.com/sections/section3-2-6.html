3.2.6 Pokročilé CSS – atributové selektory, pseudo-třídy, transition a position

Co se naučíš?

  • Vybrat prvek podle atributu pomocí attribute selectoru.
  • Stylovat prvky podle jejich stavu a pozice ve struktuře pomocí pseudo-tříd.
  • Vytvářet plynulé hover efekty pomocí vlastnosti transition.
  • Pochopit, jak funguje position v reálných situacích: navigace, badge, tlačítko v inputu, modální okno.
  • Vnímat CSS ne jen jako seznam vlastností, ale jako logiku výběru a chování prvků.

1️⃣ Attribute selector – výběr podle atributu

Attribute selector vybírá HTML prvky podle toho, jaký mají atribut nebo jakou má tento atribut hodnotu. To je užitečné hlavně tehdy, když mají dva prvky stejný tag, ale jiný význam.

Typický příklad: ve formuláři jsou všechny prvky <input>, ale některé mají type="email", jiné type="password" a jiné type="submit". Každý z nich chceme stylovat jinak. 

<form class="login-form">
  <input type="email" placeholder="Zadej email">
  <input type="password" placeholder="Zadej heslo">
  <input type="submit" value="Přihlásit">
</form>

Všechny prvky jsou input, ale každý má jinou roli. A právě to je důvod, proč potřebujeme attribute selector. 

body {
  min-height: 100vh;
  margin: 0;
  display: grid;
  place-items: center;
  font-family: system-ui, sans-serif;
  background: #f8fafc;
}

.login-form {
  padding: 20px;
  background: white;
  border-radius: 14px;
  box-shadow: 0 10px 24px rgba(0,0,0,.08);
}

Tady ještě formulář nestylujeme podle typu inputu. Jen připravujeme čisté a přehledné prostředí. 

input {
  display: block;
  margin: 8px 0;
  box-sizing: border-box;
}

input[type="email"],
input[type="password"] {
  width: 260px;
  height: 42px;
  padding: 8px 12px;
  border: 1px solid #94a3b8;
  border-radius: 8px;
}

input[type="submit"] {
  width: 260px;
  height: 42px;
  border: none;
  border-radius: 8px;
  background-color: #2563eb;
  color: white;
  font-weight: bold;
  cursor: pointer;
}

Zde už používáme tři úrovně logiky:

  • nejprve společný styl pro všechny inputy,
  • potom společný styl pro textová pole,
  • a nakonec zvláštní styl pro odesílací tlačítko.

2️⃣ Pseudo-třídy – stylování podle stavu a pozice

Pseudo-třídy stylují prvek podle jeho stavu nebo podle jeho pozice ve struktuře HTML. Nevybíráš tedy „jiný typ prvku“, ale tentýž prvek v jiné situaci.

To je velmi důležité: :hover neznamená nový prvek, ale tentýž prvek ve chvíli, kdy nad ním je myš. Stejně tak :focus, :checked nebo :nth-child(). 

<a href="https://example.com" target="_blank">Otevřít ukázkový odkaz</a>
a {
  color: #2563eb;
  text-decoration: none;
}

a:hover {
  color: #1d4ed8;
  text-decoration: underline;
}

a:visited {
  color: #7c3aed;
}

a:active {
  color: #dc2626;
}

Tady student krásně vidí, že tentýž odkaz se může vzhledově měnit podle toho, jestli je nad ním myš, jestli už byl navštíven, nebo jestli je právě klikán. 

<input type="email" placeholder="Zadej svůj email">
input[type="email"] {
  border: 2px solid #cbd5e1;
  padding: 10px;
  border-radius: 8px;
  outline: none;
}

input[type="email"]:focus {
  border-color: #2563eb;
  background-color: #eff6ff;
}

input[type="email"]:valid {
  border-color: #16a34a;
}

input[type="email"]:invalid {
  border-color: #dc2626;
}

Výchozí outline můžeme nahradit vlastním stylováním, ale musí být stále dobře poznat, že je input aktivní. 

<label>
  <input type="checkbox">
  <span>Dokončit domácí úkol</span>
</label>
input[type="checkbox"]:checked + span {
  text-decoration: line-through;
  color: gray;
}

Tady už se kombinuje víc principů najednou:

  • attribute selector vybírá checkbox,
  • :checked reaguje na jeho stav,
  • + vybírá hned následujícího sourozence, tedy span.

Proč je tam právě span?
Protože potřebujeme mít text v samostatném elementu, který půjde jednoduše stylovat.

Proč je to celé zabalené do label?
Protože label propojí checkbox a text do jednoho klikatelného celku. 

<ol>
  <li>První položka</li>
  <li>Druhá položka</li>
  <li>Třetí položka</li>
  <li>Čtvrtá položka</li>
  <li>Pátá položka</li>
  <li>Šestá položka</li>
  <li>Sedmá položka</li>
  <li>Osmá položka</li>
</ol>
li:first-child {
  border: 2px solid black;
}

li:last-child {
  border: 2px solid black;
}

li:nth-child(5) {
  background-color: #fde68a;
}

li:nth-child(even) {
  background-color: #f1f5f9;
}

li:nth-child(3n) {
  color: #2563eb;
}

To je typická ukázka strukturálních pseudo-tříd. Neřešíme stav prvku, ale jeho pořadí mezi sourozenci.

3️⃣ Transition – plynulé změny mezi normálním a hover stavem

Vlastnost transition říká prohlížeči, že změna mezi dvěma stavy nemá být okamžitá, ale plynulá. To je důležité hlavně u interaktivních prvků, jako jsou tlačítka, karty, odkazy nebo obrázky.

Největší metodická chyba bývá v tom, že se transition zapisuje až do :hover. Správně ji dáváme do výchozího stavu. 

<button class="btn-demo">Najetí myší</button>
.btn-demo {
  border: none;
  padding: 12px 20px;
  border-radius: 10px;
  background-color: #2563eb;
  color: white;
  font-weight: bold;
  cursor: pointer;

  transition: background-color 250ms ease-in-out,
              transform 250ms ease-in-out,
              box-shadow 250ms ease-in-out;
}

.btn-demo:hover {
  background-color: #1d4ed8;
  transform: translateY(-2px);
  box-shadow: 0 10px 20px rgba(37, 99, 235, .25);
}

Tohle je reálný UX příklad. Tlačítko se při hoveru lehce zvedne, ztmavne a dostane stín. 

<div class="card-demo">
  <h5>Mini karta</h5>
  <p>Na tuto kartu najeď myší a sleduj změnu stínu, posunu a barvy.</p>
</div>
.card-demo {
  width: 320px;
  padding: 20px;
  border-radius: 16px;
  background: white;
  border: 1px solid #cbd5e1;
  box-shadow: 0 4px 10px rgba(0,0,0,.06);

  transition: transform 300ms ease-in-out,
              box-shadow 300ms ease-in-out,
              border-color 300ms ease-in-out;
}

.card-demo:hover {
  transform: translateY(-6px);
  box-shadow: 0 14px 28px rgba(0,0,0,.12);
  border-color: #93c5fd;
}

Karta se „nadzvedne“ a zvýrazní. Přesně takové efekty se používají v moderních dashboardech a e-shopech.

4️⃣ Position – přesné umisťování prvků v reálných situacích

Vlastnost position není jen „posunutí boxu“. Používáme ji tehdy, když chceme prvek umístit přesněji, než dovoluje běžný tok stránky.

V praxi ji využiješ například pro:

  • badge v rohu karty,
  • ikonu uvnitř inputu,
  • sticky navigaci nahoře,
  • plovoucí tlačítko,
  • overlay a modální okno.

<div class="product-card">
  <span class="badge">NOVÉ</span>
  <h5>Produkt</h5>
  <p>Ukázka karty s odznakem v rohu.</p>
</div>
.product-card {
  position: relative;
  width: 280px;
  padding: 20px;
  border: 1px solid #cbd5e1;
  border-radius: 16px;
  background: white;
}

.badge {
  position: absolute;
  top: 10px;
  right: 10px;
  background: tomato;
  color: white;
  padding: 4px 10px;
  border-radius: 999px;
  font-size: 12px;
}

Tohle je ideální první ukázka absolute pozicování: odznak patří ke kartě, proto musí být pozicován vůči ní. 

<div class="search-box">
  <input type="text" placeholder="Hledat...">
  <button class="search-btn" aria-label="Hledat">🔍</button>
</div>
.search-box {
  position: relative;
  width: 320px;
}

.search-box input {
  width: 100%;
  padding: 12px 44px 12px 12px;
  border: 1px solid #94a3b8;
  border-radius: 10px;
}

.search-btn {
  position: absolute;
  top: 50%;
  right: 10px;
  transform: translateY(-50%);
  border: none;
  background: transparent;
  cursor: pointer;
}

Tlačítko působí, jako by bylo uvnitř inputu, ale v HTML je samostatné. To je velmi typické praktické použití. 

<nav class="navbar">
  <a href="#">Domů</a>
  <a href="#">O mně</a>
  <a href="#">Kontakt</a>
</nav>

<main class="long-content">
  <p>Dlouhý obsah stránky...</p>
  <p>Dlouhý obsah stránky...</p>
  <p>Dlouhý obsah stránky...</p>
  <p>Dlouhý obsah stránky...</p>
  <p>Dlouhý obsah stránky...</p>
</main>
.navbar {
  position: sticky;
  top: 0;
  z-index: 20;
  background: white;
  border-bottom: 1px solid #cbd5e1;
  padding: 12px 20px;
}

.navbar a {
  margin-right: 16px;
  text-decoration: none;
  color: #1e293b;
}

.long-content {
  min-height: 1500px;
  padding: 20px;
}
Sticky dává smysl jen tam, kde je opravdu co scrollovat. Proto je nutné mít dlouhý obsah pod navigací. 

<a href="#" class="floating-btn">↑</a>
.floating-btn {
  position: fixed;
  right: 20px;
  bottom: 20px;
  width: 52px;
  height: 52px;
  border-radius: 50%;
  background: #2563eb;
  color: white;
  text-decoration: none;
  display: grid;
  place-items: center;
  font-size: 22px;
  z-index: 50;
}

fixed drží prvek stále na stejném místě vzhledem k oknu prohlížeče. Typické je tlačítko „zpět nahoru“ nebo chat ikona.

Tento příklad je důležitý, protože v něm propojujeme několik věcí najednou: pseudo-třídu :checked, position, z-index a logiku „otevřít / zavřít“ bez JavaScriptu.

Hlavní myšlenka:
Na stránce si vytvoříme skrytý checkbox. Kliknutím na tlačítko nezačneme „otevírat okno“ přímo, ale jen změníme stav checkboxu. CSS pak pomocí :checked pozná, že je checkbox zapnutý, a podle toho zobrazí overlay i modální okno.
1) HTML struktura

Na stránce potřebujeme čtyři části:

  • skrytý checkbox – bude sloužit jako přepínač,
  • label, který checkbox „zapne“,
  • overlay – tmavé překrytí stránky,
  • samotné modální okno.
<input type="checkbox" id="open-modal" hidden>

<label for="open-modal" class="open-modal-btn">Otevřít modální okno</label>

<div class="overlay"></div>

<div class="modal-window">
  <label for="open-modal" class="close-modal-btn">×</label>
  <h5>Modální okno</h5>
  <p>Toto okno se zobrazí po kliknutí bez použití JavaScriptu.</p>
</div>
2) Co přesně dělá každý prvek
  • <input type="checkbox" id="open-modal" hidden> = skrytý checkbox, který si pamatuje stav zapnuto / vypnuto.
  • <label for="open-modal"> = label propojený s checkboxem. Když na něj klikneš, checkbox se přepne.
  • <div class="overlay"> = tmavá vrstva přes stránku.
  • <div class="modal-window"> = samotné modální okno.
  • <label for="open-modal" class="close-modal-btn">×</label> = zavírací křížek. Opět label, takže po kliknutí zase přepne tentýž checkbox.
Proč je zavírací křížek label, a ne span?
Protože samotný span by byl jen obyčejný obal textu. label je propojený s checkboxem přes atribut for, takže po kliknutí skutečně změní jeho stav.
3) Proč používáme label místo tlačítka

Label umí „ovládat“ formulářový prvek, ke kterému je připojen přes atribut for. 

<input type="checkbox" id="open-modal">
<label for="open-modal">Otevřít</label>

Kliknutí na label tedy změní stav checkboxu. Nepotřebujeme JavaScript typu „po kliknutí otevři okno“.

4) Výchozí stav: overlay i okno jsou schované
.overlay,
.modal-window {
  opacity: 0;
  pointer-events: none;
  transition: 250ms ease-in-out;
}
  • opacity: 0; = prvek je neviditelný
  • pointer-events: none; = nejde na něj kliknout
  • transition = zobrazení a skrytí bude plynulé
Nestačí jen opacity: 0, protože neviditelný prvek by mohl stále zachytávat kliknutí. Proto přidáváme i pointer-events: none.
5) Position a vrstvy
.overlay {
  position: fixed;
  inset: 0;
  background: rgba(0,0,0,.5);
  z-index: 100;
}

.modal-window {
  position: fixed;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
  width: 320px;
  padding: 24px;
  background: white;
  border-radius: 16px;
  z-index: 200;
  color: #1e293b;
}
  • position: fixed; = prvek je přilepený k oknu prohlížeče
  • inset: 0; = overlay vyplní celou obrazovku
  • top: 50%; a left: 50%; posunou okno do středu obrazovky
  • transform: translate(-50%, -50%) ho dorovná přesně na střed
  • z-index určuje pořadí vrstev
6) Okamžik otevření: co přesně dělá :checked
#open-modal:checked ~ .overlay,
#open-modal:checked ~ .modal-window {
  opacity: 1;
  pointer-events: auto;
}

Čti to takto:

  • vyber checkbox s ID open-modal,
  • ale jen když je :checked,
  • pak vyber jeho pozdější sourozence .overlay a .modal-window.

Jinými slovy: když je checkbox zapnutý, overlay i modální okno se zobrazí.

7) Jak funguje zavření okna
<label for="open-modal" class="close-modal-btn">×</label>

Po kliknutí se znovu přepne stejný checkbox. Pokud byl zapnutý, vypne se. Tím přestane platit selektor :checked a overlay i modální okno se zase schovají.

Jedna věta na zapamatování:
Kliknutí nejdřív změní stav checkboxu a CSS potom podle tohoto stavu rozhodne, jestli se modální okno zobrazí, nebo ne.

📘 Shrnutí kapitoly

  • Attribute selector vybírá prvky podle atributu a jeho hodnoty, například input[type="email"].
  • Pseudo-třídy stylují prvek podle stavu nebo podle jeho pozice mezi sourozenci.
  • Transition zapisujeme do výchozího stavu, aby změna probíhala plynule oběma směry.
  • Relative + absolute je nejčastější dvojice pro umístění prvku uvnitř jiného prvku.
  • Sticky se hodí pro navigaci, fixed pro plovoucí prvky a modály.
  • Z-index rozhoduje o tom, co bude nad čím.
  • Pokročilé CSS je hlavně o pochopení vztahů, stavů a logiky chování prvků.
Jak funguje attribute selector

Základní princip je jednoduchý:

input[type="email"] {
  border: 2px solid steelblue;
}
  • input vybírá všechny inputy daného typu elementu.
  • [type="email"] výběr zúží jen na ty inputy, které mají atribut type nastavený na email.

Je důležité chápat, že nejde o „nový druh CSS“, ale o přesnější filtr. Nejdřív říkáš jaký element chceš, a pak pomocí hranatých závorek říkáš s jakou vlastností ho chceš.

Stejnou logikou můžeš vybírat i jiné atributy:

a[target="_blank"] { ... }
img[alt="Profilová fotka"] { ... }
Attribute selector je velmi užitečný tam, kde nestačí běžný tag selector, ale ještě nechceš přidávat class nebo ID.
Proč mají inputy někdy jinou skutečnou velikost

Když dvěma inputům nastavíš stejnou šířku a výšku, neznamená to automaticky, že budou na stránce působit stejně. Do výsledné velikosti totiž vstupuje i padding a border.

Ve výchozím režimu je většina prvků v režimu content-box. To znamená: zadaná šířka patří jen obsahu a rámeček i padding se připočítávají navíc. 

input {
  box-sizing: border-box;
}

Pokud použiješ border-box, prohlížeč počítá padding i border dovnitř prvku. Výsledek je přehlednější a stabilnější.

Prakticky: nastavíš-li width: 260px;, chceš většinou, aby celý input měl 260 px, ne aby 260 px měl jen jeho obsah a zbytek se přičetl navrch.
Proč formulář vycentrujeme pomocí gridu

Tento zápis patří mezi nejčistší způsoby centrování obsahu přesně doprostřed obrazovky:

body {
  min-height: 100vh;
  display: grid;
  place-items: center;
}
  • min-height: 100vh; nastaví minimální výšku body na výšku celé obrazovky.
  • display: grid; zapne gridový režim.
  • place-items: center; vycentruje obsah horizontálně i vertikálně zároveň.

Je dobré studentům říct, že 100vh znamená „100 % výšky viewportu“, tedy viditelné plochy okna.

Výsledek: formulář není nalepený nahoře, ale je přehledně umístěný doprostřed.

Stavové vs strukturální pseudo-třídy

Pseudo-třídy můžeme rozdělit na dvě velké skupiny:

  • stavové – například :hover, :focus, :checked, :active
  • strukturální – například :first-child, :last-child, :nth-child()

Stavové pseudo-třídy reagují na chování uživatele nebo aktuální stav prvku. Strukturální pseudo-třídy reagují na to, kde se prvek nachází mezi ostatními sourozenci.

Právě pseudo-třídy dělají web živější a interaktivnější.
Focus, valid a invalid do detailu

Tyto pseudo-třídy se používají hlavně u formulářů, protože reagují na to, co uživatel právě dělá a jestli zadal správnou hodnotu.

1) :focus

:focus nastává ve chvíli, kdy je prvek aktivní pro psaní nebo ovládání klávesnicí. U inputu to znamená, že v něm bliká kurzor.

K focusu může dojít kliknutím myší, přesunem klávesou Tab nebo programově.

2) :valid a :invalid

:valid a :invalid se odvíjejí od toho, zda hodnota splňuje pravidla daného inputu. U type="email" to znamená, že musí mít podobu platného emailu. 

input:focus { ... }
input:valid { ... }
input:invalid { ... }
Když odstraníš výchozí outline přes outline: none;, měl bys ho nahradit jiným dobře viditelným stylem. Uživatel musí poznat, kde se právě nachází.
Jak přesně funguje :nth-child()

:nth-child() vybírá dítě podle jeho pořadí uvnitř rodiče.

li:nth-child(5)   { ... }   /* pátý prvek */
li:nth-child(even){ ... }   /* sudé prvky */
li:nth-child(odd) { ... }   /* liché prvky */
li:nth-child(3n)  { ... }   /* každý třetí prvek */

To je velmi užitečné například u tabulek, seznamů nebo opakovaných karet, kde chceš pravidelně odlišovat sudé a liché řádky.

Důležité je říct, že :nth-child() nepočítá jen prvky stejného typu, ale skutečné pořadí dítěte uvnitř rodiče.
Co je span, proč nepoužijeme div a jak funguje label
1) Co je span

span je obecný řádkový (inline) prvek. Používá se hlavně tehdy, když chceme stylovat malý kus textu uvnitř řádku. 

<p>Můj oblíbený jazyk je <span>CSS</span>.</p>
2) Proč v checkbox ukázce používáme span

Protože text „Dokončit domácí úkol“ chceme mít jako malý textový prvek vedle checkboxu, ne jako velký blok na novém řádku. 

<label>
  <input type="checkbox">
  <span>Dokončit domácí úkol</span>
</label>
3) Proč ne div

div je blokový (block) prvek. To znamená, že zabírá celý řádek a často začne na novém řádku.

U jednoduchého textu vedle checkboxu je div zbytečně „velký“ prvek. span je přirozenější.
4) Jak funguje label

label je popisek formulářového prvku. Když je input uvnitř labelu, kliknutí na text nebo jeho okolí aktivuje i samotný input. 

<label>
  <input type="checkbox">
  <span>Dokončit domácí úkol</span>
</label>

Výhoda: uživatel nemusí trefovat jen malý checkbox, může kliknout i na text vedle něj.

Proč transition zapisujeme do výchozího stavu
.btn-demo {
  transition: 300ms ease-in-out;
}

.btn-demo:hover {
  background: tomato;
}

Výchozí stav je místo, odkud i kam se prvek vrací. Když transition uložíš sem, animace proběhne plynule oběma směry.

Hover stav má obsahovat jen to, co se má změnit, ne mechanismus animace.

Co přesně transition nastavuje
transition: background-color 300ms ease-in-out;
  • property – která vlastnost se má animovat
  • duration – jak dlouho bude změna trvat
  • timing-function – jak rychle bude animace začínat a končit

Tento zápis se dá číst skoro jako věta: „Barva pozadí se bude měnit 300 milisekund a bude mít jemný rozjezd i zastavení.“

Pokud vlastnost neupřesníš, prohlížeč se pokusí animovat všechny změny, které animovat lze.

Jak funguje ease-in-out a jaké jsou další možnosti

Timing function určuje průběh rychlosti animace. Jinak řečeno: neřeší, co se mění, ale jakým tempem ta změna probíhá.

Co přesně dělá ease-in-out
  • na začátku je animace pomalejší,
  • uprostřed je rychlejší,
  • na konci zase zpomalí.

Působí to přirozeněji než rovnoměrný pohyb, protože v reálném světě se většina věcí nerozbíhá a nezastavuje úplně skokově.

Další běžné možnosti
transition: 300ms linear;
transition: 300ms ease;
transition: 300ms ease-in;
transition: 300ms ease-out;
transition: 300ms ease-in-out;
  • linear – stejná rychlost od začátku do konce
  • ease – jemný začátek i konec, obecně přirozený průběh
  • ease-in – pomalý začátek, pak zrychlení
  • ease-out – rychlý začátek, pak zpomalení
  • ease-in-out – pomalý začátek i konec
Pokročilejší možnosti
transition-timing-function: cubic-bezier(0.4, 0, 0.2, 1);
transition-timing-function: steps(4);
  • cubic-bezier(...) – vlastní křivka rychlosti
  • steps(...) – skoková animace po krocích
Do běžné praxe úplně stačí 150ms–300ms a ease-in-out. Je to rychlé, čitelné a působí to přirozeně.
Typy position do hloubky
  • static – výchozí stav, prvek je v normálním toku dokumentu
  • relative – prvek zůstává v toku, ale můžeš ho posunout vůči jeho původní poloze
  • absolute – prvek se vyjme z běžného toku a umisťuje se vůči nejbližšímu předkovi s nastavenou position
  • fixed – prvek je přichycený k oknu prohlížeče
  • sticky – prvek je nejprve normální, ale při scrollu se přilepí
Metodicky je nejdůležitější pochopit rozdíl mezi relative a absolute, protože ten se v praxi používá nejčastěji.
Proč se kombinuje relative + absolute

Prvek s position: absolute; hledá svůj referenční bod. Tím je nejbližší rodič, který má jinou hodnotu position než výchozí static. 

.parent {
  position: relative;
}

.child {
  position: absolute;
  top: 0;
  right: 0;
}

Bez position: relative; na rodiči by se dítě často umístilo vůči celé stránce, ne vůči boxu, ve kterém ho chceš mít.

Jednoduše: jak funguje relative a absolute

Nejjednodušší představa je tato:

  • rodič = pracovní plocha / místnost / nástěnka,
  • dítě = objekt, který chceme přesně umístit.

Když rodiči dáš position: relative;, říkáš: „Tady je moje souřadnicová plocha.“

Když dítěti dáš position: absolute;, říkáš: „Umísti mě přesně podle okrajů této plochy.“ 

<div class="room">
  <div class="ball"></div>
</div>
.room {
  position: relative;
  width: 300px;
  height: 180px;
  border: 3px solid #334155;
  background: #e2e8f0;
  border-radius: 12px;
}

.ball {
  position: absolute;
  top: 10px;
  right: 10px;
  width: 40px;
  height: 40px;
  background: tomato;
  border-radius: 50%;
}

Význam:

  • top: 10px; = 10 px od horního okraje rodiče
  • right: 10px; = 10 px od pravého okraje rodiče
Zapamatuj si:
relative = „nastavuji prostor, podle kterého se bude měřit“
absolute = „umísti mě přesně uvnitř tohoto prostoru“
Rozdíl mezi position: absolute; a position: relative; na konkrétním příkladu

Představ si kartičku produktu a v jejím rohu malý štítek „NOVÉ“. Samotné absolute nestačí — prvek musí vědět, vůči čemu se má umístit.

❌ Bez position: relative; na rodiči
.card-demo-bad {
  width: 280px;
  padding: 20px;
  border: 1px solid #cbd5e1;
  border-radius: 16px;
  background: white;
}

.badge-demo {
  position: absolute;
  top: 10px;
  right: 10px;
}
Badge často uteče vůči celé stránce, protože kartička není referenční rodič.
✅ S position: relative; na rodiči
.card-demo-good {
  position: relative;
  width: 280px;
  padding: 20px;
  border: 1px solid #cbd5e1;
  border-radius: 16px;
  background: white;
}

.badge-demo {
  position: absolute;
  top: 10px;
  right: 10px;
}
Badge se umístí přesně do rohu kartičky, protože rodič se stal referenční plochou.
Zapamatuj si:
relative = vytvoř referenční prostor
absolute = umísti mě přesně uvnitř tohoto prostoru
Co přesně dělá z-index

Jakmile začneš používat absolute, fixed nebo sticky, prvky se mohou překrývat. z-index určuje, který z nich bude nahoře. 

.overlay { z-index: 100; }
.modal-window { z-index: 200; }

Vyšší číslo znamená vyšší vrstvu. U modálu tedy chceme overlay nad stránkou, ale samotné okno ještě nad overlayem.

Bez správného z-index se může stát, že se modál sice otevře, ale bude schovaný pod jinými prvky.
