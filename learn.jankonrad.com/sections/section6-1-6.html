6.1.6 Finální lekce – spuštění celého informačního systému

V této závěrečné kapitole spojíme všechny předchozí bloky do jednoho celku. Už nejde jen o jednotlivé funkce nebo části serveru – teď si krok za krokem ukážeme, jak se po spuštění backendu načte stránka, jak frontend požádá o data a jak se výsledky propíšou do tabulky v prohlížeči.

Co se naučíš?

  • Spuštění finálního backendu – pochopíš, co se děje při startu programu a proč se volá init_db() a potom run_server().
  • Celý tok dat – projdeš si cestu od databáze až po zobrazení filmů v HTML tabulce.
  • Spolupráci frontendu a backendu – uvidíš, jak se propojí fetch("/api/movies") s routou /api/movies na serveru.
  • Ladění chyb – naučíš se rozpoznat nejčastější problémy, když se stránka nenačte nebo tabulka zůstane prázdná.
Tip: Tohle je chvíle, kdy už neřešíme izolované kusy kódu. Díváme se na projekt jako na jeden propojený systém.

1) Jak se celý systém spustí

Ve finálním backendu se na konci souboru nachází tato spouštěcí část. Právě ta rozhodne, co se stane ve chvíli, kdy soubor opravdu spustíš v Pythonu. :contentReference[oaicite:2]{index=2}

movies_server.py – spouštěcí část 
if __name__ == "__main__":
    init_db()
    run_server()
  1. if __name__ == "__main__": znamená, že tento blok se spustí při přímém spuštění souboru.
  2. init_db() připraví databázi a ověří, že tabulka movies existuje.
  3. run_server() vytvoří server a spustí nekonečné čekání na požadavky.
  • Nejdřív připravíme databázi.
  • Teprve potom pustíme server.
  • To je správné pořadí, protože server má startovat do připraveného prostředí.
2) Co přesně dělá run_server()

Samotné spuštění serveru je ve finálním backendu zapsáno takto: :contentReference[oaicite:3]{index=3}

movies_server.py – run_server() 
def run_server(host="127.0.0.1", port=8000):
    server = HTTPServer((host, port), RequestHandler)
    print(f"Server běží na http://{host}:{port}")
    print("Zastavíš ho: CTRL+C")
    server.serve_forever()
  1. host="127.0.0.1" znamená, že server běží lokálně na tomto počítači.
  2. port=8000 je číslo portu, na kterém server naslouchá.
  3. HTTPServer((host, port), RequestHandler) vytvoří server a propojí ho s naší třídou handleru.
  4. serve_forever() spustí trvalé čekání na požadavky.
  • Když program po spuštění „jen běží a čeká“, není to chyba.
  • Je to správné chování serveru.
  • Server má být připraven, dokud ho ručně nezastavíš.
3) Celý tok dat – od databáze až po tabulku ve stránce

Tohle je nejdůležitější myšlenka celé kapitoly. Když uživatel otevře adresu serveru v prohlížeči, odehraje se ve skutečnosti několik navazujících kroků.

Tok dat v projektu 
1) uživatel otevře http://127.0.0.1:8000
2) server vrátí index.html
3) v index.html se spustí JavaScript
4) JavaScript zavolá /api/movies
5) backend zavolá get_movies()
6) backend pošle JSON
7) frontend vypíše data do tabulky
  1. Uživatel otevře hlavní adresu serveru.
  2. Server rozpozná cestu / a pošle soubor index.html. :contentReference[oaicite:4]{index=4}
  3. Po načtení stránky se spustí JavaScript ve frontendu. :contentReference[oaicite:5]{index=5}
  4. JavaScript pošle požadavek na /api/movies. :contentReference[oaicite:6]{index=6}
  5. Server rozpozná cestu /api/movies a zavolá get_movies(). :contentReference[oaicite:7]{index=7}
  6. Server vrátí data jako JSON. :contentReference[oaicite:8]{index=8}
  7. Frontend JSON přečte a vypíše ho do tabulky. :contentReference[oaicite:9]{index=9}
Pamatuj: frontend si data pouze vyžádá a zobrazí. Samotná data připravuje backend a původně pocházejí z databáze.
4) Co přesně dělá frontend po načtení stránky

Ve frontendu se po načtení stránky spustí tato funkce a na konci se opravdu zavolá loadMovies(). :contentReference[oaicite:10]{index=10}

index.html – načtení filmů 
async function loadMovies(){
  const status = document.getElementById("status");
  const tbody = document.querySelector("#moviesTable tbody");

  try{
    const res = await fetch("/api/movies");
    const movies = await res.json();

    tbody.innerHTML = "";
    for(const m of movies){
      const tr = document.createElement("tr");
      tr.innerHTML = `
        <td>${m.id}</td>
        <td>${m.title}</td>
        <td>${m.year ?? ""}</td>
      `;
      tbody.appendChild(tr);
    }

    status.textContent = movies.length
      ? `Načteno filmů: ${movies.length}`
      : "Databáze je zatím prázdná.";
  }catch(err){
    console.error(err);
    status.textContent = "Chyba: nepodařilo se načíst data. Je spuštěný server?";
  }
}

loadMovies();
  1. Najde se prvek status a tělo tabulky tbody.
  2. Pošle se požadavek na /api/movies.
  3. Odpověď serveru se převede z JSONu na data.
  4. Tabulka se vyčistí a znovu naplní řádky.
  5. Na konci se aktualizuje stavová hláška.
  • Frontend tedy neobsahuje filmy natvrdo v HTML.
  • Filmy se doplní až po komunikaci se serverem.
  • To je velmi důležitý princip moderních webových aplikací.
5) Co přesně udělá backend při volání /api/movies

Ve finálním backendu se cesta /api/movies obsluhuje takto: :contentReference[oaicite:11]{index=11}

movies_server.py – API route 
if path == "/api/movies":
    data = get_movies()
    self._send_json(data)
    return
  1. Server v do_GET() zjistí, že cesta je /api/movies.
  2. Zavolá get_movies(), která načte filmy z databáze.
  3. Výsledek uloží do proměnné data.
  4. self._send_json(data) pošle data jako JSON odpověď.
  • Tady se propojí databáze s webem.
  • Bez této části by fetch("/api/movies") neměl co načíst.
  • To je jádro celého informačního systému.
6) Jak finální projekt správně otestovat

Při finálním testu je důležité neudělat zbytečnou metodickou chybu: nestačí otevřít index.html dvojklikem. Musí běžet server.

  1. Spusť movies_server.py v Pythonu.
  2. Počkej, až se v konzoli objeví adresa serveru.
  3. Otevři v prohlížeči http://127.0.0.1:8000.
  4. Stránka se načte a JavaScript automaticky požádá o data.
  5. Když je v databázi obsah, objeví se v tabulce.
  6. Když je databáze prázdná, uvidíš stavovou hlášku o prázdné databázi. :contentReference[oaicite:12]{index=12}
  • Tohle je správný způsob testování webového backendu.
  • Backend i frontend musí běžet jako jeden celek.
  • Teprve pak se ukáže, že projekt opravdu funguje end-to-end.
7) Nejčastější chyby a jak je poznat
7.1 Stránka se vůbec nenačte

Nejčastější příčina: server neběží nebo jsi otevřel špatnou adresu.

  • zkontroluj, že je v konzoli spuštěný movies_server.py,
  • zkontroluj adresu http://127.0.0.1:8000,
  • zkontroluj, že port odpovídá tomu, co vypisuje server. :contentReference[oaicite:13]{index=13}
7.2 Stránka se načte, ale tabulka je prázdná

Tady jsou dvě hlavní možnosti:

  • databáze opravdu neobsahuje žádné filmy,
  • API cesta /api/movies nevrací data správně.

Když je databáze prázdná, frontend podle připraveného kódu vypíše zprávu Databáze je zatím prázdná. :contentReference[oaicite:14]{index=14}

7.3 Ve stránce se objeví chyba načítání

Ve frontendu je připraven i catch blok, který při chybě vypíše zprávu o nepodařeném načtení dat. :contentReference[oaicite:15]{index=15}

  • zkontroluj, jestli server opravdu běží,
  • zkontroluj, jestli backend obsahuje cestu /api/movies,
  • zkontroluj konzoli prohlížeče i konzoli Pythonu.
7.4 Frontend používá status, ale prvek chybí

Ve vašem aktuálním index.html se v JavaScriptu používá document.getElementById("status"), ale v ukázce HTML není žádný odpovídající prvek vidět. :contentReference[oaicite:16]{index=16}

  • do stránky je potřeba doplnit třeba <p id="status">Načítám data...</p>,
  • jinak bude proměnná status prázdná,
  • a při zápisu status.textContent = ... vznikne chyba.
Doplnění do index.html 
<p id="status">Načítám data...</p>
8) Co si z celé kapitoly odnést

Tahle závěrečná lekce není hlavně o novém příkazu. Je o tom, že už umíš vidět projekt jako jeden propojený celek.

  1. Databáze ukládá filmy.
  2. Backend je načte.
  3. Server je nabídne přes API.
  4. Frontend si je vyžádá.
  5. JavaScript je vykreslí do tabulky.
  • Tohle je skutečný jednoduchý informační systém.
  • Nejde už jen o samostatný Python skript.
  • Jde o spolupráci více vrstev najednou.
Pamatuj: hodnotou takového projektu není jen to, že „něco běží“. Hodnotou je, že rozumíš, jak spolu jednotlivé části komunikují.

Proč je pořadí init_db() → run_server() správně

Server by měl startovat až tehdy, když je jeho prostředí připravené.

  • init_db() připraví databázi a tabulku,
  • run_server() potom spustí obsluhu požadavků,
  • díky tomu server nezačne běžet „napůl připravený“.

Co znamená, že projekt funguje end-to-end

End-to-end znamená, že funguje celý řetězec od vstupu až po výsledek.

  • nejen databáze,
  • nejen server,
  • nejen HTML stránka,
  • ale všechno dohromady jako jeden celek.

Jak postupovat při ladění

Když něco nefunguje, nehádej. Postupuj systematicky:

  • běží server?
  • otevřel jsem správnou adresu?
  • vrací /api/movies data?
  • má HTML všechny potřebné prvky?
  • není chyba v konzoli prohlížeče nebo Pythonu?
