3.3.5 Ternární operátor (?:) a logika rozhodování – podrobný výklad + kvíz

Co se naučíš?

Tady už kód začíná „přemýšlet“. Zopakujeme a zafixujeme rozhodování pomocí porovnání a logických operátorů a přidáme ternární operátor – krátký zápis pro jednoduché if/else.

  • Porovnání: výsledkem je vždy boolean (true/false).
  • Logické operátory: &&, ||, ! a jejich čtení.
  • Pořadí vyhodnocování: ! → && → || (a proč je to důležité).
  • Ternární operátor: podmínka ? hodnota1 : hodnota2.
  • Short-circuit: výraz se zastaví ve chvíli, kdy už je výsledek jistý.
Tip: Vždy si u podmínky řekni: „Co je otázka?“ a „Jaká je odpověď – true/false?“. U dlouhých výrazů si klidně napiš pomocné proměnné.
Než začneme: pořadí operátorů (precedence) a short-circuit

U logických výrazů se často plete, co se vyhodnotí první. V JavaScriptu (a ve většině jazyků) platí: nejdřív NOT, pak AND, potom OR.

Rychlý přehled: precedence
Operátor Čte se Priorita Příklad
! NOT (obrátí boolean) nejvyšší !false → true
&& AND (obě podmínky) střední true && false → false
|| OR (stačí jedna) nižší false || true → true
Short-circuit: JavaScript se často zastaví dřív:
  • false && (cokoliv) → hned false (druhá část se ani nemusí vyhodnotit)
  • true || (cokoliv) → hned true
Ternární operátor (?:) – kratší zápis pro jednoduché if/else

Ternární operátor je zkratka pro situace, kdy chceš vybrat jednu ze dvou hodnot. Typicky: „Pokud platí podmínka, použij A, jinak použij B.“

Klasický zápis if / else
let number = 10;
let message;

if (number > 0) {
  message = "Kladné číslo";
} else {
  message = "Není kladné";
}

console.log(message);
Stejné řešení ternárním operátorem
let number = 10;

let message = (number > 0)
  ? "Kladné číslo"
  : "Není kladné";

console.log(message);

Jak se ternární operátor čte?

Obecný tvar: 

podmínka ? hodnotaKdyžTrue : hodnotaKdyžFalse
  • Podmínka → otázka, která vrátí true/false.
  • ? → „pokud je to pravda…“
  • : → „jinak…“
  • Výsledek → celý výraz vrátí jednu ze dvou hodnot.
Kdy ho používat: když vybíráš mezi dvěma hodnotami a vejde se to přehledně do jednoho řádku.
Kdy radši ne: když je logika složitá, má víc kroků, nebo by ternary udělalo kód nepřehledný.
🧩 Cvičení: Můžeš řídit? (ternární operátor)

Máš proměnnou age. Pomocí ternárního operátoru vypiš do konzole: „Můžeš řídit.“ pokud je age >= 18, jinak „Jsi moc mladý/á na řízení.“

Zadání

  • Vstup: let age = 17; (zkoušej různé hodnoty)
  • Podmínka: age >= 18
  • Výstup: console.log(...) s ternárním výrazem
Tip: Nejprve si to klidně napiš jako if/else a pak to přepiš do ternary. 
let age = 17;

// sem napiš svůj ternární operátor

Ternární operátor má tvar:

condition ? valueIfTrue : valueIfFalse
Hotové řešení
let age = 17;

console.log(age >= 18 ? "Můžeš řídit." : "Jsi moc mladý/á na řízení.");
Pro kontrolu: ekvivalent v if/else
if (age >= 18) {
  console.log("Můžeš řídit.");
} else {
  console.log("Jsi moc mladý/á na řízení.");
}
Kvíz: Porovnání, if, logické operátory (+ short-circuit)

Klikni na odpověď. Po volbě se možnosti uzamknou. Tlačítko „Vysvětlit podrobněji“ se má objevit až po odpovědi (dle tvé logiky kvízu).

1. Jaký bude výsledek porovnání 5 === "5"?

2. Jaké klíčové slovo spustí kód na základě podmínky, která musí být pravdivá?

3. Co udělá tento kód?

let age = 20;

if (age > 18) {
  console.log("You can enter");
}

4. Co dělá operátor &&?

5. Co vypíše tento kód?

let loggedIn = true;
let isAdmin = false;

console.log(loggedIn && isAdmin);

6. Co bude výsledkem tohoto kódu?

let discount = 0;
let isMember = true;

if (isMember) {
  discount = 10;
}

console.log(discount);

7. HARD: Jaký je výsledek tohoto výrazu, když isStudent je na začátku false?

true && isStudent || false || !isStudent
Otázka 1 – Proč 5 === "5" vyjde false?

=== je strict equality – kontroluje:

  • hodnotu (value)
  • datový typ (type)

Číslo 5 (number) není totéž co "5" (string), takže: 5 === "5" → false.

Pokud bys použil ==, JavaScript by zkusil typy převést (coercion) a 5 == "5" by vyšlo true. Proto se v praxi často preferuje ===.

Otázka 2 – K čemu je if?

if je základní konstrukce pro rozhodování:

if (podminka) {
  // provede se jen když je podminka true
}

Podmínka je obvykle porovnání (např. age >= 18), které vrací boolean.

Otázka 3 – Proč se vypíše "You can enter"?

Máme age = 20. Podmínka je age > 18.

  • 20 > 18 → true
  • kód uvnitř bloku if se tedy provede

Pokud bys změnil age na 18 nebo 17, podmínka by byla false a nic by se nevypsalo.

Otázka 4 – Jak funguje && (AND)?

&& vrátí true jen tehdy, když jsou obě strany true.

ABA && B
truetruetrue
truefalsefalse
falsetruefalse
falsefalsefalse
Short-circuit: když je levá strana false, pravá se často ani nevyhodnotí.
Otázka 5 – Proč loggedIn && isAdmin vyjde false?

Máme:

  • loggedIn = true
  • isAdmin = false

Výraz loggedIn && isAdmin je tedy true && false → false.

AND vyžaduje pravdu na obou stranách. Stačí jedna false a celý výraz je false.

Otázka 6 – Proč je výstup 10?

Nejdřív je discount = 0. Pak:

  • isMember je true, takže se provede blok if.
  • Uvnitř se discount nastaví na 10.
  • Na konci se vypíše aktuální hodnota discount, tedy 10.

Kdyby bylo isMember = false, blok by se přeskočil a výstup by zůstal 0.

Otázka 7 – Detailní rozbor (HARD)

Volba false je špatně, protože výraz se vyhodnotí na true. Když je isStudent = false, rozložme to krok po kroku:

  1. true && isStudent → true && false → false
  2. Dál máme: false || false → stále false
  3. Nakonec: !isStudent → !false → true
  4. Celý výraz tedy: false || false || true → true
Klíčové je pořadí operátorů: ! se vyhodnotí před && a to před ||. A taky platí short-circuit – výraz se zastaví, jakmile je výsledek jistý.

Tohle je přesně ten typ příkladu, kde pomáhá dělat si „mezivýsledky“ a postupně to zjednodušovat.

Tahák: Shrnutí klíčových pojmů

Tady už kód opravdu rozhoduje a reaguje. Tohle jsou nejdůležitější věci, které chceš mít v hlavě, když píšeš logiku a podmínky.

1) Porovnávací operátory

Slouží k porovnání dvou hodnot. Výsledek je vždy boolean (true/false):

  • === – shodná hodnota a typ (doporučeno)
  • !== – neshodná hodnota nebo typ
  • >, < – větší / menší
  • >=, <= – větší rovno / menší rovno
2) Logické operátory
  • && (AND) – true jen když jsou obě strany true
  • || (OR) – true když je aspoň jedna strana true
  • ! (NOT) – obrátí boolean: !true → false, !false → true
3) if / else / else if
if (condition) {
  // když je condition true
} else if (otherCondition) {
  // když první neplatí, ale tahle ano
} else {
  // když neplatí žádná z podmínek
}
4) Vnořený if (nested if)

if uvnitř if – užitečné, když druhá podmínka dává smysl až po splnění první:

if (isLoggedIn) {
  if (isAdmin) {
    console.log("Admin panel");
  }
}
5) Short-circuit (zastavení vyhodnocování)
  • false && cokoliv → hned false (druhá část se nemusí řešit)
  • true || cokoliv → hned true
Pořadí: ! se vyhodnocuje před && a to před ||. Když si nejsi jistý/á, použij závorky.
6) Ternární operátor (?:)

Zkratka pro jednoduché „if/else“ při výběru ze dvou hodnot:

console.log(age >= 18 ? "Můžeš řídit." : "Jsi moc mladý/á na řízení.");
