6.1.3 Backend – načtení filmů z databáze pro web

Databázi už znáš z předchozích lekcí. Teď ale data nebudeme číst kvůli výpisu přes print() do konzole, ale kvůli tomu, aby je backend mohl poslat webové stránce. Právě k tomu slouží funkce get_movies().

Co se naučíš?

  • Funkci get_movies() – pochopíš, jak backend načte data z databáze a vrátí je dál.
  • Rozdíl mezi řádky z DB a daty pro web – uvidíš, proč nestačí jen fetchall(), ale proč data ještě převádíme.
  • Seznam slovníků – zjistíš, proč je tento tvar vhodný pro JSON a JavaScript.
  • Novou roli backendu – backend už nebude jen „pracovat s databází“, ale začne fungovat jako prostředník pro web.
Tip: SQL dotaz SELECT už znáš. V této lekci je nové hlavně to, že data načítáme pro webovou stránku, ne pro konzolový výpis.
Otevřít backendový soubor Otevřít frontend
1) Proč nestačí jen vypsat filmy do konzole

V předchozích databázových lekcích jsme data načetli a pak je třeba vypsali přes print(). To je v pořádku pro konzolovou aplikaci. Jenže webová stránka tento výpis nevidí. Web potřebuje, aby backend data vrátil v použitelné podobě, ne aby je jen vypsal do terminálu.

  1. Backend načte řádky z databáze.
  2. Řádky upraví do přehlednějšího tvaru.
  3. Funkce je vrátí jako výsledek.
  4. Server je později pošle do frontendu jako JSON.
  • Konzole je pro programátora.
  • JSON a frontend jsou pro webovou aplikaci.
  • Proto potřebujeme funkci, která data připraví pro další vrstvu systému.
2) Celý blok kódu – funkce get_movies()

V našem backendu je načítání filmů řešené takto: 

def get_movies():
    conn = connect()
    rows = conn.execute("SELECT id, title, year FROM movies ORDER BY id DESC;").fetchall()
    conn.close()

    # převedeme na seznam slovníků → snadné pro JSON
    movies = []
    for (movie_id, title, year) in rows:
        movies.append({
            "id": movie_id,
            "title": title,
            "year": year
        })

    return movies

Celá funkce vypadá delší než connect(), ale ve skutečnosti se skládá jen ze čtyř jednoduchých částí: připojit se, načíst řádky, převést je do vhodného tvaru a vrátit výsledek.

3) První část – připojení a SQL dotaz

Úvod funkce využívá to, co už znáš z předchozí kapitoly: 

conn = connect()
rows = conn.execute("SELECT id, title, year FROM movies ORDER BY id DESC;").fetchall()
conn.close()
  1. conn = connect() otevře spojení s databází.
  2. conn.execute(...) pošle databázi SQL dotaz.
  3. SELECT id, title, year FROM movies vybere tři sloupce z tabulky movies.
  4. ORDER BY id DESC seřadí výsledky od nejvyššího ID dolů.
  5. fetchall() vezme všechny nalezené řádky.
  6. conn.close() spojení zase zavře.
  • Tato část není nová databázová magie – je to navázání na předchozí SQLite lekce.
  • Nové bude až to, co s výsledkem uděláme potom.
  • Proměnná rows teď ještě není připravená přímo pro web.
4) Co je v proměnné rows

Po fetchall() dostaneme seznam databázových řádků. Každý řádek je n-tice (tuple), například takto: 

[
    (3, "Interstellar", 2014),
    (2, "Avatar", 2009),
    (1, "Titanic", 1997)
]
  1. Celá vnější hranatá závorka znamená seznam více řádků.
  2. Každý vnitřní prvek je jeden řádek z databáze.
  3. Pořadí odpovídá sloupcům v SQL dotazu: id, title, year.
  • Pro Python je to naprosto použitelné.
  • Pro web ale chceme přehlednější tvar s pojmenovanými klíči.
  • Proto následuje převod na slovníky.
5) Druhá část – převod řádků na seznam slovníků

Tohle je nejdůležitější nová část celé lekce: 

movies = []
for (movie_id, title, year) in rows:
    movies.append({
        "id": movie_id,
        "title": title,
        "year": year
    })

Tento blok znamená: „vezmi každý řádek z databáze a převeď ho na slovník se srozumitelnými názvy klíčů“.

  1. movies = [] vytvoří prázdný seznam.
  2. for (movie_id, title, year) in rows: projde všechny řádky z databáze.
  3. Python tuple rovnou rozbalí do tří proměnných.
  4. movies.append(...) přidá do seznamu nový prvek.
  5. Tím novým prvkem je slovník s klíči "id", "title" a "year".
  • Výsledek už není seznam tuple, ale seznam slovníků.
  • To je mnohem čitelnější pro JSON i JavaScript.
  • Frontend pak může snadno psát třeba m.title nebo m.year.
6) Poslední řádek – return movies

Na konci funkce stojí tento řádek: 

return movies
  1. return znamená „vrať výsledek funkce“.
  2. movies je hotový seznam slovníků.
  3. Kdokoli později zavolá get_movies(), dostane tato data jako výsledek.
  • Funkce nic nevypisuje do konzole.
  • Funkce jen připraví a vrátí data.
  • To je přesně to, co backend pro web potřebuje.
Pamatuj: konzolová aplikace často něco vypisuje. Serverové funkce ale často spíš vracejí data další části programu.
7) Proč tento tvar dat vyhovuje frontendu

Ve frontendu se při vypisování filmů používají vlastnosti m.id, m.title a m.year. To znamená, že JavaScript očekává objekt s pojmenovanými položkami, ne tuple s číselnými pozicemi. :contentReference[oaicite:2]{index=2} 

for(const m of movies){
  const tr = document.createElement("tr");
  tr.innerHTML = `
    <td>${m.id}</td>
    <td>${m.title}</td>
    <td>${m.year ?? ""}</td>
  `;
}
  1. JavaScript bere jeden film jako objekt m.
  2. Uvnitř používá klíče id, title a year.
  3. Právě proto backend převádí databázové řádky na slovníky se stejnými názvy.
  • Backend a frontend si tak „rozumí“.
  • Tohle je velmi důležitá myšlenka v informačních systémech.
  • Jedna vrstva připraví data tak, aby je druhá vrstva snadno převzala.
8) Co máš mít po této kapitole hotové

Po této kapitole už backend umí načíst seznam filmů z databáze a připravit ho ve tvaru, který se hodí pro JSON a pozdější poslání do webu.

  1. Máš funkci get_movies().
  2. Ta otevře databázi.
  3. Načte filmy přes SELECT.
  4. Převede je na seznam slovníků.
  5. Vrátí výsledek přes return.
  • Ještě neposíláme HTTP odpověď.
  • Ještě neběží samotný server.
  • Ale datová část pro API už je připravená.

Shrnutí

  • Žák chápe, že get_movies() nečte data pro konzoli, ale pro webovou vrstvu backendu.
  • Žák rozumí tomu, že fetchall() vrací seznam databázových řádků.
  • Žák ví, proč se řádky převádějí na seznam slovníků.
  • Žák chápe, že klíče id, title a year jsou připravené tak, aby je frontend snadno použil.

Proč už nestačí print()

print() vypisuje text do konzole. To je dobré pro programátora nebo pro konzolovou aplikaci. Webová stránka ale potřebuje data dostat jako hodnotu, kterou backend pošle dál.

  • Konzole = výstup pro člověka v terminálu.
  • Vrácená data = vstup pro další část programu.
  • Serverové funkce proto často vracejí, místo aby vypisovaly.

Jak číst funkci get_movies() jako celek

Celou funkci můžeš číst takto:

  • připoj se k databázi,
  • načti všechny filmy,
  • převeď řádky na slovníky,
  • vrať hotový seznam.

To je přesně úloha této funkce. Neposílá ještě HTTP odpověď, jen připraví data.

Řádky z databáze vs. data pro web

Databáze vrací výsledky jako řádky v určitém pořadí. To je výborné pro Python a SQL, ale pro web se lépe pracuje s pojmenovanými klíči.

  • (1, "Titanic", 1997) = tuple podle pozice
  • { "id": 1, "title": "Titanic", "year": 1997 } = objekt podle názvu

Druhý tvar je přirozenější pro JSON a JavaScript.

Proč právě seznam slovníků

Slovník v Pythonu velmi dobře odpovídá objektu v JSONu.

  • Python: {"id": 1, "title": "Titanic", "year": 1997}
  • JSON: { "id": 1, "title": "Titanic", "year": 1997 }

Proto je převod na slovníky praktický mezikrok mezi databází a webem.
