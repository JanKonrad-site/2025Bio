3.3.3 Porovnávací operátory a podmínky (if / else / else if)

Co se naučíš?

V této kapitole si vysvětlíme, jak v JavaScriptu porovnávat hodnoty (např. „je 5 větší než 7?“) a jak na základě výsledku rozhodovat pomocí if, else a else if.

  • Porovnávací operátory: >, <, >=, <=, ==, !=
  • Boolean výsledek: porovnání vždy vrací true nebo false
  • Striktní porovnání: proč je bezpečnější používat === a !==
  • Podmínky: jak funguje if, else, else if
  • Časté pasti: „divné“ výsledky kvůli automatické konverzi typů
Tip: V reálném kódu většinou používej === a !==. Vyhneš se překvapením z automatické konverze typů.
Porovnávací operátory (comparison operators)

Porovnávací operátory umožňují porovnat dvě hodnoty. Výsledek porovnání je vždy boolean – tedy true nebo false.

Operátor Význam Příklad Výsledek
> větší než 5 > 7 false
< menší než 5 < 7 true
>= větší nebo rovno 5 >= 5 true
<= menší nebo rovno 4 <= 5 true
== rovnost (může převádět typy) 5 == "5" true (pozor)
=== striktní rovnost (bez převodu typů) 5 === "5" false
!= nerovnost (může převádět typy) 0 != false false (pozor)
!== striktní nerovnost 0 !== false true
Pozor: JavaScript je „loosely typed“ – některé operátory jako == a != umí hodnoty automaticky převádět (coercion). To může vést k nečekaným výsledkům (např. false == 0 je true).
Podmínky – if, else, else if

Podmínky umožňují programu rozhodovat se. Když je podmínka pravdivá (true), vykoná se blok kódu v { }. Když je nepravdivá (false), blok se přeskočí.

Základní syntaxe
let age = 19;

if (age > 20) {
  console.log("Věk je větší než 20");
} else {
  console.log("Věk je 20 nebo méně");
}

else if použiješ, když chceš otestovat více možností (např. různé jazyky, různé úrovně věku apod.).

Cvičení: Kalkulačka slevy pro e-shop

Představ si, že programuješ pokladnu e-shopu. Pokud je nákup za 100 $ nebo více, zákazník dostane 30% slevu.

Zadání

  • Vytvoř proměnnou totalPrice (celková cena nákupu).
  • Zkontroluj, jestli je totalPrice dost na slevu (≥ 100).
  • Pokud ano: spočítej cenu po slevě (30% dolů) a vypiš ji do konzole.
  • Pokud ne: spočítej, kolik chybí do 100, a vypiš hlášku typu: „Utrať ještě 15 $ a získáš 30% slevu!“
let totalPrice = 85; // změň hodnotu a testuj různé případy

if (totalPrice >= 100) {
  let discountedPrice = totalPrice * 0.7; // 30% sleva = platíš 70%
  console.log(discountedPrice);
} else {
  let amountNeeded = 100 - totalPrice;
  console.log(`Utrať ještě $${amountNeeded} a získáš 30% slevu!`);
}

✅ Rychlý kvíz: porovnání a podmínky

1. Co dělá operátor >?

Příklad: 10 > 7 → true, ale 5 > 7 → false.

2. Co vypíše tento kód?

let x = 5;
console.log(x < 7);

Pozor na uvozovky: true je boolean, ale "true" je string.

3. Jaký bude výsledek porovnání false == 0?

Právě tohle je důvod, proč se často doporučuje ===. U false === 0 by výsledek byl false (typy se neshodují).

4. Co testuje operátor !=?

Striktní varianta je !== (kontroluje i typ).

5. Jaký datový typ vždy vrací porovnání?

Boolean znamená, že existují jen dvě hodnoty: true a false.

6. Jaký je účel příkazu if v programování?

if řídí tok programu: když je podmínka true, kód se provede; jinak ne.

7. Co se stane, když je podmínka v if nepravdivá (false)?

Pokud za if následuje else, provede se právě ten.

8. Co vypíše tento kód?

let age = 15;

if (age > 18) {
  console.log("Dospělý");
}

Kdyby bylo age = 19, podmínka by byla true a vypsalo by se "Dospělý".

Shrnutí: tahák (cheat sheet)

Proměnné
  • let – proměnná se může měnit: let name = "Anna";
  • const – hodnota se nemá měnit: const PI = 3.14;
  • var existuje, ale je zastaralé – používej hlavně let/const.
Datové typy
  • String – text v uvozovkách: "hello"
  • Number – číslo: 31, 3.14
  • Boolean – true / false
  • undefined – hodnota nebyla nastavena
  • null – „záměrně prázdná“ hodnota
  • typeof – zjistí typ: typeof value
Operátory
  • Matematické: + - * / % (zbytek po dělení) ** (mocnina)
  • Přiřazovací: =, zkratky: +=, -=, *=, /=, %=
  • Ink/Decr (v JS): ++, --
Převody typů (type conversion / coercion)
  • Implicitní (coercion): JavaScript někdy typ převede sám (např. "10" - 2 → 8).
  • Explicitní: převedu sám: Number("123"), String(123), Boolean(1)
  • Trik: +"5" převede string na number.
Porovnání a podmínky
  • Porovnání vždy vrací boolean.
  • Preferuj === a !== (hodnota + typ).
  • if spustí blok jen když je podmínka true; else je „záložní“ větev; else if řeší více možností.

== vs === (proč je to důležité)

Operátor == (volná rovnost) může provádět automatické převody typů. To občas vede k „překvapivým“ výsledkům.

Příklady
5 == "5"            // true  (string se převede na číslo)
5 === "5"           // false (typy se liší)

false == 0          // true
false === 0         // false

undefined == null   // true
undefined === null  // false

Doporučení: Pokud nevíš proč použít ==, použij ===.

Otázka 1 – Operátor >

Porovnání vždy vrací boolean (tedy true nebo false).

Příklady:

  • 10 > 3 → true
  • 2 > 9 → false
Otázka 2 – Výstup porovnání

x je 5, takže x < 7 je pravda → vypíše se true.

Pozor: true je boolean, ale "true" je řetězec (text).

Otázka 3 – Proč false == 0 vyjde true?

Operátor == dělá typovou konverzi (coercion).

false se při == převede na 0, takže:

  • false == 0 → true

Proto je v praxi bezpečnější používat ===:

  • false === 0 → false (lišší se typ: boolean vs number)
Otázka 4 – Operátor !=

!= vrací true, když se hodnoty nerovnají.

Pozor: stejně jako ==!==.

  • 5 != "5" → false (uvolněné porovnání po konverzi)
  • 5 !== "5" → true (lišší se typy)
Otázka 5 – Co vrací porovnání?

Jakékoli porovnání vrací vždy boolean:

  • 10 > 5 → true
  • 10 === "10" → false
Otázka 6 – K čemu je if?

if řídí tok programu: kód se provede jen tehdy, když je podmínka true.

Použití: věk uživatele, přihlášení, cena v košíku, stav hry…

Otázka 7 – Co když je podmínka false?

Když je podmínka false, blok v if se přeskočí.

Pokud chceš řešit i opačný případ, použij else:

if (podminka) {
  // když true
} else {
  // když false
}
Otázka 8 – Proč se nic nevypíše?

Při age = 15 je podmínka age > 18 nepravdivá, takže se blok neprovede.

Kdyby bylo age = 19, vypsalo by se do konzole Adult.

Shrnutí kapitoly

  • Porovnání vždy vrací true nebo false (boolean).
  • Základní operátory: >, <, >=, <=, ==, !=.
  • Bezpečnější varianta: používej === a !== (hlídají i datový typ).
  • if: kód se spustí jen když je podmínka true.
  • else: spustí se, když je podmínka v if false.
  • else if: když máš více možností, testuješ je postupně – jakmile jedna vyjde, zbytek se neřeší.
Tip: Když chceš „více větví“ rozhodování, je else if přehlednější a efektivnější než několik samostatných if pod sebou.
