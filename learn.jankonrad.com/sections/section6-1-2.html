6.1.2 Backend, server, API a virtuální prostředí – jak připravit projekt na webovou část

V minulé části vznikla databáze reservations.db a konzolový program, který do ní umí zapisovat rezervace. Teď je potřeba pochopit, jak tento projekt posunout dál: jak z lokálního Python skriptu vznikne backend, proč potřebujeme server, co je API, proč budeme používat FastAPI a jakou roli v tom hraje virtuální prostředí. V další lekci už na tuto teorii naváže skutečný backend v Pythonu.

Co se v této lekci naučíš

  • Co je frontend, backend, server a databáze.
  • Co je API a proč bez něj webová stránka přímo nepracuje s databází.
  • Proč je pro tento projekt vhodné FastAPI.
  • Že FastAPI není součást základní instalace Pythonu a musí se doinstalovat jako balíček.
  • Co je virtuální prostředí, jak funguje a proč se používá pro konkrétní projekt.
  • Jak vypadá nejmenší možný backend ve FastAPI.
  • Jak bude propojená konzole, databáze, backend a webová stránka v našem rezervačním systému.
Cíl kapitoly:
Přesně pochopit, kde backend v projektu sedí, proč vzniká, jak komunikuje s databází a proč potřebuje vlastní balíčky a vlastní prostředí.

Na co tato kapitola navazuje

  1. Existuje databáze reservations.db.
  2. Existuje Python program, který do této databáze zapisuje rezervace přes konzoli.
  3. Další krok je zobrazit tato data i na webové stránce.
  4. Webová stránka ale nebude pracovat s databází přímo.
  5. Mezi stránku a databázi proto vstoupí backend, který poběží jako server.
Jedna věta, kterou si zapamatuj:
Databáze data ukládá, backend je čte nebo zapisuje a webová stránka je zobrazuje.
Krok 1: kde v projektu sedí frontend, backend, server a databáze

Webová aplikace se obvykle neskládá z jediné části. Každá část má jiný úkol a teprve dohromady tvoří celý systém.

  • Frontend = to, co se zobrazuje v prohlížeči (HTML, CSS, JavaScript).
  • Backend = Python program, který zpracovává logiku aplikace.
  • Databáze = místo, kde jsou uložená data.
  • Server = běžící program, který čeká na požadavky a vrací odpovědi.
V našem projektu:
Frontend = stránka s výpisem rezervací
Backend = Python program využívající FastAPI
Databáze = reservations.db
Co znamenají tyto části jednoduše
  • Frontend je viditelná část aplikace.
  • Backend rozhoduje, jaká data se načtou, uloží nebo vrátí.
  • Databáze slouží jako uložiště.
  • Server je způsob, jak backend běží a komunikuje přes web.
Krok 2: co je API a proč ho projekt potřebuje

API je rozhraní, přes které spolu komunikují programy. V našem případě to znamená, že webová stránka pošle backendu požadavek a backend vrátí odpověď.

  • Stránka požádá o data.
  • Backend otevře databázi nebo zpracuje požadovanou akci.
  • Backend vrátí výsledek.
  • Stránka výsledek zobrazí.
Důležité:
Prohlížeč běžně nepracuje přímo se souborem SQLite na disku. Proto je mezi webovou stránkou a databází potřeba backend.
Proč webová stránka nepůjde přímo do databáze
  • SQLite databáze je soubor na disku.
  • Prohlížeč neumí používat Python modul sqlite3.
  • Práce s databází musí být řízená backendem.
  • Backend tedy slouží jako prostředník mezi stránkou a databází.
Krok 3: proč budeme používat právě FastAPI

V Pythonu existuje více frameworků pro tvorbu backendu. Pro tento projekt budeme používat FastAPI, protože je přehledné, krátké a dobře se na něm vysvětluje princip webového backendu.

  • Kód je stručný a dobře čitelný.
  • Rychle vznikne první funkční server.
  • Snadno se v něm vysvětlují cesty jako / nebo /reservations.
  • Dobře zapadá do Python projektu se SQLite databází.
  • Je vhodné pro malé ukázky i větší aplikace.
Nejdůležitější princip:
URL adresa → Python funkce → odpověď serveru
Krok 4: FastAPI není součást Pythonu – musíme ho doinstalovat

V této části projektu se poprvé objevuje něco, co základní Python sám o sobě neobsahuje.

  • sqlite3 = modul, který bývá součástí standardní instalace Pythonu.
  • FastAPI = externí balíček, který je potřeba do projektu doinstalovat.
Důležité:
Řádek from fastapi import FastAPI bude fungovat až tehdy, když je FastAPI v daném prostředí skutečně nainstalované.
Balíček, modul a import
  • Balíček = něco, co se instaluje přes pip.
  • Modul = něco, co se pak v Pythonu načítá přes import.
  • pip install ... = instalace do prostředí.
  • import ... = použití v kódu.
Krok 5: virtuální prostředí – proč existuje a co dělá

Virtuální prostředí je oddělené prostředí pro konkrétní Python projekt. Obsahuje vlastní sadu nainstalovaných balíčků a je navázané právě na danou složku projektu.

  • Každý projekt může mít své vlastní balíčky.
  • Různé projekty si nepřepisují verze knihoven navzájem.
  • Balíčky instalované do jednoho projektu se automaticky nepletou do jiného.
  • Backend tak přesně ví, z jakého prostředí má brát své závislosti.
Spojitost s projektem:
Virtuální prostředí patří ke konkrétní složce projektu. Když se v této složce aktivuje, Python i terminál používají balíčky nainstalované právě pro tento projekt.
Co se v něm ve skutečnosti děje
  • Vytvoří se speciální složka, často pojmenovaná .venv.
  • Uvnitř je vlastní kopie nebo odkaz na Python interpreter a místo pro balíčky.
  • Po aktivaci prostředí se příkazy jako python nebo pip vztahují právě k tomuto prostředí.
  • Tím se oddělí balíčky tohoto projektu od ostatních projektů i od globální instalace Pythonu.
Proč je virtuální prostředí důležité právě pro backend
  • Backend obvykle používá externí balíčky, například FastAPI.
  • Tyto balíčky je potřeba mít nainstalované správně a předvídatelně.
  • Virtuální prostředí zajišťuje, že backend běží nad přesně těmi balíčky, které tento projekt potřebuje.
  • Když se projekt později přesune na jiný počítač nebo server, je jasné, jaké závislosti potřebuje.
Krok 6: vytvoření prostředí a instalace FastAPI

Nejprve se vytvoří virtuální prostředí, pak se aktivuje a teprve potom se do něj nainstaluje FastAPI.

Windows – vytvoření prostředí a instalace 
python -m venv .venv
.venv\Scripts\activate
python -m pip install "fastapi[standard]"
macOS / Linux – vytvoření prostředí a instalace 
python -m venv .venv
source .venv/bin/activate
python -m pip install "fastapi[standard]"
Vysvětlení příkazů „slovo od slova“
  • python -m venv .venv = vytvoř virtuální prostředí ve složce .venv.
  • .venv\Scripts\activate = ve Windows aktivuj toto prostředí.
  • source .venv/bin/activate = na macOS/Linux aktivuj toto prostředí.
  • python -m pip install "fastapi[standard]" = nainstaluj FastAPI do právě aktivního prostředí.
Co znamená „do aktivního prostředí“:
Balíček se neinstaluje „někam obecně do počítače“, ale právě do prostředí, které je v danou chvíli aktivované pro tento projekt.
ověření instalace 
python -m pip show fastapi
Krok 7: nejmenší možný backend ve FastAPI
main.py 
from fastapi import FastAPI

app = FastAPI()

@app.get("/")
def read_root():
    return {"message": "Ahoj ze serveru"}
Vysvětlení kódu „slovo od slova“
  • from fastapi import FastAPI = importujeme třídu FastAPI z nainstalovaného balíčku.
  • app = FastAPI() = vytvoříme objekt aplikace.
  • @app.get("/") = když přijde požadavek na adresu /, spustí se následující funkce.
  • def read_root(): = Python funkce obsluhující tuto adresu.
  • return {"message": "Ahoj ze serveru"} = server vrátí data ve formátu JSON.
Co si z tohoto bloku odnést
  • Backend je stále Python kód.
  • Jednotlivé adresy na serveru odpovídají Python funkcím.
  • V této chvíli se ještě nečte databáze, vrací se jen pevně napsaná odpověď.
Krok 8: spuštění serveru
Pozor:
Tento krok funguje až ve chvíli, kdy je aktivní virtuální prostředí a v něm je opravdu nainstalované FastAPI.
FastAPI CLI 
fastapi dev main.py
alternativa přes Uvicorn 
uvicorn main:app --reload
Vysvětlení příkazů „slovo od slova“
  • fastapi dev main.py = spusť vývojový server nad souborem main.py.
  • uvicorn = program, který umí spustit Python web server.
  • main:app = vezmi soubor main.py a v něm proměnnou app.
  • --reload = při změně souboru se server automaticky restartuje.
Po spuštění:
Server poběží typicky na adrese http://127.0.0.1:8000.
Krok 9: jak to celé zapadne do rezervačního systému
struktura projektu (plán) 
rezervacni_system/
├── .venv/
├── reservations_console.py
├── main.py
├── reservations.db
└── templates/
    └── index.html
Co bude dělat každý soubor a složka
  • .venv/ = virtuální prostředí tohoto projektu.
  • reservations_console.py = konzolový program, který zapisuje rezervace do databáze.
  • main.py = backend ve FastAPI.
  • reservations.db = SQLite databáze se všemi rezervacemi.
  • templates/index.html = HTML šablona stránky pro výpis dat.
Nejdůležitější princip:
Konzolový program i backend budou pracovat se stejnou databází reservations.db. To, co se uloží v konzoli, může backend zobrazit na webové stránce.
Tok dat v projektu
  1. Rezervace se zadá v konzoli.
  2. Python ji uloží do reservations.db.
  3. Backend ve FastAPI otevře stejnou databázi.
  4. Načte data a vrátí je webové stránce.
  5. Frontend je zobrazí v prohlížeči.

Co už po této lekci víš

  • Backend je Python část aplikace, která běží na serveru.
  • API je způsob komunikace mezi frontendem a backendem.
  • FastAPI je framework pro tvorbu backendu v Pythonu.
  • FastAPI není součást základní instalace Pythonu a musí se doinstalovat.
  • Virtuální prostředí odděluje balíčky konkrétního projektu od ostatních projektů.
  • Balíčky se instalují přes pip do aktivního prostředí projektu.
  • Backend bude používat stejnou databázi jako konzolový program.
  • V další lekci už vznikne skutečný server, který zobrazí rezervace na stránce.

Frontend × Backend × Databáze

  • Frontend = viditelná část aplikace v prohlížeči.
  • Backend = logická část aplikace běžící v Pythonu.
  • Databáze = uložiště dat.

V rezervačním systému frontend rezervace zobrazí, backend je načte nebo uloží a databáze je uchová.

Co je server?

Server je program, který běží a čeká na požadavky. Když nějaký požadavek přijde, server vrátí odpověď.

  • Požadavek může přijít z prohlížeče.
  • Backend rozhodne, co se má stát.
  • Server pošle odpověď zpět.

V tomto projektu bude server tvořit Python aplikace využívající FastAPI.

Co je API?

API je rozhraní, přes které spolu komunikují programy.

  • Frontend pošle požadavek.
  • Backend ho zpracuje.
  • Backend vrátí odpověď.

V praxi to znamená, že stránka si přes API říká o data nebo o provedení určité akce.

Proč stránka neotevře SQLite databázi sama?

  • SQLite je databázový soubor na disku.
  • Webová stránka v prohlížeči neumí používat Python modul sqlite3.
  • Práci s databází proto přebírá backend.
  • Backend pak data vrací stránce přes API.

GET a POST – dvě základní HTTP metody

  • GET = požadavek na získání dat.
  • POST = požadavek na odeslání nebo vytvoření dat.

Výpis rezervací bývá typicky GET. Přidání nové rezervace přes web by typicky bylo POST.

Proč budeme používat FastAPI?

  • Má přehledný zápis.
  • Rychle se v něm vytváří první funkční server.
  • Dobře se v něm vysvětluje vazba mezi URL adresou a Python funkcí.
  • Dobře zapadá do Python projektu s databází.

Proč FastAPI musíme instalovat?

FastAPI není součást základní instalace Pythonu. Je to externí balíček, který se do projektu přidává přes pip.

  • Nejdřív se balíček nainstaluje.
  • Teprve potom funguje import from fastapi import FastAPI.
  • Instalace patří do prostředí konkrétního projektu.

Co je virtuální prostředí?

Virtuální prostředí je oddělené prostředí pro jeden konkrétní Python projekt.

  • Má vlastní balíčky.
  • Odděluje projekt od ostatních projektů i od globální instalace Pythonu.
  • Po aktivaci používají příkazy python a pip právě toto prostředí.
Prakticky:
složka .venv je technické zázemí projektu, které drží jeho závislosti pohromadě.

Co je pip?

pip je správce Python balíčků.

  • Instaluje balíčky do prostředí.
  • Aktualizuje je nebo odstraňuje.
  • Když se použije python -m pip, instaluje se přes stejný Python, který je právě spuštěný.

Balíček × modul

  • Balíček = něco, co se instaluje přes pip.
  • Modul = něco, co se potom používá v kódu přes import.
python -m pip install "fastapi[standard]"
from fastapi import FastAPI

První řádek instaluje. Druhý řádek používá.

Co je route / endpoint?

Route nebo endpoint je konkrétní adresa na serveru, kterou backend obsluhuje.

  • / = domovská adresa
  • /reservations = například adresa pro výpis rezervací

Zápis @app.get("/") znamená: „Když někdo otevře adresu /, spusť tuto funkci.“

Co je JSON?

JSON je běžný textový formát pro předávání dat mezi frontendem a backendem. 

{
  "message": "Ahoj ze serveru"
}

Vypadá podobně jako Python slovník, ale na webu se používá jako standardní datový formát.

Co je Uvicorn?

Uvicorn je program, který umí spustit Python webovou aplikaci jako server.

  • Bez něj by soubor main.py nebyl webovým serverem.
  • Uvicorn spustí aplikaci a zpřístupní ji na webové adrese.

Co znamená 127.0.0.1 nebo localhost?

  • 127.0.0.1 = adresa vlastního počítače.
  • localhost = lidsky čitelný název pro totéž.
  • Když server běží lokálně, je dostupný jen na tomto počítači.

Proč oba programy vidí stejná data?

Protože konzolový program i backend pracují se stejným souborem reservations.db.

  • Konzole data zapisuje.
  • Backend stejná data čte.
  • Jedna databáze tedy slouží více částem téhož projektu.
