6.1.2 Backend – první stavební bloky serveru

V této kapitole začneme stavět backend našeho informačního systému. Nepůjdeme ještě na celý server najednou — nejdřív si opravdu pomalu vysvětlíme, jaké moduly Python potřebuje, jak si najde databázi a jak se k ní připojí.

Co se naučíš?

  • Importy v Pythonu – pochopíš, proč si na začátku souboru načítáme cizí moduly a k čemu budou v našem projektu sloužit.
  • Cestu k databázi – zjistíš, proč nepíšeme jen prostý název souboru, ale používáme Path(__file__).resolve().with_name(...).
  • Funkci connect() – uvidíš, jak vytvořit malé, přehledné místo pro připojení k SQLite databázi.
  • Myšlení po blocích – naučíš se stavět backend z malých částí, kterým rozumíš, místo slepého kopírování celého velkého souboru.
Tip: V této lekci ještě server nic „neobsluhuje“. Jen si připravujeme nástroje, bez kterých by další části backendu nemohly fungovat.

1) Proč backend nezačíná serverem, ale importy

Když píšeme Pythonový backend, nezačínáme tím, že hned spustíme server. Nejdřív musíme Pythonu říct, jaké nástroje budeme používat. K tomu slouží importy. Import znamená: „načti mi modul nebo konkrétní část modulu, protože ji budu v tomto souboru potřebovat“.

  1. Python spustí soubor odshora dolů.
  2. Na začátku si načte potřebné moduly.
  3. Teprve potom může používat jejich funkce, třídy a nástroje.
  4. Kdyby import chyběl, Python by daný název neznal a skončil by chybou.
  • Modul je hotová část Pythonu nebo knihovna s užitečnými nástroji.
  • Import je způsob, jak si tyto nástroje zpřístupnit v našem souboru.
  • Importy patří obvykle na začátek souboru.
2) První blok kódu – importy, které backend potřebuje

Na začátku našeho backendového souboru se nachází tento blok: 

import json
import sqlite3
from pathlib import Path
from http.server import HTTPServer, BaseHTTPRequestHandler
from urllib.parse import urlparse

Tento blok zatím jen říká, jaké nástroje budeme během dalších kapitol používat. Nemusíš se leknout, že je toho více. V této lekci si detailně vysvětlíme hlavně ty části, které potřebujeme hned teď: sqlite3 a Path. Ostatní si připravujeme dopředu.

2.1 Co dělá import json

Modul json budeme později používat při posílání dat ze serveru do stránky. Server vrátí seznam filmů jako JSON, aby mu JavaScript ve frontendu rozuměl.

  • Zatím ho ještě aktivně nepoužíváme.
  • Patří ale ke skutečnému backendu, který budeme postupně skládat.
2.2 Co dělá import sqlite3

Toto je pro tuto kapitolu velmi důležitý import. Modul sqlite3 nám umožní komunikovat s SQLite databází, tedy se souborem movies.db.

  • Pomocí něj databázi otevřeme.
  • Pomocí něj budeme spouštět SQL příkazy.
  • Pomocí něj budeme číst i zapisovat data.
2.3 Co znamená from pathlib import Path

Tady už je syntaktický tvar trochu jiný. Neimportujeme celý modul pathlib, ale bereme si z něj konkrétní nástroj: Path.

  • pathlib je modul pro práci s cestami k souborům.
  • Path je třída, která umí s cestami pracovat přehledněji než obyčejný text.
  • Budeme ji používat pro nalezení databáze i HTML souboru.
2.4 Co dělá from http.server import HTTPServer, BaseHTTPRequestHandler

Tyto dvě části budou důležité později, až budeme stavět samotný server. Zatím je stačí chápat takto:

  • HTTPServer vytvoří server, který bude čekat na požadavky.
  • BaseHTTPRequestHandler je základ pro třídu, která rozhodne, co server vrátí.
2.5 Co dělá from urllib.parse import urlparse

Tento nástroj budeme potřebovat při rozpoznávání adresy požadavku. Pomůže nám zjistit, jestli uživatel žádá HTML stránku, nebo například data na adrese /api/movies.

3) Druhý blok kódu – jak backend najde soubor movies.db

Teď přichází velmi důležitý řádek. Ten uloží do proměnné cestu k databázi: 

DB_FILE = Path(__file__).resolve().with_name("movies.db")

Tento jeden řádek je krátký, ale je v něm schováno hodně významu. Proto si ho rozebereme opravdu pomalu.

  1. DB_FILE je název proměnné.
  2. = znamená přiřazení hodnoty do proměnné.
  3. Path(...) vytváří objekt cesty k souboru.
  4. __file__ je speciální proměnná Pythonu – obsahuje cestu k právě spuštěnému souboru.
  5. resolve() převede cestu na úplnou, jistou cestu.
  6. with_name("movies.db") vezme stejnou složku a nahradí název souboru jménem movies.db.
  • Výsledkem je cesta k databázi, která leží ve stejné složce jako backendový soubor.
  • To je spolehlivější než napsat jen prosté "movies.db".
  • Když celý projekt přesuneš jinam, kód si databázi najde znovu správně.
Poznámka: Název DB_FILE je velkými písmeny, protože se chová jako důležitá konfigurační hodnota, kterou si připravíme jednou a pak ji používáme dál.
4) Třetí blok kódu – funkce connect()

Když už backend ví, kde databáze leží, může si připravit malou funkci pro připojení: 

def connect():
    return sqlite3.connect(str(DB_FILE))

Tato funkce je malá, ale velmi užitečná. Díky ní nebudeme řádek pro připojení k databázi psát stále dokola.

  1. def znamená „definuji funkci“.
  2. connect je název funkce.
  3. () znamená, že funkce zatím nepotřebuje žádné vstupní parametry.
  4. : ukončuje hlavičku funkce a otevírá její tělo.
  5. return znamená „vrať výsledek“.
  6. sqlite3.connect(...) otevře spojení s databází SQLite.
  7. str(DB_FILE) převede objekt cesty na obyčejný text, který sqlite3 bezpečně přijme.
  • Po zavolání connect() dostaneme spojení s databází.
  • Toto spojení si potom uložíme třeba do proměnné conn.
  • Stejnou funkci použijeme později při vytváření tabulky i při čtení dat.
5) Proč nepíšeme připojení k databázi pokaždé znovu

Začátečník často napíše připojení k databázi přímo tam, kde ho zrovna potřebuje. To sice může fungovat, ale kód se začne opakovat. A opakující se kód bývá zbytečně nepřehledný.

  1. Když máme funkci connect(), píšeme jen conn = connect().
  2. Když budeme chtít logiku připojení změnit, změníme ji jen na jednom místě.
  3. Kód je kratší a čitelnější.
  4. Pro větší projekty je to velmi důležitý návyk.
  • Jedna malá pomocná funkce může zjednodušit celý zbytek backendu.
  • Tohle je dobrý příklad programátorského pořádku.
  • Nejde jen o to, aby kód fungoval, ale aby se v něm člověk vyznal.
6) Co máš mít po této kapitole hotové

Po dokončení této kapitoly ještě server nebude obsluhovat web ani API. To je v pořádku. Teď jen připravujeme pevné základy, na kterých budeme v dalších kapitolách stavět.

  1. Na začátku souboru máš importy.
  2. Pod nimi máš proměnnou DB_FILE.
  3. Pod ní máš funkci connect().
  • Backend už ví, jaké nástroje bude používat.
  • Backend už ví, kde je databáze.
  • Backend už se umí k databázi připojit.
Pamatuj: zatím nepíšeme celý systém naráz. Stavíme si nářadí, které budeme za chvíli potřebovat.

Shrnutí

Co je import v Pythonu

Import je příkaz, kterým si do svého souboru zpřístupníš nástroje z jiného modulu. Python má spoustu hotových modulů už zabudovaných.

  • sqlite3 – práce s SQLite databází.
  • json – převod dat do formátu JSON.
  • pathlib – práce s cestami k souborům.
  • http.server – jednoduchý HTTP server.

Bez importu by Python názvy těchto nástrojů neznal.

Jak číst jednotlivé importy

V Pythonu existují dva časté způsoby importu:

  • import modul – načte celý modul.
  • from modul import Něco – vezme z modulu konkrétní část.

Proto máme například:

  • import sqlite3 – používáme celý modul.
  • from pathlib import Path – chceme konkrétně třídu Path.

Rozbor řádku s DB_FILE

Řádek DB_FILE = Path(__file__).resolve().with_name("movies.db") znamená:

  • vezmi právě spuštěný soubor,
  • zjisti jeho přesnou cestu,

Rozbor funkce connect()

Funkce connect() je malá pomocná funkce. Její jediný úkol je otevřít spojení s databází a toto spojení vrátit.

  • def – začátek definice funkce.
  • connect – název funkce.
  • return – vrácení výsledku.
  • sqlite3.connect(...) – otevření databáze.
  • str(DB_FILE) – převod cesty na text.

Později ji použijeme třeba takto: conn = connect()
